# 数字孪生融合分析页 PRD V1.7

文档版本：V1.7  
更新日期：2026-05-24  
所属产品：慢病管理系统 - 医生 PC 端  
页面位置：患者详情页一级 Tab  
覆盖病种：睡眠呼吸障碍、慢阻肺、2 型糖尿病及两两 / 三重共病  
产品定位：基于患者现有慢病数据，构建器官状态映射、共病影响链路和干预优先级推演，用于医生快速理解风险来源、向患者解释病情，并增强商务演示中的数字孪生感知。

## 一、版本优化说明

本版本在 V1.5.2 基础上做以下优化：

1. 将“3D 人体展示”升级为“数字孪生建模逻辑”，明确数据层、疾病层、器官层、推演层的映射关系。
2. 增加数据可信度与数据完整性模块，避免数字孪生结论看起来像无依据输出。
3. 将共病分析从静态文案表格升级为“影响链路 + 器官联动 + 干预优先级”。
4. 保留演示噱头，但降低一期技术风险，明确 V1 使用 2D 高清数字孪生底图 + SVG 器官热区 + 发光动效，V2 再升级 Three.js / GLB。
5. 将干预建议统一改为“建议医生评估 / 建议优先关注”，避免越过医生决策边界。
6. 技术方案适配当前医生 PC 原型：原生 JS、现有样式体系、Chart.js、共享 mock 数据。

## 二、需求背景与目标

### 2.1 背景

慢病管理产品已经具备患者档案、指标记录、睡眠报告、血氧报告、血糖分析、慢阻肺分析、管理方案、预警和随访闭环。当前缺少一个面向医生和客户演示的“高阶总览层”，能够把分散数据转译成更直观的身体状态和共病影响。

数字孪生融合分析页承担这个展示层职责：不新增采集、不替代诊断、不生成处方，而是把现有慢病数据包装成器官状态、风险链路和干预优先级，让医生能更快解释“为什么这个患者需要先处理某个风险”。

### 2.2 核心目标

1. 演示感：通过高清蓝色人体底图、器官发光热区、流动链路、健康分变化，形成明显数字孪生卖点。
2. 可信度：所有指标来自现有数据源，并展示数据来源、更新时间、完整率和可信度。
3. 可解释：把指标异常映射为器官影响，再映射为共病链路和干预建议。
4. 可落地：一期复用当前医生 PC 技术栈，不引入 Vue / Element Plus / ECharts 等新体系。
5. 合规安全：页面输出为辅助解释和管理建议，诊断、用药、设备治疗调整均由医生判断。

## 三、产品边界

### 3.1 做什么

1. 在患者详情页新增 / 完善“数字孪生融合分析”一级 Tab。
2. 自动读取患者疾病标签，匹配单病种或共病融合分析。
3. 基于现有健康风险分计算数字孪生健康分，并将分值拆解为器官状态、共病影响和数据可信度。
4. 左侧展示 2D 高清数字孪生人体画布，支持器官热区高亮、点击聚焦、局部放大、共病链路展示。
5. 右侧展示健康分、核心指标、数据可信度、器官影响、共病链路和干预建议。
6. 支持演示模式，用预设病例一键切换轻 / 中 / 重 / 共病场景。

### 3.2 不做什么

1. 不作为医学诊断结果，不输出“确诊”结论。
2. 不直接给患者处方、治疗指令或设备参数。
3. 不新增真实采集设备，一期不强依赖真实 3D 人体模型库。
4. 不做独立病种切换器，病种来自患者详情头部疾病标签。
5. 不重复患者基础信息卡，不做导出报告按钮。

## 四、数字孪生建模逻辑

数字孪生页采用四层模型表达：

| 层级 | 内容 | 页面体现 |
| --- | --- | --- |
| 数据层 | 睡眠报告、血氧、AHI、ODI、CAT、血糖、TIR、糖化估算、症状、设备同步 | 指标卡、数据来源、完整率 |
| 疾病层 | 睡眠呼吸障碍、慢阻肺、2 型糖尿病 | 病种识别、单病种 / 共病匹配 |
| 器官层 | 气道、肺、心脏、血管、大脑、胰腺、肝脏、肾脏 | 人体器官高亮、器官状态卡 |
| 推演层 | 当前状态、共病影响链路、优先处理方向、干预后预期 | 共病链路、干预建议、演示模式变化 |

页面核心叙事为：

```text
患者数据异常 -> 疾病风险分级 -> 器官状态映射 -> 共病影响链路 -> 干预优先级建议
```

### 5.3 开发实现说明

1. 第十二章“单病种分析内容”不是独立页面板块，而是模块 4“器官影响 / 共病链路”和模块 5“干预优先级建议”的单病种配置来源。
2. 第十三章“共病融合分析”不是独立页面板块，而是患者存在两种及以上支持病种时，对模块 4 和模块 5 的增强配置来源。
3. 第八章“数字孪生健康分”不替代全局健康风险分，只在现有健康风险分基础上做器官化解释和必要修正。
4. 左侧 2D 人体画布是表现层，不单独生成医学结论；所有高亮、放大和链路线均由右侧指标、分值拆解、器官影响和共病链路驱动。
5. 当数据可信度不足时，优先执行第 10.5 节降级规则，再渲染第十二章或第十三章文案。

## 七、数据来源与可信度

### 7.1 可用数据源

一期仅使用当前项目已有数据：

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 睡眠报告 | `sleepReports` | AHI、ODI、最低血氧、平均血氧 |
| 血氧报告 | `oxygenReports` | 平均血氧、最低血氧、Ts90、低氧时长 |
| CAT 评分 | `scaleRecords` | COPD 症状负担分级 |
| 血糖数据 | `getGlucoseData(patient)` / 现有血糖分析数据 | 平均血糖、TIR、糖化估算、达标率 |
| 症状记录 | `copdSymptoms`、血糖症状 mock | 症状负担和干预建议补充 |
| 设备状态 | `patient.dataStatus`、设备同步时间 | 数据完整性与可信度 |

### 7.2 数据可信度展示

右侧面板在健康分下方展示“数据可信度条”。

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 数据完整率 | 76% | 近 7 / 14 天核心数据覆盖情况 |
| 最近更新 | 2026-05-22 07:30 | 最近一条核心数据时间 |
| 数据来源 | 睡眠监测仪、血氧仪、血糖仪、CAT 问卷 | 用 tag 展示 |
| 可信度等级 | 高 / 中 / 低 | 根据完整率和关键数据缺失计算 |
| 缺失提示 | 近 3 天血氧缺失 | 仅在缺失时展示 |

可信度规则：

| 条件 | 可信度 |
| --- | --- |
| 核心数据完整率 >= 85%，且无关键病种核心指标缺失 | 高 |
| 核心数据完整率 60%-84%，或存在 1 类关键指标缺失 | 中 |
| 核心数据完整率 < 60%，或存在 2 类及以上关键指标缺失 | 低 |

可信度低时，页面文案需改为“当前数据不足，建议补齐后再评估”，避免输出强结论。

## 八、病情分级与数字孪生健康分

### 8.0 与健康风险分的关系

数字孪生健康分不另起一套独立评分体系，必须承接患者已有健康风险分。

现有项目评分口径：

1. 患者首次健康筛查生成初始健康风险分。
2. 分值范围 0-100，分数越高表示近期慢病管理状态越稳定。
3. 后续健康风险分在筛查分基础上，叠加指标记录、睡眠报告、症状、用药、设备同步和数据完整度。
4. 医生 PC 患者详情头部展示的 `patient.riskScore` 是当前患者管理视角下的健康风险分；患者健康筛查 Tab 展示的 `patient.screening.score` 是最近一次筛查基础分。

数字孪生健康分定位：

```text
数字孪生健康分 = 当前健康风险分的器官化解释版本
```

也就是说，数字孪生页不重新替代全局健康风险分，而是把已有分值解释成：

1. 哪些器官正在扣分。
2. 哪些疾病链路造成协同影响。
3. 哪些指标证据支撑该扣分。
4. 哪些数据缺失导致可信度下降。

### 8.0.1 基础分取值优先级

| 优先级 | 字段 | 含义 | 使用场景 |
| --- | --- | --- | --- |
| P1 | `patient.riskScore` | 当前健康风险分，已融合最近管理数据 | 医生端患者详情默认使用 |
| P2 | `patient.screening.score` | 最近一次健康筛查基础分 | 无当前健康风险分时兜底 |
| P3 | 100 | 理论满分 | 仅演示模板或无任何评分数据时使用 |

基础分取值：

```text
基础健康风险分 = patient.riskScore || patient.screening.score || 100
```

### 8.0.2 分数口径要求

1. 若某个指标已经进入当前健康风险分，不应在数字孪生页重复扣同一笔分。
2. 数字孪生页可以对“器官影响”和“共病协同”做解释性拆解。
3. 只有当数字孪生页发现当前健康风险分未覆盖的新证据时，才允许产生额外修正扣分。
4. 数据可信度不足时，可增加可信度扣分或限制结论强度，但不能单独生成疾病诊断结论。
5. 若存在紧急预警，总分上限继承全局健康风险分规则：最高不超过 49 分；存在重要预警未处理时最高不超过 69 分。

### 8.1 OSA 分级

优先级：AHI > 最低血氧 > ODI。

| 等级 | AHI 次/小时 | 最低血氧 | ODI 次/小时 | 扣分 |
| --- | --- | --- | --- | --- |
| 正常 | < 5 | >= 90% | < 5 | 0 |
| 轻度 | 5-14 | 85%-89% | 5-14 | 12 |
| 中度 | 15-29 | 80%-84% | 15-29 | 25 |
| 重度 | >= 30 | < 80% | >= 30 | 40 |

### 8.2 COPD 分级

优先级：CAT > 平均血氧 > 夜间最低血氧。

| 等级 | CAT | 平均血氧 | 夜间最低血氧 | 扣分 |
| --- | --- | --- | --- | --- |
| 正常 | 0-10 | >= 95% | >= 90% | 0 |
| 轻度 | 11-20 | 90%-94% | 85%-89% | 10 |
| 中度 | 21-30 | 85%-89% | 80%-84% | 22 |
| 重度 | 31-40 | < 85% | < 80% | 35 |

### 8.3 2 型糖尿病分级

优先级：TIR > 糖化估算 / HbA1c > 平均血糖。

| 等级 | TIR | 糖化估算 / HbA1c | 平均血糖 | 扣分 |
| --- | --- | --- | --- | --- |
| 正常 | >= 70% | < 7.0% | < 7.8 mmol/L | 0 |
| 轻度 | 50%-69% | 7.0%-7.4% | 7.8-8.9 mmol/L | 15 |
| 中度 | 30%-49% | 7.5%-8.9% | 9.0-11.0 mmol/L | 28 |
| 重度 | < 30% | >= 9.0% | >= 11.1 mmol/L | 42 |

### 8.4 数字孪生健康分公式

```text
基础健康风险分 = patient.riskScore || patient.screening.score || 100

器官影响修正 = 0-15
共病协同修正 = 0-15
数据可信度修正 = 0-10
改善加分 = 0-5

数字孪生健康分 = clamp(
  基础健康风险分
  - 器官影响修正
  - 共病协同修正
  - 数据可信度修正
  + 改善加分,
  0,
  100
)
```

| 分数 | 风险等级 | 展示色 |
| --- | --- | --- |
| 85-100 | 稳定 | 绿色 |
| 70-84 | 轻度风险 | 黄色 |
| 50-69 | 中高风险 | 橙色 |
| 0-49 | 高危 | 红色 |

健康分卡片必须支持拆解展示：

1. 基础健康风险分：来自患者详情当前健康风险分或最近一次筛查分。
2. 器官影响修正：近期指标异常映射到器官后的新增影响。
3. 共病协同修正：多病种之间互相放大的风险影响。
4. 数据可信度修正：关键数据缺失、设备未同步、记录不足导致的可信度下降。
5. 改善加分：近期指标改善或任务执行恢复带来的小幅加分。
6. 当前最主要扣分来源。

### 8.5 修正项规则

| 修正项 | 轻度 | 中度 | 重度 | 说明 |
| --- | ---: | ---: | ---: | --- |
| 器官影响修正 | 3 | 8 | 15 | 仅对当前健康风险分未充分体现的近期器官影响追加 |
| 共病协同修正 | 3 | 8 | 15 | 两病共病最高 8，三重共病最高 15 |
| 数据可信度修正 | 3 | 6 | 10 | 关键数据越缺失，扣分越高 |
| 改善加分 | 1 | 3 | 5 | 近 7 天关键指标改善或任务恢复执行 |

若当前健康风险分已明确包含某项扣分，数字孪生页应优先展示“解释拆解”，不重复追加修正。

### 8.6 展示示例

```text
数字孪生健康分：62

基础健康风险分：78
器官影响修正：-8（夜间最低血氧 86%，肺部与心脏负荷提示）
共病协同修正：-5（睡眠低氧与血糖波动存在协同风险）
数据可信度修正：-3（设备同步存在 1 天缺失）
改善加分：0
```

页面文案：

```text
该分值基于当前健康风险分进行器官化解释。分值变化主要来自近期低氧负担、共病协同影响和数据完整性，不作为独立诊断结论。
```

## 九、2D 数字孪生人体画布

### 9.1 一期视觉方案

一期采用 2D 高清数字孪生底图作为主视觉，不强依赖真实 GLB 模型。目标不是做普通静态图片，而是做“活的数据模型画布”。

画布分为三层：

| 层级 | 内容 | 目的 |
| --- | --- | --- |
| 人体底图层 | 高清蓝色透明人体图，带内脏、骨骼、科技背景 | 提供高端医学科技感 |
| 器官热区层 | SVG 覆盖气道、左右肺、心脏、肝脏、胰腺、肾脏、血管等区域 | 支持 hover、click、聚焦、放大 |
| 数据链路层 | SVG 发光曲线、流动光点、器官之间的影响路径 | 表达疾病和共病的动态影响 |

视觉基调：

1. 默认状态以蓝白色为主，保持冷静医学科技感。
2. 正常器官仅保留微弱蓝白呼吸光。
3. 异常器官按风险等级叠加黄色、橙色、红色光效。
4. 重度风险器官使用低频脉冲，不做高频闪烁，避免干扰医生阅读。
5. 线条、光点、背景 HUD 圆环保持细线风格，避免页面游戏化。

## 十、右侧分析面板

右侧面板固定从上到下：

1. 数字孪生健康分。
2. 数据可信度。
3. 核心指标 4 项。
4. 器官影响 / 共病链路。
5. 干预优先级建议。

### 10.1 核心指标规则

数字孪生页的核心指标不重新定义医学口径，必须复用已建设的单病种深度分析页指标口径。数字孪生页只负责从单病种页的指标池中抽取 4 个最能解释“器官风险 / 共病链路”的指标，并增加人体联动。

复用原则：

1. 指标名称、统计周期、单位、目标值、状态标签，优先复用对应深度分析页。
2. 数字孪生页不新增与单病种页冲突的阈值。
3. 当单病种页已有指标卡时，数字孪生页只读取结果，不重复计算。
4. 当患者有 CGM 数据时，糖尿病优先使用 CGM 指标；无 CGM 时降级使用指尖血糖指标。
5. 当某项指标缺失时，不硬凑 4 项；允许展示“数据不足”卡，并触发第 10.5 节降级规则。

#### 10.1.1 指标口径复用来源

| 病种 | 可复用页面 / 文档 | 已有核心卡片 | 数字孪生复用方式 |
| --- | --- | --- | --- |
| OSA / 睡眠呼吸障碍 | 睡眠深度分析 | AHI、ODI、最低血氧、呼吸事件、报告有效性 | 复用最新有效睡眠报告和多晚趋势口径；P11 无 AHI 时不展示 AHI，降级为 ODI + 最低血氧 |
| COPD / 慢阻肺 | 慢阻肺深度分析 | 平均血氧、最低血氧、ODI、低氧次数、低氧时长、Ts90%，以及 CAT / mMRC 问卷 | 复用血氧卡片和 CAT 问卷口径；数字孪生页抽取症状负担 + 低氧负担做器官解释 |
| 糖尿病 | 血糖深度分析 | CGM：TIR、平均血糖、CV、TAR/TBR、eA1c；指尖：空腹、餐后2h、达标率、最高血糖、低血糖次数 | 优先复用 CGM 5 项；无 CGM 时使用指尖血糖 5 项中的 4 项 |

#### 10.1.2 数字孪生页默认 4 项展示

| 场景 | 4 项核心指标 |
| --- | --- |
| OSA | AHI、最低血氧、ODI、报告有效性 / 有效监测时长 |
| COPD | CAT、平均血氧、夜间最低血氧、Ts90 / 低氧时长 |
| 糖尿病（有 CGM） | TIR、平均血糖、CV / TAR/TBR、eA1c |
| 糖尿病（无 CGM，仅指尖） | 空腹血糖均值、餐后2h血糖均值、血糖达标率、低血糖次数 / 最高血糖记录 |
| OSA + 糖尿病 | AHI、最低血氧、TIR、平均血糖 |
| OSA + COPD | AHI、最低血氧、CAT、Ts90 / 低氧时长 |
| COPD + 糖尿病 | CAT、平均血氧、TIR、平均血糖 |
| 三重共病 | 最低血氧、AHI、CAT、TIR |

指标卡点击后，左侧定位对应器官；器官点击后，右侧滚动到对应链路或分析卡。

#### 10.1.3 具体指标展示口径

| 指标 | 复用来源 | 展示口径 | 目标 / 阈值 | 缺失降级 |
| --- | --- | --- | --- | --- |
| AHI | 睡眠深度分析最新有效报告 | 当晚 AHI，单位次/小时；支持多晚趋势 | 复用睡眠页 AHI 阈值和状态 | P11 或设备未输出 AHI 时隐藏，改用“睡眠呼吸事件 / 报告摘要” |
| ODI | 睡眠深度分析 / 慢阻肺深度分析 | 氧减指数，单位次/小时 | OSA 场景复用睡眠页；COPD 场景复用慢阻肺页 `<5` 目标 | 无 ODI 时降级为最低血氧 + 低氧时长 |
| 最低血氧 | 睡眠深度分析 / 慢阻肺深度分析 | 最新有效报告最低 SpO2 | 复用 `>=90%` 目标 | 无数据时显示“夜间血氧数据不足” |
| 报告有效性 / 有效监测时长 | 睡眠深度分析 | 最新报告状态、有效监测时长、设备来源 | 复用报告状态 | 数据不足时作为第 4 项提示补测 |
| CAT | 慢阻肺深度分析问卷模块 | 最新 CAT 总分、影响等级、较上次变化 | 复用 CAT 评分和 MCID 变化规则 | 无 CAT 时显示“未完成 CAT 问卷”，不输出症状负担判断 |
| 平均血氧 | 慢阻肺深度分析 | P11 血氧均值 | `>=95%` 正常，复用慢阻肺页三色规则 | 无血氧报告时隐藏或置灰 |
| Ts90 / 低氧时长 | 慢阻肺深度分析 | 优先展示 Ts90%，辅助展示低氧累计分钟 | Ts90 `<5%`，低氧时长 `0分钟` 为目标 | 缺失时仅保留 CAT 或症状提示 |
| TIR | 血糖深度分析 CGM 卡片 | 近 14 天目标范围时间占比 | `>=70%` 达标 | 无 CGM 时不展示，改用指尖血糖达标率 |
| 平均血糖 | 血糖深度分析 CGM 卡片 | 近 14 天 CGM 有效点位算术均值 | `<7.8 mmol/L` | 无 CGM 时可降级为空腹 / 餐后均值 |
| CV / TAR/TBR | 血糖深度分析 CGM 卡片 | 优先展示 CV；如高低血糖占比更异常，则展示 TAR/TBR | CV `<36%`；TAR/TBR 复用血糖页区间阈值 | 无 CGM 时改用最高血糖记录或低血糖次数 |
| eA1c | 血糖深度分析 CGM 卡片 | 14 天 CGM 平均血糖算法估算 | `<7.0%` | 无 CGM 时隐藏，不用指尖数据硬算 |
| 空腹血糖均值 | 血糖深度分析指尖卡片 | 近 7 天有效空腹记录均值 | `4.4-7.0 mmol/L` | 有效记录少于 3 条时置灰 |
| 餐后2h血糖均值 | 血糖深度分析指尖卡片 | 近 7 天有效餐后2h记录均值 | `<10.0 mmol/L` | 有效记录少于 3 条时置灰 |
| 血糖达标率 | 血糖深度分析指尖卡片 | 近 30 天有效指尖记录中处于 3.9-10.0 mmol/L 的占比 | `>=70%` | 有效记录不足时显示“记录不足” |
| 低血糖次数 / 最高血糖记录 | 血糖深度分析指尖卡片 | 取当前更异常的一项展示 | 复用血糖页状态标签 | 均无异常时展示低血糖次数 |

### 10.4 单病种指标-器官映射

单病种分析不能只展示固定文案，必须把关键指标映射到左侧人体器官热区。指标卡点击后，左侧按下表高亮器官；器官点击后，右侧反查相关指标。

| 病种 | 指标 | 主高亮器官 | 弱高亮器官 | 解释方向 |
| --- | --- | --- | --- | --- |
| OSA | AHI | 气道、肺部 | 心脏、大脑 | 呼吸暂停频次越高，提示上气道阻塞与夜间通气负担越重 |
| OSA | 最低血氧 | 肺部 | 心脏、大脑、血管 | 最低血氧下降提示氧合不足，需关注心脑血管负荷 |

#### 10.4.1 器官状态来源表

器官状态不使用固定文案写死，应由该器官关联核心指标的分级结果推导。

计算原则：

1. 每个器官维护一组关联核心指标。
2. 每个指标复用对应单病种深度分析页的既有分级口径。
3. 器官状态取其关联指标中的最高异常等级。
4. 若关键指标缺失，则器官状态降级为“数据不足”，不输出强异常结论。
5. 若多个病种共同影响同一器官，则取最高风险指标，并标记为“共病叠加影响”。

| 器官 | 关联指标 | 状态来源 |
| --- | --- | --- |
| 肺部 | 最低血氧、平均血氧、ODI、CAT、Ts90 | 慢阻肺深度分析 / 睡眠深度分析 |
| 气道 | AHI、ODI、呼吸事件、CPAP 使用 / 有效监测时长 | 睡眠深度分析 |
| 心脏 | 最低血氧、低氧时长、Ts90、心血管风险提示 | 睡眠深度分析 / 慢阻肺深度分析 |
| 胰腺 | 平均血糖、TIR、eA1c、血糖达标率 | 血糖深度分析 |
| 血管 | CV、TAR/TBR、平均血糖、低血糖次数 | 血糖深度分析 |
| 肾脏 | TIR、eA1c、长期血糖达标率、并发症风险 | 血糖深度分析 |

示例：

```text
肺部关联指标：
最低血氧 86% = 中度异常
平均血氧 93% = 轻度异常
ODI 18.2 = 重度异常
CAT 28 = 重度异常
Ts90 12% = 重度异常

肺部状态 = 重度异常
```

页面展示要求：

1. 器官状态展示为“正常 / 轻度异常 / 中度异常 / 重度异常 / 数据不足”。
2. 器官状态旁必须展示至少 1 条触发证据，例如“最低血氧 86%”。
3. 器官状态不可脱离指标证据单独展示。
4. 当状态来自共病叠加时，需在器官影响卡或链路卡中标记“共病叠加影响”。

### 10.5 数据不足降级规则

当核心数据缺失时，页面不能强行输出器官损伤或共病链路强结论，需要按可信度降级展示。

| 缺失场景 | 降级规则 | 页面文案方向 |
| --- | --- | --- |
| OSA 缺少 AHI | 不输出 OSA 严重程度强判断，保留血氧相关提示 | 当前缺少 AHI 数据，无法完整判断睡眠呼吸事件负担 |
| OSA 缺少最低血氧 / ODI | 不播放低氧链路，仅提示通气事件待补充 | 当前低氧数据不足，建议补齐睡眠血氧或设备同步 |
| COPD 缺少血氧报告 | 不输出低氧心肺负荷结论，只展示 CAT 症状负担 | 当前血氧连续缺失，肺部低氧负担可信度不足 |
| COPD 缺少 CAT | 不输出症状负担等级，仅展示血氧趋势 | 当前缺少 CAT 评分，建议提醒患者完成问卷 |
| 糖尿病缺少 TIR | 不输出动态血糖稳定性结论，降级使用平均血糖 / 达标率 | 当前无 CGM 数据，TIR 判断不可用 |
| 糖尿病缺少平均血糖 | 不输出代谢负担强结论，提示补齐记录 | 当前血糖记录不足，建议补充空腹和餐后血糖 |
| 共病任一核心指标缺失 | 共病链路降级为“可能相关”，不使用“加重 / 造成”等强表达 | 当前共病链路可信度中低，建议补齐关键指标后再评估 |

降级文案统一表达：

```text
当前数据不足，数字孪生分析仅展示可用数据下的风险提示。建议优先补齐 [缺失数据] 后再复核器官影响与共病链路。
```

## 十一、交互状态与事件规则

本页交互目标不是让医生反复操作复杂控件，而是让医生通过“点指标、点器官、点链路”快速理解患者风险来源。交互必须保持低认知负担、低干扰、强解释性。

### 11.1 页面默认态

进入数字孪生融合分析 Tab 后，系统自动进入默认态。

| 场景 | 默认聚焦 | 左侧画布 | 右侧模块 |
| --- | --- | --- | --- |
| 单病种 | 该病种最高风险器官 | 主器官强高亮，关联器官弱高亮 | 模块 4 展示该病种最高风险器官影响；模块 5 展示单病种干预建议 |
| 双病共病 | P1 主链路起点器官 | 主链路播放 1 次，链路上器官按主次高亮 | 模块 4 展示 P1 主链路；模块 5 展示共病干预优先级 |
| 三重共病 | 低氧负担链路 | 默认播放“气道 -> 肺部 -> 心脏”链路 | 模块 4 置顶低氧负担链路，后续展示代谢和并发链路 |
| 数据不足 | 数据缺失影响最大的器官 | 相关器官弱化，显示数据不足标记 | 模块 2 和模块 4 显示补数优先提示 |
| 无支持病种 | 无聚焦 | 显示底图低亮状态，不显示异常链路 | 展示空状态和可用数据说明 |

默认态加载顺序：

1. 读取疾病标签和筛查确认疾病。
2. 计算健康分、可信度、病种等级、核心指标。
3. 选择默认聚焦对象。
4. 渲染左侧底图、热区、链路。
5. 渲染右侧 5 个模块。

### 11.2 Hover 态

Hover 只做轻提示，不改变右侧主内容。

| Hover 对象 | 反馈 |
| --- | --- |
| 器官热区 | 热区亮度提升，显示小浮层：器官名称、风险等级、关联指标 |
| 指标卡 | 卡片边框增亮，左侧关联器官弱高亮 |
| 健康分扣分项 | 扣分项背景增亮，左侧相关器官弱高亮 |
| 共病链路卡 | 左侧对应链路弱显示，流动光点暂停在链路起点 |

Hover 浮层规则：

1. 浮层不超过 2 行文字。
2. 只显示事实信息，不显示新的医学判断。
3. 鼠标离开后 150ms 内淡出。
4. Hover 不写入选中态，不影响当前聚焦对象。

### 11.3 Click 选中态

点击会改变当前聚焦对象，并同步更新左右两侧内容。

| 点击对象 | 当前聚焦对象 | 左侧变化 | 右侧变化 |
| --- | --- | --- | --- |
| 器官热区 | 器官 | 目标器官放大并强高亮，关联器官弱高亮 | 模块 4 滚动到该器官影响；模块 3 高亮关联指标 |
| 指标卡 | 指标 | 关联器官高亮，播放指标对应链路 | 模块 4 展示该指标影响解释 |
| 健康分扣分项 | 扣分原因 | 扣分涉及器官和链路高亮 | 模块 4 展示扣分解释，模块 2/3 标出证据 |
| 共病链路卡 | 链路 | 播放链路流动光效，链路器官按顺序点亮 | 模块 4 展开链路详情 |
| 演示模板 | 模板 | 刷新所有器官状态和链路 | 刷新健康分、指标、链路、建议 |

选中态视觉：

1. 当前器官放大 1.04-1.10 倍，放大中心为热区中心。
2. 当前器官使用强高亮；关联器官使用弱高亮；无关器官降到默认蓝白低亮。
3. 当前指标卡 / 扣分项 / 链路卡保留 active 样式。
4. 同一时间只允许一个主聚焦对象。

### 11.4 选中优先级

当多个交互来源同时存在时，按以下优先级决定主聚焦对象：

```text
演示模板切换 > 用户点击对象 > 默认主链路 > Hover 临时态
```

具体规则：

1. 用户点击器官后，再 hover 指标卡，只显示临时弱高亮，不覆盖器官选中态。
2. 用户点击指标卡后，再点击器官，器官成为新的主聚焦对象。
3. 用户点击健康分扣分项后，右侧模块 4 展示扣分解释；再点击共病链路卡，则切换到链路解释。
4. 演示模板切换会清空所有手动聚焦状态，并进入该模板默认态。
5. 切换患者或离开患者详情页时，清空当前聚焦状态。

### 11.5 复位规则

| 触发 | 行为 |
| --- | --- |
| 点击画布空白区域 | 回到当前患者 / 当前演示模板默认态 |
| 点击“显示全部影响” | 所有异常器官按风险等级高亮，模块 4 展示链路列表 |
| 再次点击已选中的器官 / 指标 | 取消手动聚焦，回到默认态 |
| 切换详情 Tab | 保留最近一次系统默认态，不保留手动聚焦 |
| 切换患者 | 重新计算并进入新患者默认态 |
| 切换演示模式开关 | 清空真实患者聚焦态，进入演示模板默认态 |

### 11.6 链路播放规则

链路播放用于表达“疾病或指标如何影响器官”，不是装饰动画。

| 链路类型 | 播放方式 | 时长 |
| --- | --- | --- |
| 单病种指标链路 | 从指标相关器官起点到影响器官播放 1 次 | 1.2-1.6s |
| 共病主链路 | 按器官顺序逐段播放，终点器官强高亮 | 1.8-2.4s |
| 三重共病链路 | 主链路先播放，次链路延迟 300ms 弱播放 | 2.4-3.2s |
| 数据不足链路 | 不播放流动光，仅显示虚线弱提示 | - |

动画规则：

1. 流动光点不超过 2 个，避免视觉噪音。
2. 链路线宽 1-2px，异常链路可增加外发光。
3. 链路播放结束后保留静态高亮状态。
4. 医生再次点击链路卡时可重新播放。

### 11.7 动效参数

| 动效 | 参数 |
| --- | --- |
| 器官 hover 增亮 | 120-180ms ease-out |
| 器官 click 放大 | 180-240ms ease-out |
| 器官呼吸光 | 2.4-3.2s 循环，透明度变化不超过 18% |
| 重度风险脉冲 | 1.4-1.8s 循环，禁止高频闪烁 |
| 链路流动 | 1.2-3.2s，按链路复杂度调整 |
| 右侧模块滚动定位 | 200-300ms smooth scroll |
| 浮层出现 / 消失 | 120ms / 150ms |

无障碍与舒适性：

1. 动画不使用高频闪烁。
2. 红色异常必须同时配合文字标签，不能只靠颜色表达。
3. 若浏览器开启减少动态效果，可关闭链路流动，仅保留静态高亮。

### 11.8 数据不足态

数据不足时需要明确告诉医生“哪些结论不能强输出”。

| 区域 | 展示规则 |
| --- | --- |

数据不足态文案示例：

```text
当前血氧连续缺失，肺部低氧负担可信度不足。建议优先恢复血氧设备同步或创建 3 天复测任务，再复核心肺影响链路。
```

## 十二、单病种分析内容

### 12.1 OSA

默认器官：气道、肺、心脏、大脑、血管。

| 模块 | 轻度 | 中度 | 重度 |
| --- | --- | --- | --- |
| 缺氧对大脑影响 | 间歇性轻度缺氧，可能造成睡眠碎片化和日间困倦 | 反复中度缺氧破坏睡眠结构，注意力和记忆力下降风险升高 | 重度持续缺氧提示认知和脑血管风险需重点评估 |
| 缺氧对心脏影响 | 交感兴奋，心率血压负荷轻度增加 | 肺动脉压力和心室负荷可能加重 | 心律异常、心衰等风险需医生重点评估 |
| 干预建议 | 建议评估生活方式、体重管理和睡眠姿势 | 建议评估通气治疗依从性和压力设置 | 建议优先复核通气方案，必要时线下进一步评估 |

### 12.2 COPD

默认器官：气道、支气管、肺、心脏。

| 模块 | 轻度 | 中度 | 重度 |
| --- | --- | --- | --- |
| 气道阻塞影响 | 活动后轻微气短，气流受限提示需持续监测 | 日常活动气喘明显，急性加重风险升高 | 静息呼吸困难或低氧时需优先处理 |
| 全身炎症影响 | 气道局部炎症为主 | 慢性炎症可能影响体力和免疫状态 | 多系统影响和感染风险需重点关注 |
| 干预建议 | 建议坚持戒烟、疫苗防护和基础肺康复 | 建议复核吸入治疗、血氧和症状趋势 | 建议医生评估是否需要强化治疗、氧疗或线下处理 |

### 12.3 2 型糖尿病

默认器官：胰腺、肝脏、肾脏、血管、心脏。

| 模块 | 轻度 | 中度 | 重度 |
| --- | --- | --- | --- |
| 高血糖损伤胰腺 | 胰岛功能轻度负担增加 | 胰岛功能下降，降糖能力减弱 | 胰岛功能衰退风险高，需重点评估方案 |
| 高血糖损伤肾脏 | 长期风险开始累积 | 肾脏和眼底筛查优先级升高 | 并发症筛查和线下评估优先级高 |
| 干预建议 | 建议优化饮食、运动和记录完整性 | 建议医生复核降糖方案和餐后血糖控制 | 建议医生评估强化控糖和并发症筛查 |

## 十三、共病融合分析

共病页必须展示“影响链路”，不是简单并列多个病种。

### 13.0 共病链路优先级规则

多病种患者可能同时存在多条链路，页面不能全部同等强调。系统按以下优先级选择主链路、次链路和观察链路。

| 优先级 | 判定逻辑 | 展示方式 |
| --- | --- | --- |
| P1 主链路 | 涉及最高风险等级病种，且核心指标异常明确 | 左侧强高亮，右侧置顶展示，链路线播放 |
| P2 次链路 | 涉及中度及以上异常，或与主链路存在共病协同 | 左侧弱高亮，右侧第二展示 |
| P3 观察链路 | 仅轻度异常，或数据可信度不足 | 以标签 / 提示展示，不播放强链路 |

链路排序规则：

1. 先看风险等级：重度 > 中度 > 轻度 > 正常。
2. 再看数据可信度：高 > 中 > 低。
3. 再看共病协同扣分：扣分高的链路优先。
4. 若仍相同，按疾病安全优先级排序：低氧相关 > 心肺负荷 > 血糖代谢 > 慢性并发症。

### 13.0.1 共病组合链路优先级

| 共病组合 | 主链路 | 次链路 | 观察链路 |
| --- | --- | --- | --- |
| OSA + 糖尿病 | 夜间低氧 -> 胰岛素抵抗 -> TIR / 平均血糖异常 | 高血糖 -> 血管内皮损伤 | 气道炎症 / 黏膜水肿加重呼吸阻塞 |
| OSA + COPD | 气道阻塞 + 气流受限 -> 双重低氧 -> 心肺负荷 | 夜间低氧 -> 肺动脉压力风险 | 睡眠质量下降 -> 症状负担加重 |
| COPD + 糖尿病 | 慢性炎症 / 低氧 -> 胰岛素抵抗 -> 血糖波动 | 高血糖 -> 感染风险 -> COPD 急性加重 | 活动受限 -> 运动减少 -> 控糖难度增加 |
| OSA + COPD + 糖尿病 | 双重低氧 -> 心肺负荷，同时低氧 -> 代谢波动 | 高血糖 + 慢性炎症 -> 感染和血管风险 | 数据缺失或轻度异常链路作为观察提示 |

### 13.0.2 三重共病默认展示顺序

对于 P001 这类 OSA + COPD + 糖尿病患者，默认展示顺序：

1. 低氧负担链路：气道 -> 肺部 -> 心脏。
2. 低氧代谢链路：肺部 -> 胰腺 -> 血糖波动。
3. 高血糖并发链路：胰腺 -> 血管 -> 肾脏。

若某条链路涉及的数据可信度低，则该链路降级到末位，并显示“数据不足，建议补齐后复核”。

### 13.1 OSA + 糖尿病

核心链路：

```text
睡眠呼吸暂停 -> 夜间低氧 -> 胰岛素抵抗升高 -> TIR 下降 / 平均血糖升高
高血糖 -> 气道黏膜水肿和炎症负担 -> 呼吸阻塞风险加重
低氧 + 高血糖 -> 血管内皮损伤 -> 心脑血管风险升高
```

干预优先级：

1. 优先评估夜间低氧和通气依从性。
2. 同步复核血糖波动，特别是夜间和晨起血糖。
3. 建议医生综合判断是否调整睡眠干预与降糖管理节奏。

### 13.2 OSA + COPD

核心链路：

```text
上气道阻塞 -> 夜间低氧加重
COPD 气流受限 -> 氧合能力下降
双重低氧 -> 肺动脉压力和右心负荷升高
```

干预优先级：

1. 优先识别重叠综合征低氧负担。
2. 建议医生评估通气方式、血氧监测和急性加重风险。
3. 血氧数据缺失时，不输出强结论，优先提示补齐监测。

### 13.3 COPD + 糖尿病

核心链路：

```text
慢性炎症 -> 胰岛素抵抗增加 -> 血糖更难稳定
低氧状态 -> 糖代谢波动 -> 高血糖风险升高
高血糖 -> 感染风险升高 -> COPD 急性加重风险增加
```

干预优先级：

1. 同步关注控糖和感染 / 急性加重风险。
2. 建议医生评估疫苗防护、症状变化和血糖波动。
3. 中重度时优先处理低氧、感染信号和血糖失控风险。

### 13.4 OSA + COPD + 糖尿病

核心链路：

```text
OSA 夜间低氧 + COPD 气流受限 -> 持续低氧负担
持续低氧 -> 胰岛素抵抗升高 -> 血糖波动加剧
高血糖 + 慢性炎症 -> 感染风险和心血管风险升高
心肺胰血管多器官联动 -> 综合健康分明显下降
```

干预优先级：

1. 第一优先级：低氧负担和通气 / 血氧监测完整性。
2. 第二优先级：血糖波动、TIR 和餐后 / 晨起异常。
3. 第三优先级：感染、心血管和肾脏并发症筛查。
4. 数据不完整时，建议先创建复测或随访任务。

## 十五、合规与安全文案

页面底部固定展示轻量提示：

```text
数字孪生分析基于患者授权数据生成，仅用于慢病管理辅助解释和风险提示，不作为诊断、处方或治疗设备参数调整依据。具体诊疗决策需由医生结合线下检查和临床判断完成。
```

干预建议统一使用以下表达：

1. “建议医生评估是否...”
2. “建议优先关注...”
3. “建议结合线下检查判断...”
4. “当前数据不足，建议补齐后再评估...”

避免直接使用：

1. “必须使用某治疗方式”。
2. “立即调整某药物剂量”。
3. “确诊某疾病 / 并发症”。
4. “保证降低某风险”。

## 十七、验收标准

1. 糖尿病、慢阻肺、睡眠呼吸障碍任一病种患者都能看到数字孪生 Tab。
2. 无支持病种患者展示清晰空状态，不报错。
3. P001 三重共病患者能展示多器官高亮、共病链路和综合干预优先级。
4. P002 糖尿病患者展示糖尿病单病种分析。
5. P003 慢阻肺患者展示 COPD 分析，并因设备未同步降低可信度。
6. 健康分可拆解疾病扣分、共病扣分和数据缺失扣分。
7. 演示模式切换后，指标、健康分、器官热区、链路线和建议同步变化。
8. 页面文案不出现直接诊断、处方或治疗设备参数调整结论。
9. 点击指标卡、器官热区、健康分扣分项时，左侧人体画布和右侧分析内容必须双向联动。
