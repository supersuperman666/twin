# 数字孪生融合分析页 PRD V1.0

文档版本：V1.0  
更新日期：2026-05-26  
所属产品：慢病管理系统 - 医生 PC 端  
页面位置：患者详情页一级 Tab  
覆盖病种：睡眠呼吸障碍、慢阻肺、2 型糖尿病及两两 / 三重共病  
产品定位：基于现有慢病数据，给医生提供一个“像模型在综合分析”的高阶总览页，用于快速解释患者当前风险、生成可确认的管理建议，并进入现有随访、复测、医生建议闭环。

## 一、整体叙事

数字孪生融合分析页不是再做一套完整慢病管理方案，也不是新增医学诊断模型。它的核心价值是把已有的健康分、指标、病种、预警、随访和医生建议能力包装成一个更高级的“综合分析层”。

医生进入患者详情后，页面要传达三个信息：

1. 系统已经综合看过患者数据。
2. 当前主要问题不只是单个指标异常，而是几个风险之间存在关联。
3. 医生可以基于系统建议快速安排复测、创建随访或发送医生建议。

页面叙事链路：

```text
患者已有数据
  -> 识别病种 / 共病场景
  -> 提炼核心指标
  -> 生成健康分结论
  -> 解释主要影响因素
  -> 给出孪生管理建议
  -> 展示推荐依据
  -> 进入复测 / 随访 / 医生建议闭环
```

## 二、设计目标

| 目标 | 说明 |
| --- | --- |
| 简单实现 | 尽量复用现有字段、分析页、风险分和业务入口，不重做慢病管理方案 |
| 有模型感 | 用“综合结论、影响因素、推荐依据、管理建议”营造模型分析感 |
| 可解释 | 每条结论都能追溯到核心指标、数据可信度或现有深度分析页 |
| 可闭环 | 医生可以从建议进入复测、随访、医生建议，不在本页自动下发 |
| 适合演示 | 首屏有结论、有分值、有建议、有依据，看起来不像普通指标看板 |

## 三、产品边界

### 3.1 本期做什么

1. 患者详情页新增 / 调整“数字孪生融合分析”一级 Tab。
2. 默认进入患者详情时，若从指定风险入口进入，可优先打开数字孪生融合分析 Tab。
3. 展示数字孪生健康分卡片、孪生分析依据、孪生管理建议、推荐依据、后续管理路径。
4. 支持根据患者病种和现有指标生成一段“孪生推演摘要”。
5. 支持医生基于建议进入安排复测、创建随访、发送医生建议。
6. 数据不足时展示可信度提示和补齐建议，避免输出强结论。

### 3.2 本期不做什么

1. 不新增诊断结论，不输出“确诊”。
2. 不自动生成处方、用药调整、设备参数调整。
3. 不新增一套独立慢病管理方案引擎。
4. 不新增采集设备和复杂数据模型。
5. 不强制建设 3D 人体、复杂数字人模型或链路动效。
6. 不展示内部扣分明细，避免医生质疑计算口径。

## 五、页面模块字段规则

本章是研发和评审优先阅读内容。页面先按这些模块展示，后续章节只补充字段来源和生成规则。

### 5.1 数字孪生健康分卡片

模块定位：第一屏给医生一个“系统已完成综合分析”的结论感。

| 字段 | 是否展示 | 展示内容 | 生成规则 |
| --- | --- | --- | --- |
| 模块标题 | 必须 | 数字孪生健康分 | 固定展示 |
| 健康分 | 必须 | 0-100 分 | 按“基础健康分 + 有限规则修正”生成；无可用基础分且核心指标不足时展示“暂无评分” |
| 风险等级 | 必须 | 稳定 / 需关注 / 需干预 / 高危 | 85-100 稳定；70-84 需关注；50-69 需干预；0-49 高危 |
| 孪生推演摘要 | 必须 | 一句话综合结论 | 按第 6 章规则生成 |
| 主要影响因素 | 必须 | 2-4 个标签 | 从固定共病文案、异常指标、数据可信度中提取 |
| 建议动作 | 必须 | 安排复测 / 创建随访 / 发送医生建议 | 取孪生管理建议中优先级最高的 1-2 个动作 |
| 可信度提示 | 必须 | 高 / 中 / 低 | 根据当前场景核心指标是否存在生成 |

展示限制：

1. 不展示基础分、扣分项、扣分来源、修正项、公式。
2. 不展示“器官影响修正 -8”这类内部解释。
3. 低可信度时，不展示“高危”等强判断，可展示“数据不足，待复核”。

字段组成规则：

| 字段 | 组成方式 | 兜底规则 |
| --- | --- | --- |
| 健康分 | 基础分 + 指标风险修正 + 共病修正 + 可信度修正 | 无任何基础分且核心指标不足时展示“暂无评分”，风险等级展示“待评估” |
| 风险等级 | 按健康分区间映射 | 健康分为空或可信度低时，展示“待复核” |
| 孪生推演摘要 | 主要风险来源 + 关键证据 + 优先动作 | 可信度低时使用第 6.3 节低可信度模板 |
| 主要影响因素 | 先取固定共病文案，再取异常指标影响，最后取数据可信度问题 | 少于 2 个时，可补充“数据待完善”或“建议持续观察” |
| 建议动作 | 取第 8 章生成的动作，最多展示 2 个 | 无明确异常但有数据缺失时，默认展示“安排复测” |
| 可信度提示 | 取第 7.4 节可信度等级和缺失项 | 无数据时展示“低，核心数据不足” |

健康分计算逻辑：

1.0 使用“基础健康分 + 有限规则修正”的方式生成数字孪生健康分。  
该规则比直接复用现有分数更有数字孪生感，但不新增复杂疾病模型，也不回写患者全局健康风险分。

```text
基础健康分 = patient.riskScore || patient.screening.score || 100

数字孪生健康分 = clamp(
  基础健康分
  - 指标风险修正
  - 共病叠加修正
  - 数据可信度修正
  + 改善加分,
  0,
  100
)
```

修正项规则：

| 修正项 | 触发条件 | 取值 |
| --- | --- | ---: |
| 指标风险修正 | 当前场景核心指标异常 | 单项轻微异常 5；单项明确异常 10；多项异常 15 |
| 共病叠加修正 | 共病场景同时命中两个病种的异常指标，并生成固定共病文案 | 两病共病 5；三重共病 10 |
| 数据可信度修正 | 当前场景核心数据缺失 | 可信度中 5；可信度低 10 |
| 改善加分 | 最近 7 / 14 天关键指标较前次好转，或医生任务持续执行 | 5 |

计算规则：

1. 基础健康分优先取 `patient.riskScore`，其次取 `patient.screening.score`，两者都没有时按 100 作为临时基础分。
2. 指标风险修正只取最高档，不重复叠加。例如同时存在 2 项异常时取 15，不再叠加单项异常。
3. 共病叠加修正只在命中固定共病文案时生效，不做复杂共病推理。
4. 单项轻微异常用于已有深度分析页返回“轻度异常 / 边界异常 / 轻微偏离”等状态时；若只有正常 / 异常两档，则异常按 10 分计算。
5. 数据可信度修正只按高 / 中 / 低三档计算，可信度高时为 0。
6. 改善加分最多 5 分，仅在现有数据能判断趋势改善或任务执行时使用；无法判断时默认为 0。
7. 若无现成健康分且核心指标也不足，健康分展示“暂无评分”，不使用 100 强行计算。
8. 前台只展示最终分和风险等级，不展示公式、修正项和具体分值。
9. 数据可信度低时，风险等级展示“待复核”，摘要使用低可信度模板。

示例：

```text
患者 patient.riskScore = 78
命中 OSA + 糖尿病场景
最低血氧异常，TIR 异常
核心数据齐全，无近期改善

基础健康分 = 78
指标风险修正 = 15
共病叠加修正 = 5
数据可信度修正 = 0
改善加分 = 0

数字孪生健康分 = 78 - 15 - 5 - 0 + 0 = 58
风险等级 = 需干预
```

### 5.2 孪生分析依据

模块定位：让医生相信健康分摘要不是写死文案，而是由数据推出来的。

| 字段 | 是否展示 | 展示内容 | 生成规则 |
| --- | --- | --- | --- |
| 核心指标 | 必须 | 当前最关键的 3-4 个指标 | 根据患者病种 / 共病场景选择 |
| 指标状态 | 必须 | 正常 / 异常 / 数据不足 | 复用现有深度分析页状态或阈值 |
| 数据可信度 | 必须 | 高 / 中 / 低、最近更新、数据来源、缺失提示 | 根据本场景核心数据是否存在生成 |
| 影响解释 | 必须 | 例如”夜间低氧负担””血糖波动””症状负担” | 由异常指标映射生成 |
| 关联链路 | 共病时展示 | 详见 §5.2.1 | 详见 §5.2.1 |

展示规则：

1. 默认展示 3-4 个核心指标，不硬凑不存在的数据。
2. 指标卡底部提供”查看睡眠深度分析 / 血糖深度分析 / 慢阻肺深度分析”入口。
3. 数据不足时，用”数据不足”卡替代强结论。

核心指标选择规则：

1. 先按第 7.2 节当前病种 / 共病场景确定候选指标。
2. 候选指标中优先展示异常指标，其次展示数据缺失指标，最后展示正常指标。
3. 最多展示 4 个；若有效指标不足 3 个，用”数据不足”卡补位。
4. 同一类指标只保留最能解释问题的一项，例如血糖同时有 TIR 和平均血糖异常时，优先展示 TIR。
5. 每个指标必须展示数据来源或跳转入口，避免医生无法追溯。

影响解释生成规则：

| 指标类型 | 命中条件 | 影响解释 |
| --- | --- | --- |
| 睡眠 / 低氧 | AHI、ODI、最低血氧、Ts90、低氧时长异常 | 夜间低氧负担、睡眠呼吸事件负担 |
| 慢阻肺症状 | CAT、平均血氧、最低血氧异常 | 症状负担、氧合不足 |
| 血糖 | TIR、平均血糖、CV、TAR/TBR、eA1c、指尖血糖异常 | 血糖波动、血糖达标不足 |
| 数据缺失 | 核心指标缺失或数据过期 | 数据完整性不足，建议复核 |

### 5.2.1 共病风险传导链路

**模块定位**：数字孪生页的”模型感”核心来源。普通慢病管理系统只展示单个指标是否达标，数字孪生页要做到把多个病种的异常指标”串联”起来，让医生看到指标之间的传导关系，而不只是孤立的数字。展示效果目标：医生看到这里应该有”系统真的在帮我分析，不是堆数据”的感觉。

#### 展示位置

紧接在核心指标卡片下方，推荐依据之前，作为 §5.2 孪生分析依据模块的最后一个子区域独立展示，标题为”风险传导路径”。

仅患者命中 2 个及以上支持病种（OSA、COPD、2 型糖尿病任意组合）时展示本模块。单病种患者不展示，不留空位。

#### 整体结构

```text
风险传导路径
├── 场景标签（如”OSA + 糖尿病 · 缺氧代谢共病”）
├── 系统识别句（如”系统识别到 2 条风险传导路径”）
└── 链路卡片列表
    ├── 链路卡片 1（高优先级）
    └── 链路卡片 2（次级，共病组合才有）
```

#### 链路卡片字段定义

每张链路卡片由以下字段组成：

| 字段 | 是否展示 | 内容 | 生成规则 |
| --- | --- | --- | --- |
| 链路标题 | 必须 | 如”低氧 → 代谢影响” | 取该链路的起点影响域和终点影响域 |
| 严重等级标签 | 必须 | 严重 / 中度 / 观察 | 由链路内异常指标数量决定，见下方规则 |
| 激活状态 | 必须 | 激活 / 数据待补 / 正常 | 由链路触发指标是否有效且异常决定 |
| 起点证据节点 | 必须 | 病种 A 的 1-2 个核心指标值 + 状态标记 | 从 metricPool 取该链路起点指标的实际值 |
| 终点证据节点 | 必须 | 病种 B 的 1-2 个核心指标值 + 状态标记 | 从 metricPool 取该链路终点指标的实际值 |
| 传导叙事 | 必须 | 1-2 句解释两侧指标之间机制关系的文案 | 按病种组合 + 双侧指标状态从固定模板生成，见 §5.2.2 |
| 建议动作入口 | 条件展示 | 进入复测 / 随访 / 预警 | 链路为激活状态时展示；优先复用 §5.3 中已生成的建议 |

#### 链路卡片激活状态规则

| 状态 | 判定条件 | 视觉表现 |
| --- | --- | --- |
| 激活 | 起点指标和终点指标均有数据，且至少一侧异常 | 卡片高亮，左侧彩色竖条，叙事文字正常展示 |
| 数据待补 | 起点或终点至少一侧指标缺失 | 卡片虚线边框，叙事文字用缺失模板替代，展示”安排复测”入口 |
| 正常 | 起点和终点指标均有数据且均正常 | 卡片灰色弱展示，叙事改为”当前未见明显传导风险，建议持续观察”，不展示建议入口 |

正常状态的链路卡片默认折叠或弱化展示，不抢占视觉焦点。

#### 严重等级判定规则

| 等级 | 判定条件 |
| --- | --- |
| 严重 | 链路两侧均有异常指标，且至少一侧为中度及以上异常（偏离阈值 ≥ 20%） |
| 中度 | 链路两侧均有异常指标，但程度均为轻度偏离 |
| 观察 | 链路只有一侧异常，或数据不完整 |

1.0 不做精细的偏离百分比计算，按以下简化口径代替：
- `tone === “severe”` 的指标 → 视为中度及以上异常
- `tone === “medium”` 的指标 → 视为轻度偏离
- `tone === “light”` → 观察

#### 每个共病场景的链路定义

共病场景共 4 种，每种预设固定链路，最多展示 2 条。

**OSA + 糖尿病（缺氧代谢共病）**

| 链路 | 起点指标 | 终点指标 | 优先级 |
| --- | --- | --- | --- |
| 低氧 → 代谢影响 | AHI、最低血氧 | TIR、平均血糖 | 主链路（优先展示） |
| 代谢异常 → 血管风险 | 平均血糖、eA1c | 血糖达标率（reach）| 次级链路 |

**OSA + COPD（重叠综合征）**

| 链路 | 起点指标 | 终点指标 | 优先级 |
| --- | --- | --- | --- |
| 上气道阻塞 → 肺部低氧 | AHI | 最低血氧、Ts90 | 主链路 |
| 双重低氧 → 心肺负荷 | 最低血氧、ODI | CAT、平均血氧 | 次级链路 |

**COPD + 糖尿病（炎症代谢共病）**

| 链路 | 起点指标 | 终点指标 | 优先级 |
| --- | --- | --- | --- |
| 氧合不足 → 代谢影响 | CAT、平均血氧 | TIR、平均血糖 | 主链路 |
| 代谢异常 → 血管风险 | 平均血糖、eA1c | 血糖达标率 | 次级链路 |

**三重共病（OSA + COPD + 糖尿病）**

| 链路 | 起点指标 | 终点指标 | 优先级 |
| --- | --- | --- | --- |
| 双重低氧 → 心肺负荷 | AHI、最低血氧 | CAT、Ts90 | 主链路 |
| 低氧 → 代谢叠加 | 最低血氧、ODI | TIR、平均血糖 | 次级链路 |

三重共病只展示前 2 条主要链路，第三条（代谢 → 血管）在推荐依据中以文字体现，不单独展示链路卡片。

### 5.2.2 传导叙事生成规则

传导叙事不是写死文案，而是由**规则选模板 + 动态插槽**拼接生成。每条叙事分三段：

```text
{起点描述}，{机制短语}，{终点描述}。
```

- **起点描述 / 终点描述**：从 `metricPool` 取实际值动态生成，格式为 `{指标名} {实际值}{单位}，{状态词}`
- **机制短语**：固定医学描述，与具体数值无关，按链路 ID 写死
- **状态词**：由指标 `tone` 值映射，`severe/medium` → 偏高 / 偏低（视指标方向），`light` → 略偏高 / 略偏低，`normal` → 正常

#### 状态词映射规则

| tone | 高 bad 指标（AHI、CAT、平均血糖等） | 低 bad 指标（TIR、最低血氧、平均血氧等） |
| --- | --- | --- |
| severe | 明显偏高 | 明显偏低 |
| medium | 偏高 | 偏低 |
| light | 略偏高 | 略偏低 |
| normal | 正常 | 正常 |
| missing | （不生成起点/终点描述，进入缺失分支） | 同左 |

#### 机制短语定义（固定，按链路 ID）

| 链路 ID | 机制短语 |
| --- | --- |
| 低氧→代谢 | 低氧状态可抑制胰岛素分泌并加剧葡萄糖代谢紊乱，两者可能相互强化 |
| 代谢→血管 | 长期高血糖环境可加剧血管内皮损伤风险 |
| 上气道→肺部 | 上气道阻塞与气流受限并存时可加剧夜间低氧程度 |
| 双重低氧→心肺 | 睡眠低氧与慢阻肺氧合不足同时存在时对心肺系统可能产生叠加影响 |
| 氧合→代谢 | 长期氧合不足可加剧葡萄糖代谢紊乱 |

#### 四种状态下的拼接规则

**双侧均有数据（无论异常与否）**

```text
{起点指标名} {起点值}{单位}，{状态词}，{机制短语}，{终点指标名} {终点值}{单位}，{状态词}。
```

示例（低氧→代谢，双侧均异常）：
> AHI 22.3 次/小时，偏高，低氧状态可抑制胰岛素分泌并加剧葡萄糖代谢紊乱，两者可能相互强化，TIR 58%，偏低。

示例（低氧→代谢，双侧均正常）：
> AHI 8.2 次/小时，正常，{机制短语}，TIR 74%，正常。当前未见明显传导风险，建议持续观察。

**仅起点有数据，终点缺失**

```text
{起点指标名} {起点值}{单位}，{状态词}，{机制短语}，{终点指标名}数据暂缺，建议补充后进一步判断。
```

**仅终点有数据，起点缺失**

```text
{起点指标名}数据暂缺，{终点指标名} {终点值}{单位}，{状态词}，建议补充{起点数据来源}后判断传导关系。
```

**双侧均缺失（兜底，唯一写死的文案）**

各链路固定兜底文案（此处允许写死，因为无任何数据可插入）：

| 链路 ID | 兜底文案 |
| --- | --- |
| 低氧→代谢 | 低氧与代谢相关数据不足，建议优先补齐血氧和血糖监测数据。 |
| 代谢→血管 | 代谢与血管相关数据不足，建议补充血糖和代谢指标后再复核。 |
| 上气道→肺部 | 上气道与夜间低氧相关数据不足，建议安排睡眠监测和血氧记录。 |
| 双重低氧→心肺 | 双重低氧相关数据不足，建议补充睡眠监测和 CAT 评估。 |
| 氧合→代谢 | 氧合与代谢相关数据不足，建议同步补充血氧和血糖数据。 |

#### 起点/终点指标选取规则

每条链路的起点和终点各取 1 个最能代表该侧的指标，优先取该侧 `tone` 最严重的指标：

| 链路 ID | 起点候选指标 | 取值优先级 | 终点候选指标 | 取值优先级 |
| --- | --- | --- | --- | --- |
| 低氧→代谢 | AHI、最低血氧 | AHI 优先；AHI 缺失时取最低血氧 | TIR、平均血糖 | TIR 优先；TIR 缺失时取平均血糖 |
| 代谢→血管 | 平均血糖、eA1c | 平均血糖优先 | 血糖达标率、eA1c | 血糖达标率优先 |
| 上气道→肺部 | AHI | AHI | 最低血氧、Ts90 | 最低血氧优先 |
| 双重低氧→心肺 | 最低血氧、ODI | 最低血氧优先 | CAT、平均血氧 | CAT 优先 |
| 氧合→代谢 | CAT、平均血氧 | CAT 优先 | TIR、平均血糖 | TIR 优先 |

#### 文案生成限制

1. 不输出”导致””确诊”等强因果表达，机制短语统一使用”可能””存在影响”等弱推断表达。
2. 完整叙事不超过 2 句，控制在 60 字以内。
3. 双侧缺失时必须使用兜底文案，不得用带数值的模板替代。
4. 可信度为低时，所有链路叙事退化为对应兜底文案，不插入数值。

### 5.2.3 链路模块整体兜底与降级规则

| 场景 | 处理方式 |
| --- | --- |
| 单病种患者 | 不展示本模块，不留空位 |
| 共病患者但所有链路均正常 | 展示模块标题和场景标签，链路卡片使用”正常”状态灰色弱展示 |
| 共病患者但所有核心指标均缺失 | 展示模块，所有链路进入”数据待补”状态，优先展示复测入口 |
| 可信度为低 | 叙事文案统一使用兜底模板，不输出任何强结论 |

### 5.3 孪生管理建议

模块定位：给医生一个“模型不仅分析，还能产出下一步管理建议”的感觉。

| 字段 | 是否展示 | 展示内容 | 生成规则 |
| --- | --- | --- | --- |
| 模块标题 | 必须 | 孪生管理建议 | 固定展示 |
| 安全提示 | 必须 | 需医生确认后生效，不自动下发 | 固定展示 |
| 复测建议 | 条件展示 | 建议复测项目、原因、建议时机 | 数据缺失、可信度中 / 低、关键指标异常时生成 |
| 随访建议 | 条件展示 | 建议随访时间、随访重点 | 风险等级为需干预 / 高危，或多项指标异常时生成 |
| 医生建议 | 条件展示 | 可发送给患者的通俗建议草稿 | 按主要影响因素套用安全模板，医生可编辑 |

建议优先级：

| 条件 | 优先建议 |
| --- | --- |
| 核心数据缺失或可信度低 | 安排复测 / 补齐数据 |
| 单项核心指标异常 | 发送医生建议 + 安排复测 |
| 多项指标异常或命中固定共病文案 | 创建随访 + 安排复测 + 发送医生建议 |
| 已存在待处理预警 | 创建随访 + 查看相关预警 |

建议内容组成规则：

| 建议类型 | 字段组成 | 生成规则 |
| --- | --- | --- |
| 复测建议 | 复测项目、复测原因、建议时机 | 复测项目取缺失或异常的核心指标；原因取对应主要风险来源；时机默认“近期”，高危或预警场景展示“尽快” |
| 随访建议 | 随访时间、随访重点、核实内容 | 随访时间按风险等级生成；重点取主要影响因素；核实内容包括症状变化、设备使用、数据记录、是否已处理预警 |
| 医生建议 | 患者可理解文案、注意事项、需医生确认提示 | 按主要影响因素匹配 2-3 个安全模板，不做个性化长文生成 |

随访时间建议：

| 条件 | 建议时间 |
| --- | --- |
| 高危或存在待处理预警 | 1-3 天内随访 |
| 需干预，且多项核心指标异常 | 3-7 天内随访 |
| 需关注，且只有单项异常 | 7-14 天内随访 |
| 主要问题是数据缺失 | 复测完成后随访 |

### 5.4 推荐依据

模块定位：解释“为什么给这些建议”，增强模型可信度。

| 字段 | 是否展示 | 展示内容 | 生成规则 |
| --- | --- | --- | --- |
| 依据标题 | 必须 | 例如“低氧负担是当前优先关注方向” | 根据最高优先级影响因素生成 |
| 证据指标 | 必须 | 1-3 个指标值或状态 | 取触发建议的核心指标 |
| 关联解释 | 必须 | 说明指标与建议之间的关系 | 使用固定模板拼接 |
| 对应建议 | 必须 | 指向复测 / 随访 / 医生建议 | 与孪生管理建议中的动作对应 |

默认展示 2-3 条依据：

1. 第一条解释最高优先级管理建议。
2. 第二条解释主要风险来源。
3. 第三条解释数据可信度或缺失项。

推荐依据模板：

| 依据类型 | 生成模板 |
| --- | --- |
| 指标异常依据 | `[指标名称] 当前为 [指标值 / 状态]，提示 [影响解释]，因此建议 [对应建议]。` |
| 固定共病依据 | `[病种组合] 同时存在 [风险来源 A] 和 [风险来源 B]，提示 [固定共病文案]，因此建议 [对应建议]。` |
| 数据可信度依据 | `[缺失数据] 当前缺失 / 更新不及时，会影响 [判断方向] 判断，因此建议 [对应建议]。` |
| 预警依据 | `当前存在 [预警类型] 待处理，建议医生先查看预警详情，并结合随访确认患者状态。` |

生成限制：

1. 每条依据最多引用 3 个指标，避免变成报告堆砌。
2. 依据必须能对应到一个建议动作。
3. 若找不到证据来源，不展示该条依据。

### 5.5 后续管理路径

模块定位：把“看起来高级的分析”落到现有业务闭环。

| 入口 | 展示条件 | 点击后 |
| --- | --- | --- |
| 安排复测 | 有复测建议或核心数据缺失 | 进入现有复测 / 任务创建流程 |
| 创建随访 | 有随访建议或风险需干预 / 高危 | 进入现有随访创建流程 |
| 发送医生建议 | 有医生建议草稿 | 进入现有医生建议编辑 / 发送流程 |
| 查看相关预警 | 当前患者有待处理预警 | 进入现有预警模块 |
| 查看深度分析 | 有对应病种数据 | 跳转血糖 / 睡眠 / 慢阻肺深度分析 |

限制：

1. 数字孪生页不直接完成下发或处置。
2. 所有动作必须进入现有业务模块，由医生确认后生效。

入口展示规则：

| 场景 | 展示方式 |
| --- | --- |
| 可创建 | 展示主按钮，例如“创建随访”“安排复测” |
| 已存在记录 | 若现有模块能提供状态，则展示“查看随访 / 查看复测任务 / 查看医生建议”；若无法提供状态，1.0 不强制回显 |
| 当前条件不满足 | 不展示入口，或展示为次级入口 |

## 六、孪生推演摘要生成规则

孪生推演摘要不是固定写死文案，而是由“主要风险来源、关键证据、优先动作”拼接生成。

### 6.1 生成字段

| 字段 | 来源 | 优先级 |
| --- | --- | --- |
| 主要风险来源 | 主要影响因素中优先级最高的标签 | 固定共病文案 > 指标异常 > 数据不足 |
| 关键证据 | 核心指标中的异常项或缺失项 | 最多取 2 个，优先取异常程度更高的指标 |
| 影响方向 | 由主要风险来源映射 | 低氧负担、血糖波动、症状负担、心肺负荷、数据完整性不足 |
| 优先动作 | 孪生管理建议中的最高优先级动作 | 复测 > 随访 > 医生建议；若已有预警则随访优先 |

### 6.2 默认模板

```text
当前主要风险来自[主要风险来源]。[关键证据]是本次判断的主要依据。系统建议优先[优先动作]。
```

示例：

```text
当前主要风险来自夜间低氧负担与血糖波动叠加。最低血氧和 TIR 是本次判断的主要依据。系统建议优先创建随访并安排复测。
```

### 6.3 低可信度模板

当数据可信度为低时，不输出强风险结论，改用低可信度模板。

```text
当前核心数据不足，系统仅基于已有数据识别到[已有风险来源]。[缺失数据]缺失会影响判断准确性，建议优先[补齐动作]后再复核。
```

示例：

```text
当前核心数据不足，系统仅基于已有数据识别到症状负担升高。近 7 天血氧数据缺失会影响判断准确性，建议优先安排复测后再复核。
```

### 6.4 主要风险来源映射

| 命中条件 | 主要风险来源 | 影响方向 |
| --- | --- | --- |
| AHI、ODI、最低血氧、Ts90、低氧时长任一异常 | 夜间低氧负担 / 睡眠呼吸事件负担 | 低氧负担、心肺负荷 |
| CAT、平均血氧、最低血氧异常 | 症状负担 / 氧合不足 | 呼吸症状、心肺负荷 |
| TIR、平均血糖、CV、TAR/TBR、eA1c、指尖血糖异常 | 血糖波动 / 血糖达标不足 | 代谢负担、血管风险 |
| OSA + 糖尿病同时命中低氧和血糖异常 | 低氧负担与血糖波动叠加 | 固定文案：低氧代谢叠加 |
| OSA + COPD 同时命中低氧和症状异常 | 双重低氧与心肺负荷叠加 | 固定文案：心肺负荷叠加 |
| COPD + 糖尿病同时命中氧合和血糖异常 | 氧合不足与代谢负担叠加 | 固定文案：呼吸代谢叠加 |
| 核心数据缺失或可信度低 | 数据完整性不足 | 待复核 |

## 七、数据来源与核心指标

### 7.1 可用数据源

| 数据类型 | 来源 | 用途 |
| --- | --- | --- |
| 健康风险分 | `patient.riskScore`、`patient.screening.score` | 健康分展示和风险等级 |
| 睡眠报告 | 现有睡眠深度分析数据 | AHI、ODI、最低血氧、睡眠呼吸事件 |
| 血氧报告 | 现有慢阻肺 / 血氧数据 | 平均血氧、最低血氧、Ts90、低氧时长 |
| CAT 问卷 | 现有慢阻肺问卷数据 | 症状负担 |
| 血糖数据 | 现有血糖深度分析数据 | TIR、平均血糖、CV、TAR/TBR、eA1c、指尖血糖 |
| 预警 / 随访 / 医生建议 | 现有业务模块 | 后续管理路径和入口展示 |

### 7.2 病种与默认核心指标

| 场景 | 默认核心指标 | 默认关注方向 |
| --- | --- | --- |
| 睡眠呼吸障碍 / OSA | AHI、最低血氧、ODI、睡眠报告有效性 | 睡眠呼吸事件、夜间低氧 |
| 慢阻肺 / COPD | CAT、平均血氧、最低血氧、Ts90 / 低氧时长 | 症状负担、氧合不足 |
| 2 型糖尿病 | TIR、平均血糖、CV / TAR/TBR、eA1c 或指尖血糖 | 血糖达标、血糖波动 |
| OSA + 糖尿病 | 最低血氧、AHI、TIR、平均血糖 | 低氧负担与血糖波动叠加 |
| OSA + COPD | 最低血氧、AHI、CAT、Ts90 / 低氧时长 | 双重低氧与心肺负荷 |
| COPD + 糖尿病 | CAT、平均血氧、TIR、平均血糖 | 氧合不足与代谢负担 |
| 三重共病 | 最低血氧、AHI、CAT、TIR | 低氧、症状负担、血糖波动 |

### 7.3 指标异常判定

本期不新增复杂阈值模型，优先复用现有深度分析页的状态标签。若当前数据无状态标签，则按以下轻量口径兜底。

| 指标 | 正常 / 达标 | 异常口径 |
| --- | --- | --- |
| AHI | < 5 次/小时 | >= 5 次/小时 |
| ODI | < 5 次/小时 | >= 5 次/小时 |
| 最低血氧 | >= 90% | < 90% |
| Ts90 | < 5% | >= 5% |
| 低氧时长 | 0 分钟 | > 0 分钟 |
| CAT | <= 10 分 | > 10 分 |
| 平均血氧 | >= 95% | < 95% |
| TIR | >= 70% | < 70% |
| 平均血糖 | < 7.8 mmol/L | >= 7.8 mmol/L |
| CV | < 36% | >= 36% |
| eA1c | < 7.0% | >= 7.0% |
| 空腹血糖均值 | 4.4-7.0 mmol/L | 超出范围 |
| 餐后2h血糖均值 | < 10.0 mmol/L | >= 10.0 mmol/L |

说明：这些阈值仅用于数字孪生页展示和文案触发，不作为诊断依据。

### 7.4 数据可信度

1.0 不做精细完整率计算，只按当前场景核心指标是否存在判断高 / 中 / 低。

| 可信度 | 判定 | 页面表现 |
| --- | --- | --- |
| 高 | 当前场景核心指标均存在，且至少有 1 条近期有效数据 | 可展示健康分等级、摘要、建议和依据 |
| 中 | 当前场景缺少 1 类核心指标，或核心数据更新不及时 | 展示结论，但增加“建议复核 / 补齐数据”提示 |
| 低 | 当前场景缺少 2 类及以上核心指标，或关键指标全部缺失 | 不输出强结论，优先展示补齐动作 |

关键指标定义：

| 场景 | 关键指标 |
| --- | --- |
| OSA | AHI、最低血氧 |
| COPD | CAT、平均血氧 / 最低血氧 |
| 2 型糖尿病 | TIR 或指尖血糖达标率、平均血糖 |
| 共病场景 | 当前共病涉及病种的关键指标并集 |

低可信度时页面只展示：

1. 已有数据事实。
2. 缺失数据项。
3. 数据可信度提示。
4. 安排复测 / 补齐数据 / 创建随访入口。
5. 对应深度分析入口。

## 八、孪生管理建议生成规则

孪生管理建议只生成“医生可确认动作”，不自动下发。

### 8.1 复测建议

生成条件：

1. 核心指标缺失。
2. 数据可信度为中 / 低。
3. 关键指标异常但最近数据不足。

文案模板：

```text
建议安排[复测项目]，用于复核[风险来源]。当前判断受[缺失或异常数据]影响，复测后可进一步确认管理方向。
```

### 8.2 随访建议

生成条件：

1. 风险等级为需干预 / 高危。
2. 多项核心指标异常。
3. 命中共病叠加风险。
4. 当前存在待处理预警。

文案模板：

```text
建议创建随访，重点核实[随访重点]，并结合患者近期症状、设备使用和数据记录情况判断下一步管理安排。
```

### 8.3 医生建议

生成条件：

1. 存在明确主要影响因素。
2. 数据可信度不为低。
3. 医生可基于通俗建议向患者解释当前关注点。

1.0 固定使用以下安全模板，不做复杂个性化生成：

| 主要影响因素 | 医生建议草稿 |
| --- | --- |
| 夜间低氧负担 / 睡眠呼吸事件负担 | 近期建议重点关注夜间睡眠和血氧变化，保持设备监测和数据记录，如出现明显憋醒、白天困倦或不适，请及时联系医生复核。 |
| 血糖波动 / 血糖达标不足 | 近期建议继续规律记录血糖，关注空腹、餐后和夜间血糖变化，若指标持续异常或出现不适，请及时联系医生复核。 |
| 症状负担 / 氧合不足 | 近期建议关注呼吸症状和活动耐受变化，保持血氧或症状记录，如出现气促加重、咳喘明显或不适，请及时联系医生复核。 |
| 数据完整性不足 | 当前关键数据不足，建议先补齐监测或复测数据，医生将结合后续数据进一步判断管理方向。 |

兜底模板：

```text
近期需要重点关注[主要影响因素]。建议继续记录相关数据，如出现不适或指标持续异常，请及时联系医生复核。
```

## 九、推荐依据生成规则

推荐依据用于解释每条建议为什么出现。

| 建议类型 | 依据来源 | 示例 |
| --- | --- | --- |
| 复测建议 | 数据缺失、数据过期、核心指标异常 | “近 7 天血氧数据缺失，当前低氧负担判断可信度不足” |
| 随访建议 | 风险等级、固定共病文案、多项异常 | “最低血氧异常且 TIR 未达标，提示低氧与血糖波动可能叠加” |
| 医生建议 | 主要影响因素、异常指标 | “TIR 低于目标范围，提示近期血糖达标不足” |
| 查看预警 | 现有预警记录 | “当前存在待处理预警，建议医生先查看预警详情” |

每条推荐依据必须满足：

1. 能对应到一个建议动作。
2. 至少引用一个指标、缺失项、风险等级或预警状态。
3. 不输出诊断和治疗指令。

## 十、页面状态

### 10.1 正常状态

患者命中支持病种，且核心数据基本齐全。

页面展示：

1. 健康分和风险等级。
2. 孪生推演摘要。
3. 核心指标和主要影响因素。
4. 孪生管理建议。
5. 推荐依据和后续管理路径。

### 10.2 数据不足状态

患者命中支持病种，但关键数据不足。

页面展示：

1. 健康分可展示为“待复核”或弱化展示。
2. 摘要使用低可信度模板。
3. 核心指标缺失卡。
4. 数据可信度提示。
5. 优先展示安排复测、补齐数据、创建随访。

### 10.3 空状态

患者未命中支持病种，或没有任何可用数据。

文案：

```text
暂无可用于数字孪生融合分析的慢病数据。
```

可展示入口：

1. 健康档案。
2. 健康筛查。
3. 创建随访。

## 十一、演示效果要求

为了达到”简单实现但看起来像深度模型在分析”的效果，页面需要重点做好以下体验：

1. 健康分卡片必须放在第一屏，摘要文案要像系统综合判断，不像普通提示。
2. 核心指标不要堆很多，只展示 3-4 个最关键指标。
3. 每条建议后面必须有推荐依据，形成”建议不是拍脑袋”的感觉。
4. 建议动作必须能进入现有业务闭环，让页面看起来不仅能看，还能管。
5. 数据不足时不要空白，要展示”模型需要补齐数据后复核”的专业感。
6. 共病患者必须展示风险传导路径模块，每条链路嵌入实际指标值，让医生看到”系统在顺着数据推理”而不是展示静态说明文字。

推荐视觉表达：

1. 健康分可使用大号数字 + 风险等级标签。
2. 主要影响因素使用标签组展示。
3. 孪生分析依据可使用”指标卡 + 证据说明”。
4. 孪生管理建议可使用任务卡样式，突出”待医生确认”。
5. 推荐依据可用折叠列表或轻量说明卡。
6. 风险传导路径可使用”两栏证据节点 + 中间箭头 + 底部叙事文字”结构，让链路看起来像推理过程而不是图例说明。链路卡片左侧竖条颜色对应严重等级（严重红、中度橙、观察灰）。

## 十二、可选数字人表现层

数字人、器官高亮、人体联动不作为 1.0 必做内容。若演示需要，可作为轻量表现层放在页面左侧或顶部背景区域。

### 12.1 可选展示

| 能力 | 说明 | 是否必做 |
| --- | --- | --- |
| 2D 人体底图 | 用于增强数字孪生视觉心智 | 可选 |
| 器官高亮 | 根据主要影响因素高亮肺、心脏、血管、胰腺等区域 | 可选 |
| 链路线 | 共病场景展示低氧、血糖、心肺负荷等关联 | 可选 |
| 3D 人体 | 后续演示增强，不进入 1.0 必做范围 | 暂缓 |

### 12.2 表现层原则

1. 数字人只做展示，不参与健康分和建议生成。
2. 数字人失败或不做，不影响主流程。
3. 不强制使用 Three.js、GLB 模型或复杂 3D 资产。
4. 数据可信度低时，不展示强链路动效，只提示数据不足。

## 十三、合规与安全文案

页面底部固定展示轻量提示：

```text
数字孪生分析基于患者授权数据生成，仅用于慢病管理辅助解释和风险提示，不作为诊断、处方或治疗设备参数调整依据。具体诊疗决策需由医生结合线下检查和临床判断完成。
```

禁止文案：

1. “确诊某疾病 / 并发症”。
2. “必须使用某治疗方式”。
3. “立即调整某药物剂量”。
4. “保证降低某风险”。

推荐文案：

1. “建议医生评估是否...”。
2. “建议优先关注...”。
3. “建议结合线下检查判断...”。
4. “当前数据不足，建议补齐后再评估...”。

## 十四、模块规则完整性自检

| 模块 | 字段规则 | 生成规则 | 数据来源 | 兜底 / 降级 | 交互闭环 |
| --- | --- | --- | --- | --- | --- |
| 数字孪生健康分卡片 | 已定义 | 已定义 | 健康风险分、核心指标、可信度 | 暂无评分、待复核、低可信度模板 | 建议动作入口 |
| 孪生分析依据 | 已定义 | 已定义 | 深度分析页、核心指标、数据可信度 | 数据不足卡、深度分析入口 | 跳转深度分析 |
| 风险传导路径（共病） | 已定义（§5.2.1） | 已定义（§5.2.2） | 各病种核心指标、metricPool | 数据待补状态、兜底模板（§5.2.3） | 复测 / 随访入口 |
| 孪生管理建议 | 已定义 | 已定义 | 主要影响因素、可信度、预警状态 | 无强结论时优先复测 | 复测、随访、医生建议 |
| 推荐依据 | 已定义 | 已定义 | 指标异常、固定共病文案、缺失项、预警 | 无证据不展示 | 对应建议动作 |
| 后续管理路径 | 已定义 | 已定义 | 复测、随访、医生建议、预警、深度分析 | 条件不满足不展示或弱展示 | 进入现有业务模块 |

结论：1.0 版本每个页面模块均需同时具备字段定义、生成规则、数据来源、兜底规则和闭环入口；若研发实现时缺少任一项，该模块不得展示强结论。

## 十五、验收标准

1. 支持病种患者能看到“数字孪生融合分析”Tab。
2. Tab 顺序符合 PRD 要求，原“总览”不再作为一级 Tab 展示。
3. 顶部患者卡片删除“优先处理”字段。
4. 健康分卡片展示分值、风险等级、孪生推演摘要、主要影响因素、建议动作和可信度提示。
5. 健康分卡片不展示扣分明细、公式、修正项。
6. 孪生分析依据展示 3-4 个核心指标，并能跳转对应深度分析页。
7. 页面能生成复测建议、随访建议、医生建议中的至少一种。
8. 所有建议均展示“需医生确认后生效，不自动下发”。
9. 推荐依据能解释建议来源，且至少引用一个指标、缺失项、风险等级或预警状态。
10. 数据可信度低时，不输出强结论，优先展示补齐数据和复测动作。
11. 点击安排复测、创建随访、发送医生建议、查看预警时，进入现有业务闭环。
12. 页面文案不出现诊断、处方、治疗设备参数调整等越界表达。
