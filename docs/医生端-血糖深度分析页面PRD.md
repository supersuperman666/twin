# 医生端\-血糖深度分析页面PRD

## 一、页面整体概述

页面名称：血糖深度分析（医生端\-患者详情页子Tab）

页面定位：本页面为**医生端患者详情页下的一级子Tab页面**，隶属于患者全量健康数据详情体系。血糖深度分析页定位为血糖专项数据证据页，内部拆分 **CGM动态血糖、指尖血糖** 两个二级子Tab，分别承载连续监测血糖和离散自测血糖的数据复盘、趋势查看、明细追溯和症状记录。综合判断、共病融合、管理建议统一进入「数字孪生融合分析」Tab 承接。

核心能力：指标标准化量化、异常状态识别、趋势图表查看、明细记录追溯、症状辅助查看、数字孪生分析入口。

## 二、页面整体结构布局

患者详情 → 血糖深度分析（一级Tab）

├─ 数字孪生入口横幅（条件展示，命中支持病种时展示）

├─ 二级子Tab切换栏：CGM动态血糖 / 指尖血糖
│  └─ CGM专属溯源信息（Tab右侧悬浮展示）：数据最新更新时间、监测设备品牌

├─ 核心指标卡片区（CGM/指尖 双子Tab独立展示）

├─ 趋势图表区（**仅CGM子Tab展示**）


├─ 血糖明细记录区（双子Tab数据隔离展示）

├─ 患者近期糖尿病症状展示区（双Tab常驻、仅查看）




本Tab页面整体自上而下分为6类核心模块，同时页面顶部内置二级子Tab切换栏，层级固定、逻辑递进，先切换血糖数据类型，再查看对应数据证据：

模块1：数字孪生融合分析入口横幅（条件展示）

模块2：CGM动态血糖核心指标卡片区

模块3：指尖血糖核心指标卡片区

模块4：血糖趋势曲线图表分析区（CGM专属）

模块5：血糖明细记录区

模块6：患者近期糖尿病症状展示区



**页面二级子Tab规则（核心新增）**：血糖深度分析Tab内部分两个互斥二级子Tab，默认选中【CGM动态血糖分析】

### 4\.3 系统目标库规划（全局规则）

**1、本期V1\.0落地：预置标准化目标**

系统内置《中国2型糖尿病防治指南》标准化血糖控制目标，全覆盖CGM、指尖血糖所有指标，无需人工配置，全局统一评判标准、统一状态标签、统一风险研判依据。

**2、后期V2\.0迭代：个性化目标管理**

迭代支持医生自定义患者个体化控制目标、区分普通/老年/高危分层目标、支持目标迭代追溯。综合研判和管理建议仍统一进入「数字孪生融合分析」Tab。

## 五、模块3：指尖血糖核心指标卡片区（二级Tab专属模块）

### 5\.1 模块说明

基于近7/30天指尖自测血糖数据，补充离散式日常血糖管控指标，与CGM连续数据形成互补，覆盖时段均值、达标率、波动、低血糖频次等维度，卡片UI、布局、入口规则与CGM指标卡完全统一。

- **1、空腹血糖均值**
**卡片完整展示内容**：主数值（保留1位小数）\+ 状态标签（正常/偏高）\+ 辅助小字（统计周期\+目标区间）
**展示样例**：6\.5mmol/L｜正常｜近7天均值｜目标：4\.4–7\.0mmol/L
**统计口径**：近7天有效合规空腹记录算术均值；空腹定义为禁食8h以上晨起血糖；有效记录≥3条方可展示数据，否则不展示或置灰无参考；
**临床标准 \&amp; 目标值**：控制目标 4\.4–7\.0mmol/L，高于上限判定为空腹血糖偏高；
**数字孪生入口**：点击指标卡片，跳转患者详情页「数字孪生融合分析」Tab。

- **2、餐后2h血糖均值**
**卡片完整展示内容**：主数值（保留1位小数）\+ 状态标签（正常/偏高）\+ 辅助小字（统计周期\+目标区间）
**展示样例**：9\.2mmol/L｜正常｜近7天餐后均值｜目标：＜10\.0mmol/L
**统计口径**：近7天有效合规餐后记录算术均值；餐后2h定义为进食第一口饭开始计时2小时血糖；有效记录≥3条方可展示数据；
**临床标准 \&amp; 目标值**：控制目标 ＜10\.0mmol/L，高于上限判定为餐后血糖控制不佳；
**数字孪生入口**：点击指标卡片，跳转患者详情页「数字孪生融合分析」Tab。

- **3、血糖达标率**
**卡片完整展示内容**：主数值（百分比整数）\+ 状态标签（达标/轻度不达标/重度不达标）\+ 辅助小字（统计周期\+目标值）
**展示样例**：75%｜达标｜近30天达标率｜目标：≥70%
**统计口径**：近30天所有有效指尖血糖测量记录中，处于正常血糖区间（3\.9–10\.0mmol/L）的记录占比；
**临床标准 \&amp; 目标值**：控制目标 ≥70%；≥70% 达标，50%–70% 轻度不达标，＜50% 重度不达标；
**数字孪生入口**：点击指标卡片，跳转患者详情页「数字孪生融合分析」Tab。

- **4、日内最大血糖波动**
**卡片完整展示内容**：主数值（保留1位小数）\+ 状态标签（稳定/波动偏大）\+ 辅助小字（统计周期\+目标值）
**展示样例**：4\.2mmol/L｜稳定｜近7天平均波动｜目标：＜5\.0mmol/L
**统计口径**：近7天每日（当日最高血糖值\-当日最低血糖值）的算术平均值，反映短期血糖稳定性；
**临床标准 \&amp; 目标值**：控制目标 ＜5\.0mmol/L；≥5\.0mmol/L判定为血糖波动偏大，存在血管损伤风险；
**数字孪生入口**：点击指标卡片，跳转患者详情页「数字孪生融合分析」Tab。

- **5、低血糖次数**
**卡片完整展示内容**：主数值（次数整数）\+ 状态标签（正常/存在低糖风险）\+ 辅助小字（统计周期\+目标值）
**展示样例**：0次｜正常｜近7天低血糖频次｜目标：0次
**统计口径**：近7天所有有效指尖血糖记录中，血糖数值＜3\.9mmol/L的有效次数汇总；
**临床标准 \&amp; 目标值**：控制目标 0次；0次为正常，＞0次提示存在低血糖隐患，反复低血糖风险偏高；
**数字孪生入口**：点击指标卡片，跳转患者详情页「数字孪生融合分析」Tab。

仅有指尖数据、无CGM数据：CGM子Tab展示「暂无CGM连续血糖监测数据」空状态，同时隐藏设备、更新时间信息；指尖血糖子Tab正常展示指标卡及分析内容。

## 六、模块4：血糖趋势曲线图表分析区（CGM专属）

### 6\.1 模块定位

基于CGM连续监测数据，多维度可视化展示患者血糖昼夜变化规律、波动趋势、异常分布，量化呈现隐匿性血糖问题，为医生判断和数字孪生页综合分析提供血糖证据。**本模块为【CGM动态血糖分析】子Tab专属模块，切换至指尖血糖子Tab时，本模块整体隐藏，指尖血糖不做连续曲线趋势图**，具体落地规则如下。

### 6\.2 展示图表类型（固定落地类型）

本模块固定展示**2类核心趋势图表**，默认双图联动展示，无缺失、无替换：

**1、单日24小时连续血糖时序曲线图（核心主图）**

以小时为粒度，展示全天每一刻实时血糖变化轨迹，精准呈现空腹、餐前、餐后、夜间、凌晨各时段血糖峰值、谷值、波动拐点，可直观识别黎明现象、餐后高糖、夜间低血糖、时段性波动等隐匿问题。

**2、多日血糖叠加对比趋势图（辅助分析图）**

叠加展示统计周期内每日血糖曲线，形成区间对比，识别**日间重复性异常规律**，区分「偶发单次异常」和「常态化血糖问题」。

### 6\.3 指尖血糖子Tab页面展示规则（最终定稿）

**1、核心展示规则**：【指尖血糖分析】二级子Tab页面仅展示「指尖血糖5项核心指标卡 + 指尖血糖明细 + 症状记录」，**不展示任何趋势图、无时序曲线、无多日叠加分析、无昼夜节律拆解**。

**2、产品设计逻辑（临床落地依据）**：指尖血糖为离散点状自测数据，无连续时序采样能力，无法生成具备临床参考价值的昼夜趋势、波动规律，强行渲染曲线会出现虚假走势、误导诊疗判断。因此指尖血糖仅做**结果统计复盘**，深度时序、节律、隐匿异常分析全部由CGM子Tab承载。

**3、页面切换显隐规则**

切换至【CGM动态血糖分析】子Tab：展示数字孪生入口、CGM指标卡、趋势图表、CGM明细和症状记录，同时展示CGM设备品牌、数据更新时间溯源信息。
切换至【指尖血糖分析】子Tab：隐藏趋势图表模块、隐藏设备溯源信息，仅保留数字孪生入口、指尖指标卡、指尖明细和症状记录。

**4、空数据兜底规则**

仅有指尖数据、无CGM数据：CGM子Tab展示「暂无CGM连续血糖监测数据」空状态；指尖血糖子Tab正常展示指标卡及分析内容。
无任何血糖数据：整体页面展示「暂无血糖监测数据」空状态。

### 6\.4 时间维度规则（固定可切换）

页面默认展示**近14天**数据（核心诊疗统计周期），支持用户手动切换3种时间维度，切换后双图表同步刷新：

1、近7天（短期精细复盘）；2、近14天（默认标准周期，适配临床CGM评估标准）；3、近30天（长期趋势复盘）

### 6\.5 图表可视化标准与标注规则

1、阈值基准线：固定绘制三条临床标准线，直观对标正常区间：低血糖线3\.9mmol/L、正常上限10\.0mmol/L、高危高糖线14\.0mmol/L；

2、异常点位标注：自动识别并高亮标记所有**低血糖点位、重度高糖点位、峰值极值点位**，点位悬浮展示具体时间\+精准血糖数值；

3、时段分区标注：图表背景自动区分晨起、餐前、餐后、夜间、凌晨时段，辅助医生快速对应生活行为与血糖异常关系；

4、数据空值处理：监测空白时段自动断连展示，不生成虚假曲线，保证数据真实性。

### 6\.6 交互规则

1、异常点位入口：曲线所有高亮高低糖异常点位支持点击，点击后跳转患者详情页「数字孪生融合分析」Tab；

2、图表缩放交互：支持框选缩放、滚轮缩放，可聚焦查看单日/单时段精细血糖波动细节；

3、时段切换交互：切换7天/14天/30天时间维度，双趋势图、异常标注、统计数据同步实时刷新；

4、悬浮提示交互：鼠标悬浮曲线任意点位，展示精准时间、血糖数值、当前时段状态（正常/轻度异常/重度异常）。

## 七、模块5：血糖明细记录区

### 7\.1 模块定位

为医生提供患者全量血糖原始溯源数据，支持CGM连续数据、指尖自测离散数据明细查看，满足医生核对原始数据、追溯异常血糖对应时间、核实管控情况的底层溯源需求。

### 7\.2 展示规则（适配双子Tab）

1、跟随顶部二级子Tab联动切换，做到**Tab数据隔离展示**：切换CGM子Tab仅展示CGM明细数据，切换指尖血糖子Tab仅展示指尖自测明细数据；

2、默认按时间倒序排列，最新数据置顶展示，方便医生查看近期血糖记录；

3、空数据兜底：当前Tab无对应类型数据时，展示「暂无对应血糖明细记录」空状态，不跨类型展示数据。

### 7\.3 字段展示规则

**CGM明细展示字段**：监测时间、实时血糖数值、时段标签（空腹/餐前/餐后/夜间）、血糖状态（正常/偏高/偏低）

**指尖血糖明细展示字段**：测量时间、血糖数值、测量时段、测量场景（空腹/餐后/随机）、状态标注

**通用能力**：支持按时间区间筛选明细记录，支持快速查看异常记录，方便医生精准溯源问题血糖。

## 八、模块6：患者近期糖尿病症状展示区

### 8\.1 模块定位

同步展示患者在患者端自主录入的近期糖尿病相关症状，为医生查看血糖数据时提供症状维度的辅助参考。

### 8\.2 展示规则

1、纯展示模块，**医生端仅查看、不可编辑**，数据完全同步患者端录入内容；

2、展示内容：近期口干、多饮、多尿、乏力、视物模糊、肢体麻木等糖尿病典型症状及患者自主备注不适；

3、无录入症状时，展示「患者暂无录入糖尿病相关症状」默认文案；

4、双Tab通用：CGM/指尖血糖子Tab下均固定展示该模块，不隐藏。

## 九、全局交互统一规则

### 9\.1 数字孪生入口统一规则

1、指标卡和图表异常点位可提供「进入数字孪生融合分析」入口，不做当前页弹窗、局部浮层展示；

2、点击任意入口，跳转患者详情页「数字孪生融合分析」Tab；

3、血糖深度分析页不生成综合判断、不输出共病融合结论、不生成管理建议。

### 9\.2 目标库迭代规则

1、本期V1\.0：全量指标预置临床标准化控制目标，统一评判、统一展示；

2、后期V2\.0：迭代个性化目标管理、分层人群目标、患者目标追溯能力。
