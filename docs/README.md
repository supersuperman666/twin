# 文档导航

当前文档分为两层：

1. `主 PRD`：按医患闭环写，强调业务目标、状态流转、任务生成和端间衔接。
2. `端侧 SPEC`：按医生端、患者端拆开，强调页面、交互、字段和开发落地。

这样做的目的不是把旧文档推倒重来，而是把强耦合业务先拉回一条主线，避免“医生端写一份、患者端写一份，但中间流转没人写完整”。

## 推荐维护原则

- 业务闭环规则优先写入主 PRD。
- 页面、字段、弹层、交互优先写入端侧 SPEC。
- 状态机、任务类型、版本规则只维护一份，避免双端分别写出两套口径。
- 老文档作为迁移缓存，不再继续横向扩写新规则。
