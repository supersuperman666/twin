# 记录指标模块 PRD

版本：V0.1  
适用端：患者微信小程序  
关联疾病：糖尿病、慢阻肺、睡眠呼吸障碍  
关联设备：医用脉搏血氧仪、睡眠呼吸监测系统

## 1. 模块定位

记录指标模块用于承载患者日常健康数据记录、设备同步数据展示、症状补充记录和用药打卡。模块需同时满足三类需求：

- 用户可快速查看当天关键健康指标。
- 用户可按日期补录或查看历史记录。
- 医生可基于记录数据判断风险、查看趋势和准备随访。

记录页不是医学指标库首页，不应平铺所有专业指标。页面应优先展示用户能理解、管理价值高、当天需要关注的核心指标；复杂设备分析结果进入二级分析页面。

## 2. 设备与疾病覆盖

### 2.1 已对接设备

| 设备 | 可产生指标 | 主要适用疾病 |
| --- | --- | --- |
| 医用脉搏血氧仪 | SpO2、脉率、ODI、睡眠呼吸事件、夜间血氧趋势 | 慢阻肺、睡眠呼吸障碍 |
| 睡眠呼吸监测系统 | AHI、呼吸暂停、低通气、体动、血氧、脉率、睡眠分期 | 睡眠呼吸障碍、慢阻肺夜间低氧评估 |

### 2.2 三类疾病核心关注点

| 疾病 | 记录页核心关注 |
| --- | --- |
| 糖尿病 | 血糖、血压、体重/BMI、低血糖症状、降糖药/胰岛素使用 |
| 慢阻肺 | SpO2、脉率、夜间低氧、咳嗽咳痰/气促、吸入药/氧疗；呼吸频率作为可选补充指标 |
| 睡眠呼吸障碍 | 睡眠报告、AHI、ODI、最低血氧、睡眠时长、呼吸事件、CPAP 使用 |

## 3. 记录页信息架构

记录页采用“日期选择 + 锚点导航 + 分区内容”的结构。

### 3.1 顶部日期

- 展示格式：`YYYY年M月D日 周X`。
- 星期几与日期同一行展示，减少纵向空间占用。
- 支持前一天/后一天切换。
- 后续可扩展日期选择器。

### 3.2 锚点导航

锚点不是筛选 Tab，而是页面定位工具。点击锚点后页面滚动到对应分区，页面内容仍保持完整展示。

| 锚点 | 对应分区 |
| --- | --- |
| 全部 | 回到页面顶部 |
| 基础指标 | BMI、身高、体重、血压、血糖 |
| 血氧呼吸 | SpO2、脉率；呼吸频率作为更多指标/可选补充 |
| 睡眠 | 睡眠报告卡片 |
| 症状 | 症状记录 |
| 用药 | 用药记录 |

交互要求：

- 锚点栏需要吸顶，用户滚动到任意位置仍可快速切换。
- 当前点击的锚点需要有选中态。
- 锚点文案单行展示，不换行。

### 3.3 设备入口

为了提升设备绑定转化率，记录模块需要在用户最可能感知“需要设备”的位置预留设备入口。

| 页面/位置 | 入口形式 | 点击去向 | 说明 |
| --- | --- | --- | --- |
| 记录页顶部 | “设备管理”轻量按钮 | `/pages/device/index` | 不抢占记录主任务，但让用户随时可绑定设备 |
| 血氧详情页 | “添加设备/绑定血氧仪”入口 | `/pages/device/index?type=oxygen` | SpO2、脉率强依赖血氧仪，适合提示绑定 |
| 睡眠报告卡片空状态 | “绑定睡眠设备”主按钮 | `/pages/device/index?type=sleep` | 无睡眠报告时优先引导绑定 |
| 睡眠分析页空/无效状态 | “查看/更换设备”入口 | `/pages/device/index?type=sleep` | 解决设备未绑定、报告生成中、报告无效 |
| 我的/健康档案 | “我的设备”入口 | `/pages/device/index` | 作为设备管理的常驻入口 |

设备管理页 MVP 先作为占位页，展示已绑定设备、可绑定设备和绑定状态；后续再细化蓝牙/扫码/设备授权流程。

### 3.4 手动录入与修改规则

除必须通过设备或算法生成的指标外，其余指标均应支持手动录入。手动录入的数据建议支持修改，但需要保留修改痕迹。

| 数据类型 | 是否支持手动录入 | 是否支持修改 | 原则 |
| --- | --- | --- | --- |
| 血糖、血压、体重、身高、呼吸频率 | 是 | 是 | 用户可能录错，必须允许修改 |
| SpO2、脉率 | 是，可手动录入设备读数 | 是 | 允许用户录入血氧仪屏幕读数 |
| 症状、用药、氧疗、CPAP 执行记录 | 是 | 是 | 属于用户行为记录，需支持补录和修正 |
| 睡眠时长、入睡/起床时间 | 是，作为无设备时补充 | 是 | 手动睡眠记录只作为基础记录，不生成 AHI/ODI |
| AHI、ODI、呼吸暂停、低通气、睡眠分期、体位、鼾声、连续血氧趋势 | 否，必须设备/算法生成 | 不直接修改 | 可添加备注、标记无效、重新同步或重新监测 |

修改规则：

- 手动记录支持编辑和删除。
- 设备原始记录不允许直接改数值，只允许添加备注、标记无效、重新同步或发起复测。
- 修改后的记录需保留 `created_at`、`updated_at`、`edit_count` 和 `edit_reason`。
- 医生端查看时应能区分“手动录入”“设备同步”“医生录入/修正”。
- 异常数据即使被用户修改，也需要保留修改前后的审计记录，避免影响风险判断追溯。

### 3.5 手动录入控件规范

手动录入不建议全部使用键盘输入。根据指标单位和数值范围选择更合适的控件，降低录入成本。

| 指标 | 推荐录入控件 | 说明 |
| --- | --- | --- |
| 血糖 | 数字滑动选择器 + 可切换键盘精确输入 | 以 0.1 mmol/L 为步进，常用范围 3.0-20.0 |
| SpO2 | 数字滑动选择器 | 以 1% 为步进，常用范围 70-100 |
| 脉率 | 数字滑动选择器 | 以 1 次/分为步进，常用范围 30-220 |
| 血压 | 双列滚轮/滑动选择器 | 收缩压、舒张压分别选择 |
| 体重 | 数字滑动选择器 | 以 0.1 kg 为步进 |
| 呼吸频率 | 数字滑动选择器 | 以 1 次/分为步进 |
| 氧流量 | 数字滑动选择器 | 以 0.5 L/min 为步进 |
| 氧疗/CPAP 时长 | 时长选择器 | 小时 + 分钟组合 |

所有手动记录表单均需要备注字段，备注为非必填，用于记录饮食、运动、症状、用药、佩戴情况等个性化信息。

#### 3.5.1 血糖记录控件落地方案

血糖记录页不得使用静态图片模拟滑动步进器。小程序实现需使用可交互控件，保证真机可滑动、可点选、可精确输入。

推荐实现优先级：

| 方案 | 微信生态控件 | 推荐程度 | 说明 |
| --- | --- | --- | --- |
| 自定义横向刻度尺 | `movable-area` + `movable-view` + 刻度 `view` | 首选 | 最接近参考图效果，可按 0.1 mmol/L 步进吸附，支持拖动游标 |
| 横向滚动刻度尺 | `scroll-view scroll-x` + 居中指示线 | 可选 | 实现更稳，滚动结束后根据 `scrollLeft` 计算血糖值 |
| 原生滑块 | `slider` | 备选 | 实现成本低，但视觉和参考图差距较大，适合作为 MVP 保底 |
| 滚轮选择 | `picker-view` | 不推荐用于血糖主输入 | 更像表单选择器，不符合参考页“刻度尺选择数值”的视觉心智 |

血糖数值输入规则：

- 默认范围：`3.0-20.0 mmol/L`，步进 `0.1 mmol/L`。
- 支持点击数值切换键盘精确输入，输入后同步移动刻度位置。
- 支持医生方案下发个体化范围，例如空腹目标、餐后 2h 目标、睡前目标。
- 超出默认范围时允许录入，但需二次确认并提示“数值异常，建议确认是否录入正确”。

#### 3.5.2 添加血糖记录页设计规范

页面标题使用小程序原生导航栏：`添加血糖记录`，页面内不再重复自定义标题。

页面结构参考用户提供的血糖记录页面，按以下顺序排布：

```text
添加血糖记录

血糖时点 Tab
凌晨 / 空腹 / 早餐后2h / 午餐前 / 午餐后2h / 晚餐前 / 晚餐后2h / 睡前 / 随机

当前时点目标范围
午餐后2h：4.4 mmol/L - 10.0 mmol/L

血糖数值刻度尺
6.6
刻度尺 + 中心指示线 + 单位 mmol/L

测量日期       2026-05-17
测量时间       23:07
血糖状态       达标
添加备注       去添加

提交
```

血糖时点 Tab 规则：

- 顶部 Tab 使用横向 `scroll-view`，每个时点为文本 Tab，当前选中项使用蓝色文字和底部短横线。
- Tab 文案不换行；空间不足时允许横向滑动。
- 默认选中规则：根据当前时间自动推荐时点，例如 05:00-09:00 默认空腹，餐后任务进入时带入对应餐后 2h，无法判断时默认随机。
- 用户切换时点后，目标范围、血糖状态和保存字段同步变化。

血糖时点与默认目标范围：

| 时点 | 字段值 | 默认目标范围 | 状态判定 |
| --- | --- | --- | --- |
| 凌晨 | `dawn` | 3.9-7.8 mmol/L | 低于 3.9 偏低，高于 7.8 偏高 |
| 空腹 | `fasting` | 4.4-7.0 mmol/L | 低于 4.4 偏低，高于 7.0 偏高 |
| 早餐后2h | `breakfast_2h` | 4.4-10.0 mmol/L | 低于 4.4 偏低，高于 10.0 偏高 |
| 午餐前 | `before_lunch` | 4.4-7.0 mmol/L | 低于 4.4 偏低，高于 7.0 偏高 |
| 午餐后2h | `lunch_2h` | 4.4-10.0 mmol/L | 低于 4.4 偏低，高于 10.0 偏高 |
| 晚餐前 | `before_dinner` | 4.4-7.0 mmol/L | 低于 4.4 偏低，高于 7.0 偏高 |
| 晚餐后2h | `dinner_2h` | 4.4-10.0 mmol/L | 低于 4.4 偏低，高于 10.0 偏高 |
| 睡前 | `bedtime` | 6.1-7.8 mmol/L | 低于 6.1 偏低，高于 7.8 偏高 |
| 随机 | `random` | 4.4-10.0 mmol/L | 低于 4.4 偏低，高于 10.0 偏高，>=11.1 提示明显偏高 |

说明：

- 上述范围为患者端默认目标，后续应支持医生端或方案配置覆盖。
- 低血糖风险优先级高于高血糖，低于 3.9 mmol/L 应提示“血糖偏低，请关注低血糖症状”。
- 若用户选择“随机”，页面仍展示参考范围，但文案应使用“参考范围”，避免误导为严格诊断标准。
- 保存时必须记录时点字段，血糖详情页、趋势图、明细页均按该字段筛选。

## 4. 记录页指标清单

### 4.1 基础指标

| 指标 | 对应疾病 | 单位/格式 | 来源 | 展示方式 |
| --- | --- | --- | --- | --- |
| BMI | 三类共用 | kg/m² | 身高体重自动计算 | 宽卡片 |
| 身高 | 三类共用 | cm | 档案/手动 | BMI 卡片内展示 |
| 体重 | 三类共用 | kg | 手动/体脂秤 | BMI 卡片内展示 |
| 血压 | 糖尿病、睡眠呼吸障碍共病风险 | mmHg | 手动/血压计 | 指标卡片 |
| 血糖 | 糖尿病 | mmol/L | 手动/血糖仪 | 单一指标卡片，展示最新血糖记录 |

说明：

- BMI 卡片应紧凑展示：`BMI + 数值 + 状态` 同行，身高和体重在卡片底部横向展示。
- BMI、身高、体重属于健康档案型基础资料，不单独建设基础指标详情页；记录页 P0 不承担健康档案入口。
- 基础指标区只展示一个“血糖”卡片，不拆分为空腹血糖、餐后 2 小时血糖、随机血糖多个卡片。
- 血糖卡片展示当前日期下最新一条血糖记录值，测量时点通过标签体现，例如“空腹”“午餐后2h”“随机”“睡前”。
- 血糖测量时点标签应与数值行右侧对齐，不单独占行，避免卡片被撑高。
- 若用户需要补录不同类型血糖，在添加/编辑记录表单中选择测量时点，不在基础指标区新增独立卡片。

### 4.2 血氧呼吸

| 指标 | 对应疾病 | 单位/格式 | 来源 | 展示方式 |
| --- | --- | --- | --- | --- |
| SpO2 | 慢阻肺、睡眠呼吸障碍 | % | 血氧仪/睡眠监测设备 | 指标卡片 |
| 脉率 | 慢阻肺、睡眠呼吸障碍 | 次/分 | 血氧仪/睡眠监测设备 | 指标卡片 |
| 呼吸频率 | 慢阻肺 | 次/分 | 手动/设备 | 更多指标/可选补充，不默认展示 |

说明：

- 当前设备采集更准确称为“脉率”，页面可展示为“脉率”，不要同时展示“心率”和“脉率”造成重复。
- 血氧呼吸模块默认只展示 SpO2 和脉率，二者可由血氧仪直接采集，用户理解成本低。
- 呼吸频率可通过静息状态下数 1 分钟胸腹起伏次数手动记录，但普通用户自测误差较大，不作为默认高频指标。
- 夜间最低血氧属于整晚连续监测结果，归入睡眠报告卡片和睡眠分析二级页，不放在血氧呼吸模块。
- 氧疗时长和氧流量不是测量指标，而是治疗执行记录，应归入用药/治疗记录。

### 4.3 症状记录

症状不作为每日必填，不应默认铺开所有症状项。用户先选择是否有不适，再选择具体症状。

| 症状类型 | 对应疾病 | 记录方式 |
| --- | --- | --- |
| 呼吸相关 | 慢阻肺 | 咳嗽、咳痰、痰色、气促、胸闷、喘息 |
| 睡眠相关 | 睡眠呼吸障碍 | 打鼾、夜间憋醒、白天嗜睡、晨起头痛 |
| 血糖相关 | 糖尿病 | 心慌、出汗、手抖、饥饿、头晕、乏力 |

记录页展示：

- 有记录时展示最近一条症状摘要、严重程度、时间。
- 无其他症状时展示空状态。
- 分区右侧提供“添加”入口。

### 4.4 用药记录

| 指标 | 对应疾病 | 单位/格式 | 来源 |
| --- | --- | --- | --- |
| 降糖药使用 | 糖尿病 | 已服/漏服、剂量、时间 | 手动/处方 |
| 胰岛素使用 | 糖尿病 | 剂量 U、注射时间 | 手动/处方 |
| 吸入药使用 | 慢阻肺 | 已用/漏用、频次 | 手动/处方 |
| 氧疗使用 | 慢阻肺 | 是否吸氧、氧流量、开始时间、结束时间、累计时长 | 手动/设备 |
| CPAP 使用 | 睡眠呼吸障碍 | 小时/晚 | 手动/设备 |

记录页展示：

- 展示今日用药记录列表。
- 每条包含时间、药品/治疗名称、剂量、服用状态。
- 支持补打卡和漏服原因。
- 氧疗使用属于治疗记录，不归入血氧呼吸指标卡。用户根据制氧机/氧气设备显示的流量和实际使用时长填写。

## 5. 睡眠模块设计

### 5.1 产品判断

睡眠数据应作为一个整体模块展示，不应把 AHI、ODI、呼吸暂停、低通气、睡眠分期等全部拆成普通指标卡片。

原因：

- 睡眠数据是“一整晚连续监测报告”，不是某个时间点的单次记录。
- AHI、ODI、睡眠分期、低氧累计时长等多为设备/算法二次加工结果。
- 用户在记录页需要快速看结论，医生和高级用户才需要进入二级页看原因、趋势和细节。

产品原则：

> 记录页看结果，睡眠二级页看原因和趋势。

### 5.2 睡眠相关指标类型

设备口径：

- ZG-P11H / ZG-P11G 指环血氧仪：核心采集 SpO2、脉率、连续夜间血氧趋势，可分析 ODI、低氧事件、睡眠呼吸事件。
- ZG-M11A 睡眠呼吸监测系统：由呼吸体动传感器、指环式脉搏血氧传感器、贴片式体位传感器和软件组成，可监测脉搏血氧饱和度、脉率、胸腹呼吸、体位，并通过软件生成睡眠呼吸报告。
- ZG-M11B 睡眠呼吸监测系统：不含体位传感器，体位相关模块需要展示“本设备/本次报告暂不支持”。

| 指标 | 类型 | 来源 | 是否放记录页 |
| --- | --- | --- | --- |
| 睡眠时长 | 基础结果 | 睡眠监测系统/手动 | 是 |
| 入睡时间 | 基础结果 | 睡眠监测系统/手动 | 二级页 |
| 起床/醒来时间 | 基础结果 | 睡眠监测系统/手动 | 二级页 |
| 入睡耗时 | 分析指标 | 睡眠监测系统 | 二级页 |
| 夜醒次数/清醒时长 | 分析指标 | 睡眠监测系统 | 二级页 |
| AHI | 分析指标 | 睡眠呼吸监测系统 | 是，作为核心结论 |
| ODI | 分析指标 | 血氧仪/睡眠监测系统 | 是，作为核心结论 |
| 最低血氧 | 夜间血氧分析 | 血氧仪/睡眠监测系统 | 是，作为风险指标 |
| 平均血氧 | 夜间血氧分析 | 血氧仪/睡眠监测系统 | 二级页 |
| 低氧累计时长 | 夜间血氧分析 | 血氧仪/睡眠监测系统 | 记录页摘要 + 二级页 |
| 呼吸暂停次数 | 事件统计 | 睡眠呼吸监测系统 | 二级页 |
| 低通气次数 | 事件统计 | 睡眠呼吸监测系统 | 二级页 |
| 睡眠呼吸事件总数 | 事件统计 | 血氧仪/睡眠监测系统 | 二级页 |
| 呼吸事件类型 | 分型统计 | 睡眠呼吸监测系统 | 二级页；阻塞型/中枢型/混合型如设备报告支持则展示 |
| 睡眠分期 | 算法分析 | 睡眠呼吸监测系统 | 二级页；清醒、浅睡、深睡、REM |
| 体动次数/指数 | 算法分析 | 睡眠呼吸监测系统 | 二级页 |
| 体位 | 体位传感器 | ZG-M11A 支持；ZG-M11B 不支持 | 二级页；仰卧、左侧、右侧、俯卧占比及体位相关 AHI |
| 鼾声 | 声音/算法模块 | 设备或报告能力支持时展示 | 二级页；鼾声时长、鼾声强度、鼾声分布 |
| SpO2 趋势 | 连续数据 | 血氧仪/睡眠监测系统 | 二级页 |
| 脉率趋势 | 连续数据 | 血氧仪/睡眠监测系统 | 二级页 |

### 5.3 记录页睡眠展示

记录页睡眠分区应展示一个“睡眠报告卡片”，而不是多个普通小卡片。

卡片内容：

| 区域 | 内容 |
| --- | --- |
| 顶部 | 昨晚睡眠报告、风险等级、查看详情 |
| 时间 | 监测区间，例如 `05.15 22:48 - 05.16 06:20` |
| 核心指标 | 睡眠时长、AHI、最低血氧 |
| 辅助指标 | ODI、低氧累计时长 |
| 摘要 | 例如“血氧低于 90% 累计 12 分钟，建议继续监测” |

推荐布局：

```text
睡眠
  昨晚睡眠报告                         轻度风险
  05.15 22:48 - 05.16 06:20

  睡眠时长        AHI             最低血氧
  6.5 小时        3.2 次/小时      88%

  ODI 2.8 次/小时    血氧低于90%累计12分钟
```

交互：

- 点击整张卡片进入睡眠分析二级页。
- “查看详情”进入同一二级页。
- 若当晚无设备数据，展示空状态：“昨晚暂无睡眠报告”，提供“手动记录睡眠时长”或“绑定设备”入口。

### 5.4 睡眠二级分析页

睡眠二级页用于展示整晚报告详情，解释 AHI/ODI 的来源、血氧风险、睡眠质量和长期趋势。

#### 5.4.1 页面结构

二级页建议分为 7 个模块：

1. 报告概览
2. 呼吸事件分析
3. 血氧分析
4. 睡眠结构
5. 体位分析
6. 鼾声与体动
7. 趋势与建议

#### 5.4.2 报告概览

用途：让用户和医生一眼判断整晚睡眠风险。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 睡眠时长 | 小时/分钟 | 基础睡眠情况 |
| 入睡时间 | HH:mm | 睡眠开始时间 |
| 起床/醒来时间 | HH:mm | 睡眠结束时间 |
| 入睡耗时 | 分钟 | 入睡难易程度 |
| AHI | 次/小时 | OSA 严重程度核心指标 |
| ODI | 次/小时 | 夜间氧减频率 |
| 最低血氧 | % | 夜间低氧风险 |
| 平均血氧 | % | 整晚氧合水平 |
| 低氧累计时长 | 分钟 | 夜间低氧负担 |
| 风险等级 | 正常/轻度/中度/重度 | 用户理解用 |

推荐布局：

```text
睡眠分析
昨晚睡眠报告                       轻度风险
05.15 22:48 入睡  05.16 06:20 起床

睡眠时长  6.5小时
入睡耗时  18分钟
AHI       3.2次/小时
ODI       2.8次/小时
最低血氧  88%
平均血氧  94%
低氧累计  12分钟
```

#### 5.4.3 呼吸事件分析

用途：解释 AHI 从哪里来，帮助医生判断呼吸暂停和低通气负担。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 呼吸暂停次数 | 次/晚 | 整晚识别出的暂停事件 |
| 低通气次数 | 次/晚 | 整晚识别出的低通气事件 |
| 睡眠呼吸事件总数 | 次/晚 | 暂停 + 低通气等事件汇总 |
| 阻塞型事件 | 次/晚 | 如报告支持，展示阻塞型呼吸暂停/低通气 |
| 中枢型事件 | 次/晚 | 如报告支持，展示中枢型呼吸暂停 |
| 混合型事件 | 次/晚 | 如报告支持，展示混合型事件 |
| 事件分布 | 时间轴/柱状图 | 展示事件集中发生时段 |

推荐布局：

```text
呼吸事件
呼吸暂停  12次
低通气    18次
事件总数  30次
阻塞型 8次  中枢型 2次  混合型 2次

事件分布：按小时柱状图/时间轴
```

#### 5.4.4 血氧分析

用途：评估夜间低氧风险，慢阻肺和睡眠呼吸障碍都需重点关注。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 最低血氧 | % | 判断夜间危险低氧 |
| 平均血氧 | % | 判断整体氧合水平 |
| 低于 90% 累计时长 | 分钟 | 判断低氧负担 |
| ODI | 次/小时 | 判断血氧下降频率 |
| SpO2 趋势 | 曲线 | 查看低氧发生时段 |

推荐布局：

```text
血氧分析
最低血氧 88%     平均血氧 94%
低于90%累计 12分钟
ODI 2.8次/小时

SpO2 趋势曲线
```

#### 5.4.5 睡眠结构

用途：展示睡眠质量，而不是直接判断呼吸风险。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 入睡时间 | HH:mm | 睡眠开始时间 |
| 醒来时间 | HH:mm | 睡眠结束时间 |
| 清醒时长 | 分钟 | 夜醒/睡眠中断 |
| 浅睡时长 | 小时/分钟 | 睡眠结构 |
| 深睡时长 | 小时/分钟 | 恢复质量 |
| REM 时长 | 小时/分钟 | 睡眠结构 |
| 睡眠效率 | % | 实际睡眠时长 / 卧床时长 |
| 体动次数/指数 | 次/次每小时 | 睡眠稳定性 |

推荐布局：

```text
睡眠结构
入睡 22:48    醒来 06:20

清醒 35分钟
浅睡 3.8小时
深睡 1.4小时
REM  0.8小时

横向堆叠条/环形图
```

#### 5.4.6 体位分析

用途：判断呼吸事件是否与体位有关。ZG-M11A 含贴片式体位传感器，支持展示；ZG-M11B 若未返回体位数据，则模块展示为不可用状态。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 仰卧时长/占比 | 分钟/% | 体位分布 |
| 左侧卧时长/占比 | 分钟/% | 体位分布 |
| 右侧卧时长/占比 | 分钟/% | 体位分布 |
| 俯卧时长/占比 | 分钟/% | 体位分布 |
| 体位相关 AHI | 次/小时 | 如报告支持，展示不同体位下的事件频率 |

推荐布局：

```text
体位分析
仰卧 46%   左侧 28%   右侧 22%   俯卧 4%

仰卧位 AHI 5.8次/小时
侧卧位 AHI 1.6次/小时
建议减少仰卧睡眠
```

#### 5.4.7 鼾声与体动

用途：辅助解释睡眠质量、呼吸事件和用户主诉。鼾声是否可展示取决于设备报告是否输出鼾声字段；如果未输出，不应在页面伪造数据。

| 指标 | 单位 | 说明 |
| --- | --- | --- |
| 鼾声时长 | 分钟 | 整晚鼾声累计时间 |
| 鼾声强度 | 轻/中/重或 dB | 取决于设备能力 |
| 鼾声分布 | 时间轴 | 查看鼾声集中时段 |
| 体动次数 | 次 | 睡眠稳定性 |
| 体动指数 | 次/小时 | 如设备报告支持则展示 |

推荐布局：

```text
鼾声与体动
鼾声累计 48分钟     体动 36次

鼾声/体动时间轴
02:00-04:00 鼾声和低氧更集中
```

#### 5.4.8 趋势与建议

用途：支持长期管理和医生随访判断。

| 内容 | 展示方式 |
| --- | --- |
| 近 7 天 AHI 趋势 | 折线图 |
| 近 7 天最低血氧趋势 | 折线图 |
| 近 7 天睡眠时长趋势 | 柱状/折线图 |
| 近 7 天低氧累计时长 | 柱状图 |
| 近 7 天仰卧占比 | 折线/柱状图；仅 ZG-M11A 或有体位数据时展示 |
| 系统/医生建议 | 文本建议卡片 |

推荐建议文案：

- 今晚继续佩戴监测设备。
- 若最低血氧持续低于 90%，建议联系医生复评。
- 若白天嗜睡明显或驾驶困倦，建议尽快线下就医评估。

## 6. 二级页面设计

### 6.1 页面分工

记录页所有主要卡片均可点击，但二级页深度按业务价值分级，不做低价值页面堆叠。

| 记录页入口 | 点击去向 | 页面类型 | 原因 |
| --- | --- | --- | --- |
| BMI/身高/体重卡片 | 无二级详情页 | - | 身高、体重、BMI 属于基础档案与长期体重管理信息；记录页 P0 不承担健康档案入口 |
| 血压卡片 | 血压详情页 | 单点指标详情页 | 血压需查看收缩压/舒张压趋势和历史记录 |
| 血糖卡片 | 血糖详情页 | 独立指标详情页 | 糖尿病核心指标，需要按凌晨、空腹、三餐前后 2h、睡前、随机等时点查看趋势 |
| SpO2 卡片 | 血氧详情页 | 独立指标详情页 | 慢阻肺和睡眠呼吸障碍共用核心指标，需要查看 SpO2 与脉率趋势 |
| 脉率卡片 | 血氧详情页 | 独立指标详情页 | 脉率通常与 SpO2 同设备采集，合并在血氧详情页查看 |
| 睡眠报告卡片 | 睡眠分析页 | 深度分析页 | 睡眠为整晚报告，需查看 AHI、ODI、呼吸事件、血氧趋势、睡眠结构 |
| 症状记录 | 症状详情页 | 中等详情页 | 症状需按类型、程度、时间趋势查看 |
| 用药记录 | 用药/治疗详情页 | 中等详情页 | 用药、吸入药、氧疗、CPAP 需要查看依从性和执行历史 |

页面层级原则：

- 指标详情页用于展示最新值、趋势、目标范围和最近少量记录，不展示全部历史明细。
- 完整历史数据进入“指标记录明细页”，仅支持顶部日期范围筛选，最多查看 30 天数据，列表按日期倒序分组展示。
- 单条记录完整信息不新开独立页面，使用半屏详情抽屉承载，避免患者端层级过深。
- 睡眠报告页是单次整晚报告分析页，不在底部展示全部历史报告；历史睡眠报告使用独立列表页。

### 6.2 血糖详情页

血糖详情页必须独立建设，不与其他指标混用。

页面目标：

- 查看最新血糖值和测量时点。
- 按测量时点查看趋势。
- 支持查看历史记录、编辑记录和新增记录。
- 支持异常值复测提示。

页面内容：

```text
血糖详情

最新血糖
5.0 mmol/L        正常
空腹 · 05.16 14:53

时点筛选
全部 / 凌晨 / 空腹 / 早餐后2h / 午餐前 / 午餐后2h / 晚餐前 / 晚餐后2h / 睡前 / 随机

趋势
近7天 / 近30天
折线图

目标范围
空腹 4.4-7.0 mmol/L
餐后2小时 <10.0 mmol/L
睡前 6.1-7.8 mmol/L

记录列表
05.16 14:53   5.0 mmol/L   空腹
05.15 13:05   8.6 mmol/L   午餐后2h
```

交互：

- 点击新增记录进入血糖记录表单。
- 点击历史记录进入编辑状态。
- 时点筛选会同步影响趋势图和记录列表。
- 异常值保留，但提示“建议复测/联系医生”。
- 详情页仅展示最近 3 条记录，点击“查看全部记录”进入指标记录明细页。

### 6.3 血氧详情页

血氧详情页必须独立建设，承载 SpO2 与脉率两个强关联指标。

页面目标：

- 查看最新 SpO2 和脉率。
- 查看不同场景下的血氧变化。
- 支持慢阻肺日常血氧管理。
- 区分日常血氧和睡眠夜间低氧，避免数据语义混乱。

页面内容：

```text
血氧详情

最新血氧
SpO2 98%         正常
脉率 72次/分
静息 · 05.16 08:30

场景筛选
全部 / 静息 / 活动后 / 睡前

趋势
SpO2 趋势
脉率趋势

记录列表
05.16 08:30   98%   72次/分   静息
05.15 21:20   94%   78次/分   睡前
```

交互：

- 点击 SpO2 或脉率卡片均进入血氧详情页。
- 点击新增记录进入血氧记录表单。
- SpO2 偏低时提示复测。
- 夜间最低血氧不在血氧详情页作为核心字段展示，进入睡眠分析页查看。
- 详情页仅展示最近 3 条记录，点击“查看全部记录”进入指标记录明细页。

### 6.4 血压详情页

页面目标：

- 查看最新血压和控制状态。
- 查看收缩压、舒张压双线趋势。
- 支持历史记录编辑和补录。

页面内容：

```text
血压详情

最新血压
120/90 mmHg      正常
05.16 22:21

趋势
近7天 / 近30天
收缩压 / 舒张压双线图

目标范围
建议控制在 <140/90 mmHg

记录列表
05.16 22:21   120/90 mmHg
05.15 07:40   132/84 mmHg
```

交互：

- 详情页仅展示最近 3 条记录。
- 点击“查看全部记录”进入指标记录明细页。
- 手动血压记录可编辑、删除；设备同步记录仅支持备注、标记无效或重新同步。

### 6.5 健康档案边界

- 健康档案页面结构以 [健康档案页 SPEC](/Users/ks-hz/Desktop/数字孪生/docs/spec/健康档案页SPEC.md) 为准。
- 记录页 P0 不承担健康档案入口，BMI、身高、体重卡片仅作为基础指标摘要展示。
- 体重/BMI 日常趋势若建设，归入记录或后续体重管理能力，不在本记录页 PRD 展开。

### 6.6 睡眠分析页

睡眠分析页为深度分析页，承接记录页的整晚睡眠报告卡片。记录页不拆开平铺 AHI、ODI、最低血氧等睡眠事件指标，避免用户误以为这些指标需要手动记录。

页面目标：

- 解释昨晚睡眠报告是否正常，以及异常来自哪里。
- 展示 AHI、ODI、最低血氧、低氧累计时长、呼吸暂停/低通气次数等设备分析结果。
- 帮助用户判断是否需要复测、规范佩戴设备或联系医生。
- 支持查看近 7 天/近 30 天睡眠趋势。

页面结构：

```text
睡眠分析

报告概览
睡眠时长 6.5小时    偏低
AHI 3.2次/小时      正常
ODI 2.8次/小时      正常
最低血氧 88%        偏低

呼吸事件
呼吸暂停次数 / 低通气次数 / 事件总数
事件发生时间分布

血氧分析
最低血氧 / 平均血氧 / 低于90%累计时长
夜间血氧趋势图

睡眠结构
清醒 / 浅睡 / 深睡 / REM 占比
体动次数

趋势与建议
近7天 AHI、最低血氧、睡眠时长趋势
系统建议 / 医生建议
```

交互：

- 点击记录页睡眠报告卡片进入睡眠分析页。
- 若报告未生成，页面展示“报告分析中”，不展示空图表。
- 若设备未绑定，页面展示绑定设备入口。
- 趋势周期支持近 7 天、近 30 天切换。
- 异常项可点击查看解释，例如 AHI、ODI、最低血氧的含义和风险提示。

详细字段和展示规则详见本 PRD 第 5.4 节。

### 6.7 症状详情页

页面目标：

- 查看今日症状、症状类型、严重程度和近 7 天变化。
- 支持呼吸相关、睡眠相关、血糖相关症状筛选。
- 支持快捷记录“今日无不适”。

页面结构：

```text
症状详情

今日状态
今日有症状 / 今日无不适
最高严重程度：轻度/中度/重度
```

## 7. 数据结构建议

### 7.1 指标记录 `health_metric_record`

用于承载手动记录和单点设备数据。该表也是后续数字孪生建模的基础事实表，必须保留数据来源、测量场景、目标范围、症状关联和修改痕迹。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 记录 ID |

| `metric_code` | string | 指标编码，例如 `fpg`、`spo2`、`bp` |
| `metric_name` | string | 指标名称 |
| `value` | number/string | 指标值 |
| `unit` | string | 单位 |
| `measure_point` | string | 测量时点，血糖必填，例如 `fasting`、`lunch_2h`、`bedtime`；其他指标可为空 |
| `target_min` | number | 记录时使用的目标下限，便于后续医生目标变更后仍可追溯 |
| `target_max` | number | 记录时使用的目标上限，便于状态复算和历史展示 |
| `recorded_at` | datetime | 记录时间 |
| `source` | enum | `manual`、`device`、`doctor` |

| `status` | enum | `normal`、`warning`、`danger`、`unknown` |

| `related_symptom_ids` | array | 关联症状记录 ID，用于判断指标异常是否伴随体感异常 |

- 患者备注、饮食运动、用药、睡眠等上下文信息应尽量结构化，不只作为纯文本。
- 设备原始数据不允许覆盖修改，标记无效时仍保留原始值和无效原因。
- 记录时使用的目标范围必须随记录保存，避免医生后续调整目标后历史状态失真。

## 9. 验收标准

- 记录页可以按日期查看当天基础指标、血氧呼吸、睡眠、症状、用药记录。
- 记录页、血氧详情页、睡眠空状态等适合位置需预留设备管理入口。
- 非设备/算法强依赖指标均支持手动录入，手动录入数据支持修改。
- 设备原始数据不允许直接修改数值，只允许备注、标记无效、重新同步或复测。
- 所有手动录入表单均包含非必填备注字段。
- 血糖、血氧、脉率、血压、体重等数值录入优先使用滑动选择/滚轮选择，保留键盘精确输入能力。
- 添加血糖记录页必须支持 9 个测量时点：凌晨、空腹、早餐后2h、午餐前、午餐后2h、晚餐前、晚餐后2h、睡前、随机。
- 添加血糖记录页切换时点后，必须同步展示该时点默认目标范围，并实时计算血糖状态。
- 血糖滑动步进器必须是真实可交互控件，不允许使用静态图片；优先采用 `movable-area` + `movable-view` 或横向 `scroll-view` 实现。
- BMI/身高/体重卡片不进入独立指标详情页；记录页 P0 不承担健康档案入口。
- 血糖卡片点击后进入独立血糖详情页。
- SpO2 和脉率卡片点击后进入独立血氧详情页。
- 指标详情页仅展示最近少量记录，不展示全部历史明细。
- 指标记录明细页仅支持顶部日期范围筛选，精确到日，默认最近 30 天，最大跨度 30 天。
- 指标记录明细页不提供全部来源、全部场景、按年、按月、按日、全部等筛选项。
- 指标记录明细页按日期倒序分组展示记录，字段包含记录时间、指标、数值、单位、状态、来源、设备号、备注摘要。
- 点击单条记录打开半屏详情抽屉，不新增独立详情页面。
- 单条详情抽屉中，手动记录支持编辑/删除，设备记录支持备注/标记无效/重新同步。
- 锚点导航点击后可定位到对应分区，且锚点栏吸顶。
- 睡眠区域以整体报告卡片展示，不平铺所有睡眠事件指标。
- 睡眠报告卡片点击后进入睡眠二级分析页。
- 睡眠报告页不展示全部历史报告；历史报告进入睡眠历史报告页。
- 睡眠二级页包含报告概览、呼吸事件、血氧分析、睡眠结构、体位分析、鼾声与体动、趋势与建议七类信息。
- 睡眠二级页需展示入睡时间、起床时间、入睡耗时、清醒/浅睡/深睡/REM 等睡眠阶段数据。
- 体位数据需按设备能力展示：ZG-M11A 可展示，ZG-M11B 或未返回体位数据时展示不可用状态。
- AHI、ODI、呼吸暂停、低通气、睡眠分期等设备分析指标不要求用户手动填写。
- 设备未绑定、无报告、报告生成中、报告无效均有明确状态。
- 记录提交或设备同步后，首页、今日待办、风险预警状态同步更新。
