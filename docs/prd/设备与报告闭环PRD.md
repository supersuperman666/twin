# 设备与报告闭环PRD

版本：V0.1  
适用范围：医生 PC 管理端 + 患者微信小程序  
文档定位：承接设备绑定、数据同步、报告生成、可信度判断和回流预警/方案/随访的闭环

## 1. 业务定位

设备与报告不是独立孤岛，它的价值在于让居家设备和睡眠/血氧数据真正进入管理闭环。

它承接的核心包括：

- 设备绑定与同步
- 睡眠报告和设备报告展示
- 数据可信度判断
- 将数据回流到预警、方案、随访和医生判断

## 2. 设备管理

医生端和患者端都需要看到基础设备状态。

### 2.1 展示内容

- 已绑定设备
- 设备型号、设备号、绑定时间
- 最近同步时间
- 电量和同步状态
- 支持指标能力
- 异常和无效数据

### 2.2 患者端作用

- 绑定设备
- 查看同步状态
- 检查连接问题
- 在必要时转手动记录

### 2.3 医生端作用

- 识别是否有关键设备未绑定
- 识别是否存在同步异常
- 判断报告是否可用于方案与预警

## 3. 睡眠报告

### 3.1 关键展示内容

- 睡眠时长、入睡时间、起床时间
- AHI、ODI、最低血氧、平均血氧、低氧累计时长
- 呼吸暂停、低通气、事件类型
- 睡眠分期、体位、鼾声、体动
- CPAP 使用时长、漏气、残余 AHI
- 报告有效性和设备能力说明

### 3.2 医生端动作

- 查看完整报告
- 标记报告无效
- 建议复测睡眠
- 建议睡眠门诊或 CPAP 随访

## 4. 数据可信度

系统需要显式提示以下维度：

- 手动记录、设备采集、医生录入、院内检查的来源差异
- 设备原始数据是否被标记无效
- 是否存在连续缺失
- 是否存在设备不同型号导致的指标能力差异

## 5. 与任务、预警、方案的关系

### 5.1 与任务

- 设备绑定和同步异常可以生成设备任务
- 睡眠报告可以生成报告查看、确认或补充说明任务
- 设备采集成功可自动完成对应任务

### 5.2 与预警

- 夜间低氧、AHI/ODI 异常、CPAP 使用不足可以直接触发预警
- 设备同步异常可以触发提醒或重要预警

### 5.3 与方案

- 设备依赖型方案必须判断数据来源是否可靠
- AHI、最低血氧、CPAP 使用时长等目标依赖设备数据时，设备监测方案不应随意关闭

### 5.4 与随访

- 睡眠报告、血氧数据、CPAP 执行情况是随访前重要准备材料
- 随访时需要把设备数据和患者主诉结合判断

## 6. 患者端展示原则

- 患者端看到的是“是否已同步、是否需要处理、报告核心摘要”
- 不展示复杂医学解释和过多原始技术字段
- 异常时优先给动作建议，而不是大量图表

## 7. 医生端展示原则

- 医生端可以看更完整的原始数据和可信度标识
- 必须能看出当前数据是否适合用于临床判断
- 设备异常和报告无效要显式标记，不能默默混在正常数据里

## 8. 一句话总结

**设备与报告的核心不是“有数据”，而是“这些数据能不能可靠地进入医生决策”。**
