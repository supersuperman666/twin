# 患者管理闭环PRD

版本：V0.1  
适用范围：医生 PC 管理端  
文档定位：承接医生端从找患者、筛患者、看状态，到进入患者 360 深度处理的工作流

## 1. 业务定位

患者管理列表用于帮助医生、家庭医生和慢病管理人员快速完成“找患者、看状态、筛选处理”。列表页不承载深度分析，深度分析进入患者 360。

P0 阶段患者管理是医生端默认首页，同时承担轻量工作台能力：通过顶部统计卡和快捷筛选帮助医生发现需要关注的患者，但不建设独立待办工作台。

## 2. 核心任务

- 找到某个患者
- 找到当前最需要处理的患者
- 识别高风险、待随访、数据缺失、待确认方案患者
- 快速进入预警、方案、随访、疾病确认等后续处理流

## 3. 页面布局

```text
顶部工具区
  搜索：患者姓名 / 患者ID / 患者手机号
  主按钮：添加患者（绑定二维码）

统计概览
  全部患者 / 重点关注 / 疾病风险待确认 / 待处理预警 / 待确认方案 / 今日待随访 / 数据缺失 / 更多筛选

高级筛选区
  默认收起
  点击更多筛选后展开

患者列表
  紧凑表格
  患者信息 + 状态标签 + 快捷入口
```

## 4. 顶部统计卡

| 统计卡 | 口径 | 点击行为 |
| --- | --- | --- |
| 全部患者 | 当前医生授权患者总数 | 清空快捷筛选 |
| 重点关注 | 医生主动标记重点关注的患者 | 仅看重点关注 |
| 疾病风险待确认 | 风险已生成但医生尚未确认 | 筛选待确认疾病风险患者 |
| 待处理预警 | 存在待医生处理预警的患者 | 筛选预警患者 |
| 待确认方案 | 系统已生成方案但尚未医生确认 | 筛选待确认方案患者 |
| 今日待随访 | 今天计划随访或逾期未处理 | 筛选今日/逾期随访患者 |
| 数据缺失 | 连续缺失关键记录或设备未同步 | 筛选数据缺失患者 |
| 更多筛选 | 当前已选择的高级筛选数量 | 展开/收起筛选面板 |

## 5. 筛选条件

| 筛选组 | 选项 |
| --- | --- |
| 重点关注 | 全部、仅看重点关注 |
| 风险状态 | 稳定、需关注、需干预 |
| 疾病风险 | 糖尿病风险、慢阻肺风险、睡眠呼吸障碍风险、高血压风险、待确认 |
| 确诊疾病 | 糖尿病、慢阻肺、睡眠呼吸障碍、高血压、多病共管 |
| 待处理事项 | 有预警、待确认方案、今日待随访、数据缺失 |
| 方案状态 | 无方案、草稿、待患者确认、执行中、已停用、已完成 |
| 随访状态 | 无随访、今日待随访、逾期随访、近期已随访 |
| 数据状态 | 正常、关键指标缺失、设备未同步、记录完整率低 |
| 医患关系 | 主责医生患者、协作医生患者 |
| 设备状态 | 已绑定、未绑定、同步异常 |

## 6. 患者列表字段

| 字段 | 说明 |
| --- | --- |
| 患者信息 | 星标、姓名、性别、年龄、手机号脱敏、患者 ID、绑定时间 |
| 医患关系 | 当前主责医生、医生角色、是否存在协作医生 |
| 疾病标签 | 医生确认疾病、疾病风险标签、待确认状态 |
| 今日状态 | 今日待办完成度、是否有新记录 |
| 最新关键指标 | 血糖、血压、SpO2、睡眠摘要、症状摘要 |
| 数字孪生状态 | 稳定、趋势偏离、风险升高、干预中 |
| 最新预警 | 预警等级、名称、触发时间 |
| 设备状态 | 已绑定、未绑定、同步异常 |
| 方案状态 | 无方案、草稿、待患者确认、执行中、已停用、已完成 |
| 随访状态 | 下次随访日期、是否逾期、随访类型 |
| 操作 | 查看详情、更多 |

## 7. 行内快捷操作

### 7.1 主操作

- 查看详情

### 7.2 更多菜单

P0 只保留必要入口：

| 操作 | 触发条件 | 结果 |
| --- | --- | --- |
| 去处理预警 | 有待处理预警 | 跳转预警中心 |
| 去确认方案 | 有方案草稿待审核 | 跳转方案管理 |
| 去完成随访 | 今日待随访或逾期 | 跳转随访计划 |
| 去确认疾病 | 有疾病风险待确认 | 进入患者详情健康筛查区域 |
| 查看数据缺失 | 关键指标缺失或设备同步异常 | 进入患者详情对应区域 |

## 8. 星标重点关注

- 星标固定在患者信息首列
- 点击空心星标立即标记重点关注
- 点击实心星标取消重点关注
- 不弹确认框
- 操作写入时间轴

## 9. 全局分页

所有主列表页面统一使用分页组件。

规则：

- 展示当前范围和总数
- 支持上一页、下一页
- 支持页码跳转
- 默认 20 条/页
- 筛选、搜索、切换统计卡后回到第 1 页

## 10. 空状态与异常状态

| 场景 | 页面表现 |
| --- | --- |
| 无患者 | 展示暂无患者，主按钮为添加患者 |
| 无筛选结果 | 展示当前筛选无患者，支持清空筛选 |
| 数据同步失败 | 列表顶部提示部分数据同步延迟 |
| 患者已解绑 | 行置灰，保留历史记录 |
| 权限不足 | 不展示敏感信息，提示无查看权限 |

## 11. 患者 360

患者 360 是医生端单患者主页面，按“概览 - 证据 - 决策 - 执行 - 反馈”组织信息。

### 11.1 页面结构

```text
顶部患者条
  基础信息 / 医患关系 / 疾病标签 / 风险等级 / 快捷操作

左侧锚点导航
  总览、数字孪生、指标趋势、记录明细、睡眠报告、设备、预警、方案、随访、医生建议

中间主内容区
  当前选中模块详情

右侧决策侧栏
  当前风险、系统建议、医生操作、最近处理记录
```

### 11.2 顶部患者条

展示：

- 姓名、性别、年龄、手机号脱敏
- 当前主责医生、医生角色
- 疾病风险标签、医生确认疾病、风险等级
- 绑定设备、最近同步时间
- 当前方案、方案执行率
- 待处理预警和待随访

### 11.3 总览模块

目标是让医生在 1 分钟内理解当前状态。

核心内容：

- 当前结论
- 疾病标签
- 近期异常
- 今日待办完成度
- 方案执行
- 最近医生建议

### 11.4 时间轴模块

时间轴按时间倒序聚合关键事件：

- 筛查
- 记录
- 设备
- 预警
- 方案
- 随访
- 医生建议

### 11.5 右侧决策侧栏

根据当前状态动态变化：

| 状态 | 主操作 |
| --- | --- |
| 紧急预警 | 处理预警、建议就医、创建随访 |
| 待确认疾病标签 | 确认疾病、标记资料不足 |
| 疑似治疗反应不足 | 复核方案、建议检查、转专科 |
| 数据连续缺失 | 发送提醒、创建随访 |
| 方案到期 | 方案复盘、创建新方案 |

## 12. 与其他闭环的关系

- 患者管理负责把医生带到正确患者
- 患者 360 负责把证据、方案、预警、随访和建议聚起来
- 它本身不是深度规则模块，但承担主导航作用

## 13. 一句话总结

**患者管理是医生端的入口，患者 360 是医生端的主战场。**
