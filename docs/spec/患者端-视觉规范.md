# 患者端-视觉规范

> 依据：现有小程序 PRD、患者端首页/方案页/记录页 SPEC、记录模块视觉设计稿。本文档用于统一患者微信小程序本期的视觉语言、页面层级和组件表达，优先服务前端落地与后续设计对齐。

## 1. 规范目标

本期视觉规范只解决三件事：

1. 让首页、方案、记录三大主页面看起来是同一套产品
2. 让患者在慢病管理场景下读得懂、找得到、点得准
3. 让设计与前端在颜色、间距、卡片、状态表达上有统一标准

本期不追求品牌化视觉包装，不做营销式大插画，不做过度装饰。

---

## 2. 视觉定位

患者端整体视觉建议定位为：

**轻医疗、轻管理、低压力、可持续使用**

设计关键词：

- 清晰
- 稳定
- 温和
- 克制
- 可读

不建议走这些方向：

- 过深色、压迫感强
- 大面积渐变、强装饰
- 高饱和多色并列
- 类社交产品的热闹提醒风格
- 类营销页的大 Banner、大图文块

慢病患者会高频、长期使用，小程序应该更像“管理工具 + 医护陪伴界面”，而不是活动页。

---

## 3. 页面总原则

### 3.1 页面结构原则

- 页面统一采用 `浅底色 + 白卡片` 结构
- 首屏优先展示当前最重要的信息，不做复杂 tab 套 tab
- 每个页面控制 1 个主目标，避免一个页面同时承担“理解 + 执行 + 历史回看”
- 信息顺序遵循：
  - 当前状态
  - 当前动作
  - 详细说明
  - 历史/补充信息

### 3.2 内容密度原则

- 首页：高信息密度，但每块卡片内部保持简洁
- 方案页：中等密度，强调层级和解释
- 记录页：中高密度，强调分组和扫描效率

### 3.3 情绪原则

- 异常提醒要清楚，但不要制造恐慌
- 正向完成态要明确，但不过度奖励化
- 文案偏医疗服务语气，不用夸张情绪词

---

## 4. 色彩规范

结合现有 `记录模块视觉设计稿`，本期建议沿用以下色板。

### 4.1 基础色

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 页面背景 | `#EEF2F6` | 整体外层底色 |
| 页面内容底 | `#F6F8FB` | 手机屏内主要背景 |
| 卡片白底 | `#FFFFFF` | 所有主卡片底色 |
| 主文本 | `#16202D` | 标题、主信息 |
| 次文本 | `#5D6A7A` | 解释文案 |
| 弱文本 | `#8D99A8` | 辅助标签、说明 |
| 分割线 | `#E7ECF2` | 卡片边界、列表分隔 |

### 4.2 品牌/功能色

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 主色蓝 | `#1F82FF` | 主按钮、主标签、选中态 |
| 蓝浅底 | `#E9F3FF` | 轻提示、选中背景 |
| 成功绿 | `#10B981` | 完成态、正常同步、正向结果 |
| 绿浅底 | `#E8F8F1` | 完成类标签背景 |
| 警示橙 | `#F59E0B` | 待处理、提醒、待准备 |
| 橙浅底 | `#FFF4DB` | 提醒类卡片背景 |
| 风险红 | `#EF4444` | 重要/紧急异常、错误提示 |
| 红浅底 | `#FEECEC` | 异常类标签背景 |
| 辅助紫 | `#7C6DF0` | 睡眠、量表等次级模块识别色 |
| 紫浅底 | `#F0EFFF` | 睡眠/问卷弱背景 |

### 4.3 色彩使用规则

- 一个页面内主导色以蓝色为主，不要同时让红、橙、蓝、紫大面积竞争
- 红色只用于异常、错误、重要风险，不做普通强调色
- 绿色只用于“已完成 / 稳定 / 已同步”这类正向状态
- 紫色优先保留给睡眠、量表、报告类模块，形成长期识别

## 5. 字体规范

微信小程序端默认采用系统字体即可：

`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", Arial, sans-serif`

### 5.1 字号层级

| 层级 | 字号 | 字重 | 用途 |
| --- | --- | --- | --- |
| 页面主标题 | 22px | 700-800 | 首页、方案、记录页标题 |
| 模块标题 | 18px | 700 | 卡片标题、区块标题 |
| 卡片重点值 | 18-20px | 700-800 | 健康分、数值、倒计时 |
| 正文主文本 | 16px | 500-600 | 卡片主内容、表单输入 |
| 正文辅助 | 14px | 400-500 | 解释说明、时间、摘要 |
| 标签/状态 | 12-13px | 600-700 | badge、状态、胶囊标签 |
| 极弱辅助 | 11-12px | 500 | 次级说明、脚注 |

### 5.2 使用规则

- 患者端正文不要小于 `14px`
- 卡片内部若信息多，优先分行，不要压缩到难读
- 不使用负字距
- 重要数值与单位之间保持自然间距，单位可稍弱但不可过小

---

## 6. 间距与圆角规范

### 6.1 间距

建议统一 4 的倍数系统：

- 4
- 8
- 12
- 16
- 20
- 24

推荐落地：

| 场景 | 间距 |
| --- | --- |
| 页面左右边距 | 16px |
| 卡片内边距 | 16px |
| 卡片与卡片间距 | 12px |
| 模块标题与内容间距 | 10-12px |
| 表单字段垂直间距 | 12px |
| 页面大区块间距 | 20-24px |

### 6.2 圆角

| 元素 | 圆角 |
| --- | --- |
| 主卡片 | 14px |
| 次卡片/列表项 | 12px |
| 输入框 | 10-12px |
| 标签胶囊 | 999px |
| 弹层/底部抽屉 | 20px 顶部圆角 |

建议继续沿用现有设计稿里偏柔和但不夸张的圆角风格。

---

## 7. 阴影与边框

### 7.1 卡片表现

- 默认优先用 `浅边框 + 极轻阴影`
- 不建议厚重悬浮阴影

建议：

- 边框：`1px solid #E7ECF2`
- 阴影：`0 1px 0 rgba(20, 36, 54, .02)` 或极轻投影

### 7.2 特殊卡片

- 顶部状态卡、重要预警卡可增加轻投影
- 仍保持克制，避免像营销 Banner

---

## 8. 组件规范

## 8.1 顶部状态卡

用于首页、方案页顶部的关键信息块。

### 结构

- 主标题
- 当前状态 / 阶段
- 医生信息
- 关键摘要
- 1 个主按钮 + 可选次按钮

### 视觉要求

- 可用主色蓝浅背景或白底蓝边
- 标题与状态层次分明
- 不放超过 2 个主操作

---

## 8.2 标准信息卡

用于：

- 医生建议
- 复测指标
- 随访安排
- 睡眠报告
- 症状记录摘要

### 结构

- 标题
- 摘要
- 辅助信息（时间、医生、状态）
- 底部主按钮

### 视觉要求

- 卡片主操作保持单一
- 同一卡片不要放 3 个以上并列按钮
- 摘要最多 2 行，超出折叠

---

## 8.3 状态标签

状态统一用小 badge，不用大红条。

### 推荐状态色

| 状态 | 文字色 | 背景色 |
| --- | --- | --- |
| 未读 | `#1F82FF` | `#E9F3FF` |
| 待完成 | `#F59E0B` | `#FFF4DB` |
| 已完成 | `#10B981` | `#E8F8F1` |
| 已读 | `#627084` | `#EDF2F8` |
| 已到期 | `#EF4444` | `#FEECEC` |
| 待准备 | `#F59E0B` | `#FFF4DB` |
| 已准备 | `#10B981` | `#E8F8F1` |

### 使用规则

- 状态标签用于补充，不替代标题
- 一个卡片最多显示 1-2 个状态标签

---

## 8.4 按钮规范

### 主按钮

- 背景：`#1F82FF`
- 文字：白色
- 高度：44px
- 圆角：12px

适用：

- 去记录
- 查看准备项
- 我已知晓，开始执行
- 发送/提交

### 次按钮

- 白底 / 浅灰底
- 边框 `#E7ECF2`
- 文字 `#425063`

适用：

- 查看详情
- 稍后处理
- 取消

### 风险按钮

- 仅在确认性风险动作中使用红色线索
- 患者端尽量不用红底按钮作为常规入口

---

## 8.5 输入框与表单

适用：

- 报告上传说明
- 问卷填写
- 反馈问题
- 记录录入

### 表现

- 白底
- 1px 浅边框
- 高度 44px（单行）
- 多行输入最小高度 88px

### 规则

- 标签在上，输入区在下
- 错误提示放字段下方，用红色文字，不要只用 toast

---

## 8.6 锚点与胶囊导航

适用：

- 记录页锚点
- 次级信息切换

### 样式建议

- 未选中：浅灰底、深灰字
- 选中：主色蓝底、白字
- 高度：34px
- 左右内边距：14px
- 圆角：18px

---

## 9. 三大主页面视觉规则

## 9.1 首页

### 视觉目标

- 让患者一进来就知道今天最重要的事
- 重点是“执行”和“提醒”

### 首页首屏优先级

1. 顶部状态卡
2. 风险预警 / 医生最新建议
3. 今日待办
4. 快捷记录
5. 随访计划

### 视觉要求

- 首屏模块数量控制在 3-4 块内
- 风险卡和建议卡风格可区分，但不要太花
- 快捷记录用规则宫格，不要做成大插图入口

---

## 9.2 方案页

### 视觉目标

- 让患者看懂当前长期管理方案
- 承接最近新增动作和随访安排

### 待确认态

首屏应更完整，重点是“理解与确认”：

1. 当前方案头卡
2. 当前阶段目标
3. 方案明细
4. 重要提醒
5. 底部固定确认按钮

### 执行态

首屏重点切到“最近安排”：

1. 当前方案头卡
2. 最近安排
3. 随访安排
4. 今日任务轻入口

### 视觉要求

- 方案明细区采用分组卡或折叠组
- 最近安排卡优先突出主动作按钮
- 不把今日任务再做成首页式大列表

---

## 9.3 记录页

### 视觉目标

- 提供高效补录和历史回看能力

### 结构要求

- 日期栏固定清晰
- 锚点吸顶
- 分区卡片连续展示

### 视觉要求

- 记录页要比首页更“工具化”
- 数值、时间、标签对齐清楚
- 睡眠卡允许信息更丰富，但要分层展示

---

## 10. 状态与反馈规范

## 10.1 页面状态

必须统一支持：

- 空状态
- 加载中
- 成功反馈
- 失败反馈

### 空状态文案风格

- 不悲观
- 不技术化
- 给下一步建议

示例：

- `今日暂无预警`
- `今天的任务已经完成`
- `当前还没有随访安排`

## 10.2 完成反馈

- 普通完成用 toast + 卡片状态变化
- 不用全屏成功页
- 复测、上传、问卷完成后优先回到原上下文

## 10.3 错误反馈

- 表单错误：字段下提示
- 操作失败：toast
- 网络异常：页面内轻提示 + 重试

---

## 11. 图标规范

建议使用统一线性图标风格，避免混用拟物和填充图标。

推荐图标适用场景：

- 血糖
- 血压
- 血氧
- 睡眠
- 用药
- 随访
- 报告上传
- 问卷填写
- 风险提醒

规则：

- 同一级入口图标尺寸统一
- 不要给每张卡都配复杂彩色插图
- 图标是辅助识别，不代替文字

---

## 12. 文案展示规范

### 12.1 患者端表达原则

- 避免专业术语堆砌
- 避免“规则”“阈值”“版本”等后台语言
- 尽量说“要做什么”，少说“系统如何判断”

### 12.2 文案长度建议

| 场景 | 建议长度 |
| --- | --- |
| 卡片标题 | 6-12 字 |
| 卡片摘要 | 1-2 行 |
| 患者说明 | 30-80 字 |
| 阶段说明 | 3-5 条短句 |

---

## 13. 页面间分工原则

为避免页面内容打架，视觉设计要同步体现页面职责。

### 首页

更强调：

- 风险
- 待办
- 快速执行

### 方案页

更强调：

- 当前长期方案
- 最近安排
- 随访准备

### 记录页

更强调：

- 工具性
- 录入效率
- 历史回看

视觉上不能把三页做成完全同构，否则患者会觉得“到处都差不多”。

---

## 14. 本期建议避免的问题

- 首页、方案页都大块展示同样的今日任务列表
- 方案页只有摘要，没有方案明细
- 颜色过多导致重点不清
- 风险提醒过红、过满，造成心理压力
- 记录页像表格，不像移动端工具
- 所有准备项都做成任务卡，页面爆炸
- 一个卡片放多个平级主按钮

---

## 15. 一句话总结

**这套小程序视觉规范应以“浅底白卡、蓝色主导、分组清晰、状态克制、长期可用”为核心，做成一套稳定的慢病管理工具，而不是营销页或复杂后台的患者投影。**
