# 患者端-方案页SPEC

> 来源：主 PRD《管理方案闭环PRD》《执行任务闭环PRD》、患者端总览 SPEC 与旧版小程序 PRD 方案/今日待办章节。本文档聚焦患者端方案页的结构、任务承接与确认态。

## 1. 本期目标

P0 必须跑通：

```text
方案页
  -> 查看当前方案
  -> 确认已知晓
  -> 承接今日任务
  -> 查看随访计划
```

## 2. 页面定位

方案页不是给患者研究后台模块结构的，而是：

- 看当前阶段要做什么
- 完成今日任务
- 查看随访安排
- 接收最近调整和提醒

## 3. 页面结构

建议使用 3 个分区或 Tab：

- 今日任务
- 方案说明
- 随访计划

## 4. 待确认态

医生下发方案后，患者需先在方案页完成确认。

### 4.1 展示内容

- 主责医生信息
- 方案范围：适用疾病、方案周期、计划生效时间
- 管理目标
- 本阶段需要完成的主要事项
- 重要提醒

### 4.2 操作

- 我已知晓，开始执行
- 我有疑问

当前不提供“无法完成”作为通用操作。

## 5. 今日任务区

### 5.1 展示字段

- 任务名称
- 建议时间 / 截止时间
- 一句话说明
- 状态：待完成 / 已逾期 / 已完成
- 主按钮：去记录 / 去打卡 / 去同步 / 去查看

### 5.2 来源

- 管理方案
- 随访准备
- 风险预警
- 医生建议

### 5.3 交互原则

- 同一时间段任务尽量合并展示
- 患者只看到执行动作，不展示后台模块名
- 任务逾期后仍允许补录完成

## 6. 方案说明区

展示：

- 当前方案名称
- 适用疾病
- 当前阶段目标
- 方案周期
- 异常时怎么做

不展示：

- 医生内部备注
- 复杂规则阈值
- 方案版本 diff

## 7. 随访计划区

展示：

- 下次随访时间
- 倒计时
- 准备材料
- 最近一次随访结论摘要

## 8. 页面跳转

| 入口 | 去向 |
| --- | --- |
| 今日任务卡 | 具体执行页 |
| 风险任务 | 预警处理或复测页 |
| 睡眠任务 | 睡眠报告页 |
| 设备任务 | 设备管理页 |

## 9. 一句话总结

**方案页本质上是患者执行中枢，不是配置查看器。**
