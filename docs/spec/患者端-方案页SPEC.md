# 患者端-方案页SPEC

> 来源：主 PRD《管理方案闭环PRD》《执行任务闭环PRD》、医生端《管理方案SPEC》与患者端首页重构结论。本文档聚焦患者端底部 `方案` Tab 的定位、页面结构、展示字段和交互边界。

## 1. 页面定位

方案页不是首页，也不是今日任务页。

**方案页 = 医生确认后的长期管理依据页。**

它回答：

- 医生现在让我长期怎么管理？
- 本阶段目标是什么？
- 我需要长期配合哪些记录、用药、设备和随访？
- 最近医生是否调整过当前方案？

它不回答：

- 今天先做什么。
- 当前有没有异常。
- 快捷记录入口。
- 完整数据趋势洞察。
- 全部今日任务。

## 2. 与底部 Tab 分工

| Tab | 定位 |
| --- | --- |
| 首页 | 今天怎么做、当前状态如何、风险和任务优先处理 |
| 记录 | 数据记录、历史记录、指标详情和趋势洞察 |
| 方案 | 医生确认后的长期管理依据 |
| 我的 | 账户、档案、医生、设备、设置 |

一句话原则：

```text
首页负责今天怎么做。
方案页负责医生为什么这样安排，以及长期怎么管理。
记录页负责数据和趋势。
```

## 3. 页面状态

| 状态 | 触发条件 | 页面重点 |
| --- | --- | --- |
| 无方案 | 没有医生确认并下发的生效方案 | 告知医生下发后可查看方案 |
| 待确认 | 医生已下发新方案，患者未确认 | 完整展示方案内容，固定确认操作 |
| 执行中 | 患者已确认，方案生效 | 展示当前方案依据、目标、要求和随访安排 |
| 已结束/历史 | 方案被替换、停用或结束 | 进入历史方案页查看 |

## 4. 执行态页面结构

执行态按以下顺序展示：

```text
当前方案头卡
本阶段目标
医生确认的管理要求
指标目标
用药 / 治疗要求
设备要求
随访安排
最近方案调整
历史方案入口
```

执行态不展示：

- 今日状态。
- 今日待办列表。
- 快捷记录。
- 风险预警卡。
- 核心指标速览。
- 全部医生建议列表。
- 完整历史趋势图。

## 5. 当前方案头卡

展示内容：

```text
糖尿病 + 睡眠呼吸管理方案
控糖与夜间低氧改善期

状态：执行中
主责医生：林医生
生效时间：2026-05-20
方案周期：30 天
适用疾病：糖尿病 / 睡眠呼吸暂停 / 慢阻肺
```

字段：

| 字段 | 说明 |
| --- | --- |
| 方案名称 | 医生确认并下发的当前方案名称 |
| 阶段名称 | 当前管理阶段，例如控糖巩固期 |
| 状态 | 待确认 / 执行中 |
| 主责医生 | 当前方案负责人 |
| 生效时间 | 方案开始执行时间 |
| 方案周期 | 本阶段管理周期 |
| 适用疾病 | 当前方案覆盖的疾病或风险标签 |

不展示：

- 医生内部备注。
- 后台模板来源。
- 复杂版本 diff。
- 风险分和今日状态。

## 6. 本阶段目标

模块定位：解释本阶段医生希望患者达成什么。

展示示例：

```text
本阶段目标
- 稳定空腹血糖，减少餐后波动。
- 改善夜间低氧情况。
- 保持规律记录，便于下次随访评估。
```

展示规则：

- 展示 2-4 条患者可理解的目标。
- 使用健康管理语言，不使用诊断结论。
- 不堆叠复杂规则或模型解释。

## 7. 医生确认的管理要求

模块定位：解释这阶段患者长期需要配合什么。

展示示例：

```text
本阶段你需要配合：
- 按要求完成血糖和血压记录。
- 睡眠设备保持正常同步。
- 按当前用药安排打卡。
- 出现明显异常时及时复测或联系医生。
```

规则：

- 展示医生确认后的长期要求。
- 不展示后台模块名。
- 不展示今天是否完成，今日执行结果由首页和我的任务页承接。

## 8. 指标目标

模块定位：展示医生确认的个体化目标范围。

展示示例：

```text
血糖目标
- 空腹血糖：4.4-7.0 mmol/L
- 餐后 2 小时血糖：< 10.0 mmol/L

血压目标
- 家庭血压：< 140/90 mmHg

睡眠/血氧目标
- 夜间低氧减少
- 睡眠报告按医生要求同步
```

规则：

- 可展示目标范围和目标说明。
- 不展示复杂预警公式。
- 不展示“今日是否已完成”。
- 点击具体指标目标可跳转对应记录详情页或记录页，作为弱交互。

## 9. 用药 / 治疗要求

模块定位：展示医生确认的用药、治疗和执行要求。

展示示例：

```text
用药配合
- 二甲双胍 0.5g，每日 2 次。
- 按医生要求完成服药打卡。
- 如有漏服，请记录实际情况和原因。
```

规则：

- 患者不能自行修改药品、剂量、频次。
- 患者只能记录实际执行情况和备注。
- 药品调整必须来自医生端。
- 用药打卡主执行入口在首页今日待办或用药页，不在方案页展开执行流。

## 10. 设备要求

模块定位：展示医生要求配合的设备或同步要求。

展示示例：

```text
设备配合
- 睡眠监测设备：保持正常同步。
- 血氧仪：按医生要求完成复测。
- 若设备连续未同步，请去设备页处理。
```

规则：

- 不展示设备专业参数。
- 不展开完整设备列表。
- 设备异常处理入口可跳转设备页。
- 设备同步异常如果需要当日处理，应进入首页风险预警或今日待办。

## 11. 随访安排

模块定位：展示当前方案内的随访计划和准备要求，比首页随访提醒更完整。

展示示例：

```text
下次随访
时间：5月28日 10:00
方式：电话随访
医生：林医生

随访前请准备：
- 近 7 天血糖记录。
- 近 7 天血压记录。
- 睡眠报告。
- 用药执行情况。
```

规则：

- 展示下次随访时间、方式、医生和准备项。
- 可提供 `查看全部随访`。
- 随访准备任务仍进入首页今日待办和我的任务页。
- 方案页只解释随访安排，不替代随访详情页。

## 12. 最近方案调整

模块定位：展示当前方案最近发生的正式调整，不承接普通医生建议。

展示示例：

```text
最近方案调整
2026-05-25
林医生更新了本阶段管理要求：
- 增加晚餐后 2 小时血糖记录。
- 下次随访前补齐睡眠报告。
```

规则：

- 只展示与当前方案变更相关的内容。
- 普通医生建议进入 `我的 -> 医生建议`。
- 复测、临时任务进入首页今日待办和我的任务页。
- 临时随访进入随访安排或随访详情页。

## 13. 历史方案入口

模块定位：弱入口，用于查看已结束、已停用或被替换的旧方案。

展示示例：

```text
历史方案
查看已结束或已替换的方案
```

跳转：

```text
/pages/plan/history/index
```

## 14. 待确认态

待确认态在医生下发新方案后展示，患者确认前必须看到比执行态更完整的方案内容。

页面结构：

```text
当前方案头卡
本阶段目标
医生确认的管理要求
指标目标
用药 / 治疗要求
设备要求
随访安排
底部固定操作
```

底部操作：

```text
主按钮：我已知晓，开始执行
次按钮：我有疑问
```

交互规则：

- 点击 `我已知晓，开始执行` 后，方案状态转为执行中。
- 点击 `我有疑问` 后，弹出轻量文本反馈框。
- 反馈提交后进入医生端方案反馈或医患沟通记录。
- 当前不提供“无法执行”作为通用操作。

待确认态不展示：

- 今日状态。
- 今日待办。
- 快捷记录。
- 风险预警。
- 核心指标速览。

## 15. 空状态

无医生确认并下发的生效方案时展示：

```text
当前还没有生效中的管理方案
医生确认并下发后，你可以在这里查看当前方案和长期管理要求。
```

可选操作：

- 去完成健康筛查。
- 去绑定医生。
- 返回首页。

## 16. 跳转关系

| 模块 | 去向 |
| --- | --- |
| 指标目标 | 对应指标详情页或记录页 |
| 用药要求 | 用药详情页 |
| 设备要求 | 设备页 |
| 随访安排 | 随访详情页或随访记录页 |
| 最近方案调整 | 当前方案调整记录详情，P0 可先不做独立页 |
| 历史方案 | `/pages/plan/history/index` |
| 待确认主按钮 | 当前页内状态变更 |
| 我有疑问 | 方案反馈弹窗 |

## 17. 与首页的边界

首页展示：

- 今日状态。
- 核心指标速览。
- 风险预警。
- 今日待办。
- 快捷记录。
- 医生新安排。
- 下次随访轻提醒。
- 当前方案轻摘要。

方案页展示：

- 当前方案。
- 本阶段目标。
- 长期管理要求。
- 指标目标。
- 用药 / 治疗要求。
- 设备要求。
- 随访安排。
- 最近方案调整。
- 历史方案入口。

方案页不重复首页的大任务流，不做快捷记录入口，不做风险预警主处理入口。

## 18. 验收标准

- 无方案状态能明确提示医生下发后查看。
- 待确认态展示完整方案内容，并支持“我已知晓，开始执行”和“我有疑问”。
- 执行态默认展示当前方案头卡、本阶段目标、管理要求、指标目标、用药/治疗要求、设备要求、随访安排、最近方案调整和历史方案入口。
- 执行态不展示今日状态、今日待办列表、快捷记录、风险预警卡和核心指标速览。
- 患者不能在方案页修改医生确认的目标、用药、治疗执行任务和随访计划。
- 随访准备任务、复测任务和今日任务应回到首页或我的任务页执行。

## 19. 一句话总结

**患者端方案页是医生确认后的长期管理依据页，不是今日执行页，也不是数据洞察页。**
