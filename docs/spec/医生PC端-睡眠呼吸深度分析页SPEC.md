# 医生PC端-睡眠呼吸深度分析页SPEC

> 来源：医生端 SPEC 总览、患者管理 SPEC、设备与报告 SPEC、本轮睡眠呼吸分析讨论与用户提供设备清单《提供的设备清单.xlsx》。本文档聚焦医生 PC 端患者详情中的睡眠报告承接、睡眠呼吸深度分析页面布局、数据指标、图表、图标与交互落地边界。

## 1. 页面定位

### 1.1 页面名称

患者详情 Tab 名称统一为：

```text
睡眠深度分析
```

页面内容标题建议使用：

```text
睡眠呼吸深度分析
```

### 1.2 页面目标

本页面不是泛睡眠健康大盘，也不是单份报告 PDF 预览页。

医生进入后应能完成：

1. 判断患者最近睡眠呼吸风险是否异常。
2. 判断夜间低氧、氧减和呼吸事件是否持续或恶化。
3. 查看最近一次睡眠报告的关键证据。
4. 查看多晚趋势，服务慢病随访和方案复盘。
5. 从异常数据继续发起复测、随访和医生建议。

### 1.3 P0 设计结论

- 睡眠报告先承接在患者详情内，不先建设独立一级导航。
- 患者详情原 `数据分析` Tab P0 调整为 `睡眠深度分析`。
- 仅当患者疾病标签中存在 `睡眠呼吸障碍` 时展示该 Tab。
- P0 先按当前设备清单已明确能力设计，不等待未确认指标。
- 对不同设备能力做模块显隐，不展示伪空态。

## 2. 展示条件与入口

### 2.1 Tab 展示规则

| 条件 | 是否展示 `睡眠深度分析` Tab | 说明 |
| --- | --- | --- |
| 患者医生确认疾病标签含 `睡眠呼吸障碍` | 展示 | P0 主条件 |
| 患者疾病风险仅含睡眠风险待确认 | 暂不展示 | P1 可评估在健康筛查或报告入口承接 |
| 患者仅有睡眠报告，但无睡眠呼吸障碍疾病标签 | 暂不展示 | 报告仍可从设备与报告区域查看 |
| 患者无睡眠相关疾病和报告 | 不展示 | 避免患者详情 Tab 噪音 |

### 2.2 疾病标签兼容规则

历史数据中可能存在 `睡眠呼吸暂停` 文案，前端和后端判断展示条件时需归一：

| 原始标签 | 归一展示 |
| --- | --- |
| 睡眠呼吸障碍 | 睡眠呼吸障碍 |
| 睡眠呼吸暂停 | 睡眠呼吸障碍 |
| OSA | 睡眠呼吸障碍 |

P0 页面展示统一使用 `睡眠呼吸障碍`。

### 2.3 入口

| 入口 | 路径 | P0 |
| --- | --- | --- |
| 患者详情 Tab | `患者管理 > 患者详情 > 睡眠深度分析` | 必做 |
| 患者详情总览睡眠卡 | `患者详情 > 总览 > 睡眠摘要 > 查看深度分析` | 建议 |
| 睡眠报告卡 | `设备/报告摘要 > 查看睡眠深度分析` | 建议 |
| 睡眠预警 | `预警证据 > 查看患者睡眠分析` | P1 |

## 3. 当前设备能力边界

### 3.1 当前设备清单

| 设备 | 型号 | 清单中已明确指标 | 本页 P0 定位 |
| --- | --- | --- | --- |
| 医用脉搏血氧仪 | `ZG-P11H / ZG-P11G` | 血氧饱和度、脉率、睡眠呼吸事件、ODI | 夜间低氧与睡眠呼吸风险观察 |
| 睡眠呼吸监测系统 | `ZG-M11A / ZG-M11B` | 呼吸暂停、低通气、体动、血氧、心率、睡眠分期；示例包含 AHI | 睡眠呼吸事件与报告分析 |

### 3.2 设备能力适配

#### P11 指环血氧

P0 可展示：

- 夜间血氧摘要
- 脉率摘要
- ODI
- 睡眠呼吸事件摘要
- 历史睡眠血氧报告

P0 不强制展示：

- AHI
- 呼吸暂停/低通气明细
- 睡眠分期
- 体位与体动分析

#### M11 睡眠呼吸监测系统

P0 可展示：

- AHI
- 呼吸暂停
- 低通气
- 血氧
- 心率
- 体动
- 睡眠分期
- 历史睡眠呼吸监测报告

### 3.3 模块显隐原则

| 数据能力 | 有数据时 | 无数据时 |
| --- | --- | --- |
| AHI | 展示 AHI 指标卡和趋势 | 模块隐藏，不展示 `暂无 AHI` |
| ODI | 展示氧减负担 | 若无 ODI，氧减模块降级为血氧摘要 |
| 连续血氧曲线 | 展示单夜曲线 | 展示报告摘要与关键值 |
| 睡眠分期 | 展示睡眠结构区 | 模块隐藏 |
| 体动 | 展示体动摘要 | 模块隐藏 |

**展示条件**：患者命中至少一个支持病种（睡眠呼吸障碍 / 慢阻肺 / 2 型糖尿病）时展示，置于页面最顶部。

**模块定位**：本模块只作为跳转入口，不在睡眠深度分析页复刻健康分、共病链路、推演摘要或管理建议。

**展示字段**：

| 字段 | 展示内容 | 规则 |
|------|----------|------|
| 入口标题 | 数字孪生融合分析 | 固定文案 |
| 入口说明 | 查看综合健康分、共病影响、管理建议和推荐依据 | 固定文案 |
| 操作按钮 | 进入数字孪生融合分析 | 点击跳转患者详情页「数字孪生融合分析」Tab |

入口横幅使用轻量提示样式。横幅渲染失败时不阻断本页正常展示。

### 5.2 当前睡眠风险摘要

#### 展示目标

用一段医生可扫读的结论回答：

- 当前是否异常
- 异常证据是什么
- 数据是否足够支撑趋势判断

#### 摘要模板

| 场景 | 摘要 |
| --- | --- |
| M11 异常 | 最近一次睡眠呼吸监测提示呼吸事件负担与夜间低氧异常，AHI、ODI 和最低血氧需结合症状与随访继续评估。 |
| P11 异常 | 最近夜间血氧监测提示氧减风险，ODI 与最低血氧异常，建议结合睡眠呼吸症状和后续报告复核。 |
| 数据不足 | 当前有效睡眠报告不足，暂不输出多晚趋势结论。 |

#### 视觉层级

- 摘要卡采用浅医疗蓝背景，不使用大面积红底。
- 严重风险仅在结论标签和关键异常指标上使用红色。
- 数据可信度提示在摘要底部使用次级提示条。

### 5.3 核心睡眠指标卡

#### M11 指标卡

| 卡片 | 主指标 | 辅助信息 |
| --- | --- | --- |
| 呼吸事件负担 | AHI | 较上周期变化，若有 |
| 氧减负担 | ODI | 夜间氧减风险 |
| 最低血氧 | 最低 SpO2 | 异常等级 |
| 呼吸事件 | 呼吸暂停 + 低通气 | 事件数量 |
| 报告状态 | 最新报告有效性 | 设备来源、监测时间 |

#### P11 指标卡

| 卡片 | 主指标 | 辅助信息 |
| --- | --- | --- |
| 氧减负担 | ODI | 较上次变化，若有 |
| 最低血氧 | 最低 SpO2 | 夜间低氧提示 |
| 睡眠呼吸事件 | 事件摘要 | 设备报告输出 |
| 报告状态 | 最新报告有效性 | 设备来源、监测时间 |

#### 指标卡视觉要求

- 卡片半径沿医生 PC 视觉规范，使用 4px 到 8px。
- 主数字大于标签，但不能做运营大屏式夸张数字。
- 指标卡上方可使用细色条区分：
  - 呼吸事件：医疗蓝
  - 血氧异常：青蓝
  - 风险异常：功能橙/红
  - 报告质量：中性色
- 卡片图标仅作识别，不抢主数字层级。

### 5.4 多晚趋势分析

#### 时间窗

| 时间窗 | P0 | 使用场景 |
| --- | --- | --- |
| 最近 1 晚 | 必做 | 预警或单报告分析 |
| 最近 7 晚 | 必做，默认 | 短期趋势 |
| 最近 14 晚 | 必做 | 随访前后观察 |
| 最近 30 晚 | 必做 | 慢病阶段复盘 |
| 最近 90 天 | P1 | 长期管理 |
| 自定义 | P1 | 专项复盘 |

#### 时间窗交互规则

- 时间窗控件必须真实筛选当前页全部趋势、有效报告数和默认报告选择。
- 切换 `最近 1 晚` 时，仅展示最新一份报告，当前报告详情同步为最新报告。
- 切换到 `最近 7 晚 / 14 晚 / 30 晚` 时，若当前选中报告仍在该时间窗内，则保持选中；若不在，则自动选中该时间窗内最新报告。
- 有效报告数格式为 `N/时间窗晚数`，例如 `5/7 晚有效`。
- 时间窗不应只改变按钮选中态，必须改变趋势图、报告列表范围或当前报告上下文。

#### 指标切换

| 设备能力 | 趋势指标 |
| --- | --- |
| P11 | ODI、最低血氧、睡眠呼吸事件 |
| M11 | AHI、ODI、最低血氧、呼吸暂停/低通气事件、体动、鼾声 |

#### 指标切换交互规则

- 多晚趋势分析右上方必须支持趋势指标切换。
- P0 至少支持 `AHI / ODI / 最低血氧 / 体动 / 鼾声` 的有数据项切换。
- 若当前患者报告中无某指标，隐藏该指标切换项。
- 切换指标后，图表曲线、阈值线、点位数值和图例必须同步变化。
- 点击某个趋势点或下方日期点后，当前报告详情切换到该晚。

#### 趋势图规则

- 横轴为夜晚日期，不使用过密小时刻度。
- 使用折线图作为主图，保留异常阈值线，不用柱状图替代趋势判断。
- 缺失夜显示断点和缺失提示，不用 0 值补齐。
- 异常点高亮，可显示报告摘要 Tooltip。
- 点击某一晚后联动下方 `当前报告详情` 为该晚详情。
- 趋势图右上角展示当前有效晚数，例如 `5/7 晚有效`。

### 5.5 当前报告详情

#### 布局

```text
左 2/3：单夜图表或报告摘要
右 1/3：结构化指标摘要
```

#### 左侧图表

有连续曲线时，优先展示：

1. SpO2 夜间趋势。
2. 睡眠呼吸事件标记。
3. 体动轨道。
4. 鼾声轨道。
5. 脉率/心率辅助轨道，P1 可补。
6. 睡眠分期时间带，若 M11 输出。

#### 单夜多轨图表规则

- 默认展示当前选中报告生成那晚的数据，而不是永远展示最新报告。
- `SpO2` 轨道使用折线图，并保留 `90%` 低氧参考线。
- `呼吸事件 / 体动 / 鼾声` 轨道使用事件密度条或强度条。
- 血氧设备未输出体动/鼾声时，对应轨道展示“当前设备未输出”，不伪造数据。
- 图表右侧必须有数值摘要，便于医生不依赖视觉估算。

无连续曲线时，展示：

- 报告摘要图卡。
- 关键指标值。
- 当前报告结论。
- 原始报告查看入口。

#### 右侧摘要

| 分组 | 指标 |
| --- | --- |
| 呼吸事件 | AHI、呼吸暂停、低通气、睡眠呼吸事件 |
| 夜间血氧 | 最低血氧、ODI、夜间血氧异常摘要 |
| 体动鼾声 | 体动段数、体动指数、鼾声摘要、鼾声占比 |
| 睡眠结构 | 睡眠分期、报告监测时长 |
| 数据来源 | 设备型号、报告时间、报告类型 |

### 5.7 历史报告列表页

历史报告列表页用于医生从多份睡眠报告中快速定位某一晚，并进入该晚完整详情。

P0 可以先以患者详情内弹窗或右侧抽屉承载；后续若报告数量增多，可升级为独立二级页。

#### 入口

| 入口 | 说明 |
| --- | --- |
| 睡眠深度分析顶部 `历史报告列表` | 打开完整报告列表 |
| 页面底部历史报告卡片 `查看全部` | 打开完整报告列表 |
| 多晚趋势点位 | 直接切换当前报告详情 |

#### 列表字段

| 字段 | 说明 |
| --- | --- |
| 监测日期 | 报告对应夜晚 |
| 报告类型 | 睡眠血氧报告 / 睡眠呼吸监测报告 |
| 设备型号 | P11 / M11 |
| AHI | 有能力时展示 |
| ODI | 有能力时展示 |
| 最低血氧 | 有能力时展示 |
| 事件摘要 | 呼吸事件或睡眠呼吸事件 |
| 报告状态 | 有效、异常、待复核 |
| 操作 | 查看报告、进入该晚详情 |

#### 点击行为

- 点击历史报告列表任意一行后，关闭列表页或抽屉。
- 患者详情停留在 `睡眠深度分析` Tab。
- 当前报告详情切换为该报告生成那晚的数据。
- 多晚趋势中的对应夜晚高亮。
- 顶部 `当前报告时间 / 数据来源` 同步更新。
- 若该报告不在当前时间窗内，系统可自动切换到覆盖该报告的最小时间窗，或提示医生切换时间窗；P0 优先保持当前时间窗内报告。

## 6. 图表 SPEC

### 6.1 图表基调

图表应符合医疗工作台场景：

- 清晰、克制、证据优先。
- 不使用营销型霓虹渐变、发光曲线和夸张面积图。
- 不使用会误导临床判断的平滑插值覆盖真实缺失段。
- 异常高亮必须保留原始数值和时间上下文。

### 6.2 推荐图表类型

| 场景 | 推荐图表 |
| --- | --- |
| 多晚趋势 | 折线图 + 异常点标记 |
| 单夜 SpO2 | 时间序列折线图 |
| 呼吸事件 | 时间轴事件条 |
| 睡眠分期 | 分期时间带 |
| 体动 | 事件条或密度条 |
| 鼾声 | 强度条 / 占比条 / 分时段密度条 |
| 指标摘要 | 数值卡 + 小型 Sparkline |

### 6.3 图表配色

| 数据 | 色彩建议 |
| --- | --- |
| SpO2 | 青蓝或医疗蓝 |
| 脉率/心率 | 青绿 |
| AHI/呼吸事件 | 医疗蓝 |
| ODI/氧减 | 青色 |
| 低氧异常 | 橙或红，仅用于异常 |
| 体动 | 医疗绿 |
| 鼾声 | 克制紫，不做娱乐化表达 |
| 缺失数据 | 中性灰 |

### 6.4 图表交互

- Tooltip 必须展示时间、数值、数据来源。
- 异常点 Tooltip 可附报告摘要，但不直接输出诊断结论。
- 单夜图表若支持缩放，P0 仅需 `全夜` 与 `局部查看`。
- 图表旁必须保留数值摘要，避免医生只能依赖视觉估算。

## 7. 图标 SPEC

### 7.1 图标风格

医生 PC 端睡眠分析图标应采用：

- 线性图标为主。
- 统一 1.5px 到 2px 线宽。
- 圆角端点，减少消费级可爱感。
- 优先使用现有高质量图标库，推荐 Lucide 或项目已接入医疗图标体系。
- 不手绘复杂插画型 SVG。
- 图标只承担识别和分组，不替代医学术语。

### 7.2 推荐图标语义

| 模块 | 图标语义 | 说明 |
| --- | --- | --- |
| 睡眠分析 | 月牙 + 呼吸/波形 | 不使用卡通睡脸 |
| 血氧 | 氧气/波形/血氧探头语义 | 避免心形占主语义 |
| 呼吸事件 | 肺部/气流波形 | 用于 AHI 与事件分组 |
| 氧减负担 | 下行趋势/氧减波形 | 用于 ODI/低氧 |
| 报告 | 文档/夹板 | 用于原始报告 |
| 数据质量 | Shield Check 或 Signal | 用于可信度 |
| 随访动作 | Calendar/Clipboard | 与随访模块一致 |

### 7.3 图标使用位置

| 位置 | 使用要求 |
| --- | --- |
| Tab | 可用简洁睡眠呼吸图标，尺寸 16px |
| 指标卡 | 左上角图标，尺寸 18px 到 20px |
| 图表图例 | 优先用色点和文本，不堆图标 |
| 操作按钮 | 只在高频按钮使用图标 + 文本 |
| 报告状态 | 使用状态图标时必须有文字标签 |

### 7.4 禁止项

- 不使用装饰性月亮星星插画作为数据背景。
- 不用过度具象的患者睡姿图片替代图表。
- 不用大量医疗十字图标覆盖不同指标。
- 不用仅靠颜色表达异常，必须有文字或数值。

## 9. 空态与异常态

### 9.1 页面级空态

| 场景 | 文案 |
| --- | --- |
| 已有睡眠呼吸障碍标签但无报告 | 当前暂无睡眠报告，可建议患者完成睡眠相关监测。 |
| 报告正在同步 | 睡眠报告同步中，稍后刷新查看。 |
| 报告异常 | 当前报告存在异常或质量待复核，请优先查看原始报告。 |

### 9.2 模块级空态

- 设备不支持的模块直接隐藏。
- 设备支持但暂无数据时展示空态。
- 多晚趋势不足时提示 `有效报告不足，暂无法形成趋势`。

## 11. P0 与 P1 边界

### 11.1 P0 必做

1. 患者详情 `睡眠深度分析` Tab 条件展示。
2. 当前睡眠风险摘要。
3. 核心睡眠指标卡。
4. 最近 1 晚、7 晚、14 晚、30 晚时间窗。
5. 多晚趋势图。
6. 当前报告单夜多轨详情。
7. 历史报告列表页与点击进入该晚详情。
8. 查看报告、复测、创建随访、发送医生建议。
9. P11/M11 能力显隐。

### 11.2 P1

1. 体位相关呼吸事件分析。
2. 多轨道图表缩放和事件定位。
3. CPAP 治疗执行与依从分析。
4. 睡眠数据与血压、血糖、慢阻肺症状的跨指标关联。
5. 数字孪生风险预测和干预模拟。

## 12. 验收标准

| 验收项 | 标准 |
| --- | --- |
| Tab 条件 | 仅睡眠呼吸障碍患者展示 |
| 名称 | 原 `数据分析` Tab P0 替换为 `睡眠深度分析` |
| 设备适配 | P11 不强行显示 AHI，M11 可显示呼吸事件分析 |
| 时间窗 | 最近 1 晚、7 晚、14 晚、30 晚切换真实生效 |
| 趋势指标 | AHI、ODI、最低血氧、体动、鼾声可按数据能力切换 |
| 图表 | 支持多晚趋势、单夜多轨分析与当前报告详情 |
| 历史报告 | 历史报告列表可打开，点击报告后进入该晚详情 |
| 医生动作 | 可以从页面发起报告查看、复测、随访、建议 |
| 医疗视觉 | 图表和图标克制、专业、数据优先 |

## 13. 一句话总结

**睡眠深度分析页解决的是“这个睡眠呼吸障碍患者最近睡眠呼吸风险和夜间低氧负担是否值得医生进一步处理”。**
