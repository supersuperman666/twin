# 患者端SPEC总览

版本：V0.1  
适用端：微信小程序  
文档定位：承接主 PRD 中患者端页面、路径、展示和交互规则

## 1. 患者端SPEC怎么用

患者端 SPEC 默认业务闭环以 [医患闭环主PRD](../prd/医患闭环主PRD.md) 为准。

这份文档更关注：

- 患者从哪里进入功能
- 每个页面承担什么任务
- 方案如何转成任务视角
- 医患沟通如何在患者端呈现

## 2. 患者端页面主线

当前建议按以下页面组维护患者端 SPEC：

1. 首页
2. 方案页
3. 记录页
4. 随访页
5. 设备页
6. 我的 / 档案
7. 医患沟通与消息

## 3. 患者端关键路径

### 3.1 主路径

```text
首页看提醒
  -> 进入方案页执行今日任务
  -> 进入记录页补录或查看历史
  -> 收到预警后完成复测或查看医生提醒
  -> 查看随访准备事项
```

### 3.2 方案页定位

方案页不是给患者“研究方案结构”的，而是：

- 看当前阶段要做什么
- 完成今日任务
- 查看随访安排
- 接收最近调整和提醒

### 3.3 记录页定位

记录页是通用记录工具页，不替代方案页。

- 方案页偏“按任务执行”
- 记录页偏“主动补录和查看历史”

## 4. 各页面职责

### 4.1 首页

目标：

- 让患者快速知道今天是否需要处理事情。

核心内容：

- 当前管理状态
- 今日待办
- 风险提示
- 最近医生提醒
- 快捷记录入口

### 4.2 方案页

目标：

- 作为患者执行方案的主入口。

建议分为三个分区或 Tab：

- 今日任务
- 方案说明
- 随访计划

当前页面原则：

- 方案页是“执行页”，不是“查看后台配置页”。
- 患者看到的是今天该做什么，不是模块结构。
- 方案调整后，患者只承接最新生效版本，不参与版本选择。

#### 今日任务

- 展示今天待完成任务
- 主按钮直接去执行

#### 方案说明

- 展示本阶段目标
- 展示需要患者配合的事项
- 展示异常时怎么做

#### 随访计划

- 展示下次随访时间
- 展示随访前要准备什么

### 4.3 记录页

目标：

- 承接主动记录、补录、查看历史趋势。

核心内容：

- 各指标快捷记录
- 历史记录入口
- 最近记录摘要

### 4.4 随访页

若后续独立成页，目标是：

- 集中承接随访提醒、准备事项和历史随访结论

当前也可先挂在方案页内。

### 4.5 设备页

目标：

- 让患者绑定、查看和排查设备同步问题。

核心内容：

- 已绑定设备
- 同步状态
- 异常提示
- 去同步/去处理入口

### 4.6 我的 / 档案

目标：

- 承接个人信息、健康档案、我的医生、历史医生建议、我的任务和设置类内容。

关联深度 SPEC：

- [健康档案页SPEC](./健康档案页SPEC.md)

### 4.7 医患沟通与消息

目标：

- 让沟通尽量结构化，而不是直接做重 IM。

建议形式：

- 医生提醒卡
- 结构化问题卡
- 患者回复摘要

当前页面原则：

- 不先做复杂聊天。
- 沟通入口可以先挂在方案页、风险提示和随访准备里。
- 每次沟通尽量围绕一个主题，避免长链路打扰患者。

## 5. 患者端任务展示说明

### 5.1 当前建议任务类型

- 指标记录
- 用药/治疗执行
- 症状评估
- 睡眠报告
- 生活方式
- 随访准备
- 复测任务
- 设备任务
- 确认知晓

### 5.2 任务卡展示字段

每张任务卡建议只展示：

- 任务名称
- 建议时间/截止时间
- 一句话说明
- 状态
- 主按钮

### 5.2A 今日待办定义

`今日待办` 指患者在当天需要主动完成、且完成结果会影响当前管理闭环的执行动作。

本期建议纳入：

- 当天应完成的计划内记录任务
- 当天应完成的复测任务
- 当天必须完成的随访准备动作（如上传检查报告、填写量表问卷）

本期默认不纳入：

- 医生建议
- 普通随访准备说明
- 长期方案明细
- 历史未完成但今天不需要继续处理的事项

### 5.3 当前建议任务状态

- `pending`
- `completed`
- `closed`

说明：

- `overdue` 只作为 `pending` 的逾期标签
- 当前不对患者开放“无法完成”

### 5.4 患者可操作内容

- 去完成
- 补录
- 查看详情
- 查看方案说明
- 查看随访安排
- 发起问题反馈或设备异常反馈

### 5.5 患者端不建议暴露的内容

- 风险等级判定规则
- 预警阈值细节
- 医生内部备注
- 方案版本差异详情
- 复杂任务状态机

## 6. 页面进入路径建议

结合当前小程序结构，建议统一如下：

| 入口 | 去向 | 说明 |
| --- | --- | --- |
| 首页今日待办 | 方案页-今日任务或具体执行页 | 高频主入口 |
| 首页风险提示 | 复测页/记录页/方案页 | 处理异常 |
| Tab 方案 | 方案页 | 主执行入口 |
| Tab 记录 | 记录页 | 主动补录入口 |
| 方案页任务卡 | 具体执行页 | 按任务执行 |
| 方案页随访区 | 随访内容 | 查看准备事项 |
| 我的-设备管理 | 设备页 | 设备处理 |

## 7. 患者端与主 PRD 的边界

应该写在主 PRD 的内容：

- 为什么这些模块会生成这些任务
- 方案、预警、随访如何串联
- 状态流转和版本规则

应该写在患者端 SPEC 的内容：

- 页面层级
- 进入路径
- 卡片展示内容
- 表单交互
- 结构化沟通呈现形式
