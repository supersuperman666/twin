# 医生PC端-医生建议SPEC

> 来源：主 PRD《医生建议与结构化沟通PRD》与旧版医生 PRD 医生建议章节。本文档聚焦医生端医生建议列表、发送、患者回执和结构化沟通承接。

## 1. 本期目标

P0 必须跑通：

```text
医生发送建议
  -> 患者接收
  -> 患者确认已读或执行
  -> 医生查看回执和执行状态
```

## 2. 页面清单

| 页面/弹层 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 医生建议列表 | 查看已发送建议 | 来源、状态、患者回执 | 查看详情 |
| 发送建议抽屉 | 创建建议 | 模板、文案、是否生成待办 | 发送 |
| 结构化沟通卡 | 承接患者回复 | 结构化问题与摘要 | 查看回复 |

## 3. 医生建议类型

- 复测建议
- 执行建议
- 就医/转诊建议
- 健康指导

## 4. 发送建议抽屉

### 4.1 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 建议类型 | select | 是 | 复测/执行/就医/健康指导 |
| 来源 | auto | 是 | 预警/随访/方案/手动 |
| 模板 | select | 否 | 选择后可编辑 |
| 标题 | input | 是 | 患者端显示 |
| 内容 | textarea | 是 | 简洁、可执行 |
| 是否生成待办 | switch | 是 | 默认关闭，按需开启 |
| 是否需要已读确认 | switch | 是 | 默认开启 |

### 4.2 交互原则

- 优先模板选择后编辑
- 内容必须患者可理解
- 不能混入医生内部推理过程

## 5. 医生建议列表

### 5.1 列表字段

| 字段 | 说明 |
| --- | --- |
| 患者 | 姓名、年龄 |
| 建议类型 | 复测/执行/就医/健康指导 |
| 来源 | 预警/随访/方案/手动 |
| 标题 | 患者端显示标题 |
| 发送时间 | 创建时间 |
| 状态 | 已发送、已读、已执行、已过期 |
| 是否生成待办 | 是/否 |

## 6. 结构化沟通

### 6.1 当前建议承接类型

- 数据异常沟通
- 执行依从性沟通
- 随访准备沟通

### 6.2 医生端需要看到

- 患者是否已读
- 是否已执行
- 患者结构化回复摘要

## 7. 与其他模块的跳转

| 来源/去向 | 行为 |
| --- | --- |
| 预警详情 -> 发送建议 | 自动带入预警来源 |
| 随访详情 -> 发送建议 | 自动带入随访来源 |
| 患者360 -> 发送建议 | 手动发送 |

## 8. 一句话总结

**医生建议页解决的是“我说了什么，患者看了没、做了没”。**
