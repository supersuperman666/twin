# 医生PC端-管理方案交互开发SPEC

> 来源：医生PC端PRD 15 管理方案模块。本文档承接管理方案页面交互、组件字段、状态按钮、下发校验、患者可见内容预览、前端 Mock Schema 与开发拆分。主 PRD 仅保留业务规则与范围说明。

## 管理方案交互开发 SPEC

本 SPEC 用于进入前端 Mock 闭环开发。目标不是一次性做完整模板后台，而是先把管理方案从“空草稿”升级为可查看、可编辑、可校验、可预览、可下发、可流转的生产级闭环。

### 1 本期开发目标

本期必须跑通以下闭环：

```text
方案列表
  -> 创建/查看方案草稿
  -> 进入方案详情编辑页
  -> 编辑必须模块和条件模块
  -> 保存草稿
  -> 查看患者可见内容预览
  -> 确认下发
  -> 方案进入“已下发待患者确认”
  -> 模拟患者确认已知晓
  -> 方案进入“执行中”
  -> Mock 生成患者端待办、计划内随访和时间轴事件
```

P0 暂不开发：

- 模板后台配置。
- 复杂规则引擎。
- 多医生协同编辑。
- 批量下发。
- 字段级版本 diff。
- 真实患者端页面联调。

### 2 页面与弹层清单

| 页面/弹层 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 方案管理列表页 | 集中处理待确认、执行中、待调整方案 | 状态筛选、患者信息、方案名称、来源、版本、关键模块摘要、患者确认状态 | 查看/编辑、确认下发、复制方案、创建随访 |
| 创建方案入口弹窗 | 避免医生进入空白页 | 患者、适用疾病、方案周期、创建方式 | 下一步进入方案编辑页 |
| 方案详情编辑页 | 医生审核和编辑方案的主页面 | 顶部患者条、证据摘要、模块卡片、审核面板 | 保存草稿、患者可见内容预览、确认下发、驳回推荐 |
| 模块编辑抽屉 | 编辑单个方案模块 | 动态字段、推荐值、医生修改原因 | 保存模块、取消、恢复系统推荐 |
| 患者可见内容预览弹窗 | 医生确认患者端展示内容 | 阶段目标、今日待完成、随访提醒、异常处理提示 | 返回编辑、确认下发 |
| 确认下发弹窗 | 下发前二次确认 | 校验结果、冲突提示、旧方案处理方式 | 确认下发、返回修改 |
| 驳回推荐弹窗 | 驳回系统草稿 | 驳回原因、医生备注 | 确认驳回 |

### 3 方案管理列表页 SPEC

顶部筛选：

- 方案状态：全部、待医生确认、已下发待患者确认、执行中、患者有疑问、患者反馈无法执行、待调整、已停用、已完成。
- 方案来源：全部、系统草稿、医生创建、模板创建、历史复制、随访调整、预警调整。
- 疾病：全部、糖尿病、慢阻肺、睡眠呼吸障碍、高血压、多病共管。
- 搜索：患者姓名、患者 ID、手机号、方案名称。

列表字段：

| 字段 | 展示内容 |
| --- | --- |
| 患者 | 姓名、性别年龄、患者 ID、主责医生关系 |
| 方案 | 方案名称、版本、来源、创建/更新时间 |
| 疾病与风险 | 适用疾病、风险等级、健康风险分 |
| 方案摘要 | 管理目标摘要、已纳入模块数量、执行负担 |
| 状态 | 方案状态、患者知晓状态 |
| 待处理 | 缺失模块、冲突、患者疑问、患者无法执行 |
| 操作 | 查看/编辑、确认下发、更多 |

列表操作规则：

- 待医生确认：主按钮“审核方案”。
- 草稿：主按钮“继续编辑”。
- 已下发待患者确认：主按钮“提醒患者确认”，更多中支持“标记已知晓”。
- 执行中：主按钮“查看方案”，更多中支持“调整方案”“停用方案”“创建随访”。
- 待调整：主按钮“编辑新版本”。
- 已停用/已完成：主按钮“查看历史”，更多中支持“复制为新方案”。

### 4 创建方案入口弹窗 SPEC

打开入口：

- 方案管理列表页“新建管理方案”。
- 患者详情 - 管理方案 Tab“新建管理方案”。
- 预警处理后“调整管理方案”。
- 随访结论“需要调整方案”。

字段：

| 字段 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 患者 | patient selector | 是 | 从入口带入时不可改；从列表新建时需选择 |
| 适用疾病 | multi-select | 是 | 默认带入医生确认疾病；至少选择 1 个 |
| 方案周期 | select/date range | 是 | 默认 14 或 30 天，支持 7/14/30/90 天 |
| 创建方式 | radio card | 是 | 系统推荐模板、复制历史方案、空白创建 |
| 历史方案 | selector | 条件必填 | 选择“复制历史方案”时必填 |
| 创建说明 | textarea | 否 | 医生内部备注，患者不可见 |

交互：

- 默认选中“系统推荐模板”。
- 选择疾病后，右侧展示将自动带出的核心目标和模块摘要。
- 选择“空白创建”时提示：系统仍会提供基础测量、预警、随访建议，但不会生成治疗类内容。
- 点击“开始编辑”后进入方案详情编辑页。

### 5 方案详情编辑页 SPEC

布局：

```text
顶部患者条
  患者基础信息、疾病标签、风险分、方案状态、版本、来源

左侧锚点导航
  方案基础信息
  管理目标
  指标测量方案
  症状记录方案
  用药方案
  设备监测方案
  生活方式方案
  预警规则
  随访计划
  患者指导

中间编辑区
  模块卡片列表

右侧审核面板
  完成度、风险提示、执行负担、患者可见内容入口、固定操作按钮
```

顶部患者条：

- 姓名、性别、年龄、手机号脱敏。
- 医生确认疾病、疾病风险标签。
- 健康风险分、风险状态。
- 当前方案状态、版本号、创建来源。
- 最近预警或最近随访结论摘要。

左侧锚点导航：

- 展示模块名称和状态点。
- 点击定位到对应模块。
- 有缺失字段时展示红点。
- 已修改模块展示蓝点。
- 未纳入条件模块展示灰点。

右侧审核面板：

| 区块 | 内容 |
| --- | --- |
| 必填完成度 | 例如 6/6 必须模块完成，2/4 条件模块纳入 |
| 下发风险 | 缺失字段、关闭建议模块、设备依赖冲突、方案冲突 |
| 执行负担 | 每日待办数、预计耗时、连续记录天数、随访频率 |
| 患者可见内容 | 进入预览弹窗 |
| 操作按钮 | 保存草稿、患者可见内容预览、确认下发、驳回推荐 |

按钮状态：

- 保存草稿：始终可用。
- 患者可见内容预览：必须模块至少有基础内容后可用；缺字段时仍可预览并标记“待补充”。
- 确认下发：所有下发校验通过后可用。
- 驳回推荐：仅系统草稿审核模式展示。

### 6 模块卡片与编辑抽屉通用 SPEC

模块卡片结构：

```text
模块标题 + 状态标签
摘要内容
系统推荐依据 / 最近数据
风险或缺失提示
操作区
```

通用字段：

| 字段 | 说明 |
| --- | --- |
| `module_key` | 模块编码 |
| `module_name` | 模块名称 |
| `module_type` | required/conditional/optional |
| `included` | 是否纳入本方案 |
| `status` | completed/incomplete/warning/modified/excluded |
| `summary` | 模块摘要 |
| `recommendation_reason` | 系统推荐依据 |
| `exclude_reason` | 未纳入原因 |
| `patient_visible` | 是否患者端可见 |
| `doctor_modified` | 是否医生修改过 |
| `fields` | 模块字段对象 |

编辑抽屉通用交互：

- 抽屉宽度建议 520-640px。
- 抽屉顶部展示模块名称、是否患者可见、系统推荐标签。
- 字段按“基础设置、执行规则、患者端说明、医生备注”分组。
- 修改系统推荐值后，字段旁展示“已修改”。
- 支持“恢复系统推荐”。
- 保存模块时只校验当前模块字段。
- 关闭抽屉时若有未保存修改，弹出离开确认。

### 7 各模块字段 SPEC

方案基础信息：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 方案名称 | input | 是 | 疾病 + 风险等级 + 周期生成 |
| 适用疾病 | multi-select | 是 | 从确诊疾病和风险标签选择 |
| 风险等级 | select | 是 | 低/中/高/危急或急性风险 |
| 方案周期 | select/date range | 是 | 7/14/30/90 天或自定义 |
| 计划开始日期 | date | 是 | 默认今日或患者确认次日 |
| 患者端摘要 | textarea | 是 | 30-120 字，患者可见 |
| 医生内部备注 | textarea | 否 | 患者不可见 |

管理目标：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 阶段目标 | textarea/template | 是 | 系统推荐，可编辑 |
| 目标周期 | select/date range | 是 | 默认跟随方案周期 |
| 量化目标列表 | target list | 是 | 至少启用 1 项 |
| 个体化原因 | select + textarea | 条件必填 | 修改默认目标范围时必填 |

指标测量方案：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 指标名称 | select | 是 | 血糖、血压、SpO2、呼吸频率、体重等 |
| 测量场景 | multi-select | 条件必填 | 血糖需选择空腹/餐后 2h/睡前/随机等 |
| 测量频率 | frequency selector | 是 | 每日、每周 N 天、按异常复测 |
| 推荐时间窗 | time window | 否 | 如 06:00-09:00 |
| 数据来源 | multi-select | 是 | 手动记录、设备采集、报告 |
| 目标范围 | target link | 否 | 可关联量化目标 |
| 患者端说明 | textarea | 是 | 面向患者说明何时测、怎么记录 |

症状记录方案：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 症状项 | multi-select | 是 | 按疾病推荐，如气促、咳痰、低血糖症状 |
| 记录方式 | select | 是 | 出现时记录、每日记录、随访前记录 |
| 严重程度 | switch | 否 | 是否记录轻/中/重 |
| 触发后动作 | multi-select | 否 | 提醒复测、进入随访准备、提示联系医生 |
| 患者端说明 | textarea | 是 | 避免专业术语 |

用药方案：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 药品名称 | input/select | 是 | 可从既有用药带入 |
| 单次剂量/用量 | input | 是 | 如 0.5g、1 片、2 吸 |
| 服用频次 | select/input | 是 | 每日 1 次、每日 2 次等 |
| 服用时间 | multi-time/select | 是 | 早餐后、晚餐后、睡前 |
| 开始/结束日期 | date range | 是 | 默认跟随方案周期 |
| 用药来源 | select | 是 | 既有处方、线下医嘱、出院记录、专科方案、患者自述、随访确认 |
| 是否关键用药 | switch | 否 | 关键药漏服可提示医生 |
| 患者端说明 | textarea | 是 | 不写处方决策语言 |

设备监测方案：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 设备类型 | multi-select | 是 | 血糖仪、血压计、血氧仪、睡眠设备、CPAP |
| 是否推荐绑定 | switch | 是 | 默认按设备依赖目标判断 |
| 必须设备采集指标 | list | 条件必填 | AHI、ODI、睡眠阶段、CPAP 使用时长等 |
| 支持手动补录指标 | list | 否 | 睡眠时长、入睡时间、起床时间等 |
| 同步频率 | select | 是 | 每次测量后、每日、每次睡眠报告后 |
| 同步失败提醒 | switch | 是 | 默认开启 |
| 患者端设备入口 | switch | 是 | 患者端展示添加设备入口 |

生活方式方案：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |

| 执行频率 | select | 否 | 每日、每周、自主执行 |
| 完成反馈 | select | 否 | 不反馈、患者自评、打卡 |
| 患者端说明 | textarea | 是 | 至少 1 条可执行建议 |

预警规则：

| 字段 | 类型 | 必填 | 默认值/规则 |

| 停用原因 | select + textarea | 条件必填 | 停用核心安全预警时必填 |

随访计划：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 首次随访时间 | date/days after start | 是 | 如下发后第 7 天 |

| --- | --- | --- | --- |

### 8 状态机与按钮规则

方案状态枚举：

| 状态 | 含义 |
| --- | --- |
| draft | 医生创建或编辑中的草稿 |
| pending_doctor | 系统生成或调整生成，待医生确认 |
| rejected | 医生驳回系统推荐 |
| pending_patient | 医生已确认下发，待患者知晓 |
| active | 患者已确认知晓并开始执行 |
| patient_question | 患者反馈有疑问 |
| patient_unable | 患者反馈暂时无法执行 |
| adjusting | 基于执行中方案生成的新版本调整草稿，待医生编辑下发 |
| review_due | 到期或触发复评，待医生复盘 |
| stopped | 医生停用 |
| completed | 阶段完成 |

按钮规则：

| 状态 | 主按钮 | 次级操作 |
| --- | --- | --- |
| draft | 继续编辑 | 保存草稿、删除草稿 |
| pending_doctor | 审核方案 | 修改后下发、驳回推荐 |
| rejected | 查看详情 | 复制为新方案 |
| pending_patient | 提醒患者确认 | 标记已知晓、撤回下发 |
| active | 查看方案 | 调整方案、停用方案、创建随访 |
| patient_question | 查看患者反馈 | 发送医生建议、调整方案 |
| patient_unable | 查看无法执行原因 | 调整方案、停用方案、发送医生建议 |
| adjusting | 编辑新版本 | 保存草稿、确认下发 |
| review_due | 方案复盘 | 继续当前方案、调整方案、结束本阶段 |
| stopped | 查看历史 | 复制为新方案 |
| completed | 查看复盘 | 复制为新方案 |

状态流转：

```text
draft -> pending_patient: 医生确认下发
pending_doctor -> pending_patient: 医生审核确认
pending_doctor -> rejected: 医生驳回推荐
pending_patient -> active: 患者确认已知晓并开始执行
pending_patient -> draft: 医生撤回下发
active -> adjusting: 医生选择调整方案
active -> review_due: 到期或触发复评
active -> stopped: 医生停用方案
active -> completed: 医生结束本阶段
active -> patient_question: 患者反馈有疑问
active -> patient_unable: 患者反馈无法执行
patient_question -> adjusting: 医生决定调整
patient_question -> active: 医生处理后继续执行
patient_unable -> adjusting: 医生决定调整
patient_unable -> stopped: 医生停用
review_due -> active: 医生复盘后继续当前方案
review_due -> adjusting: 医生复盘后调整方案
review_due -> completed: 医生复盘后结束本阶段
adjusting -> pending_patient: 新版本确认下发
```

### 9 下发校验规则

确认下发时必须通过以下校验：

| 校验项 | 规则 | 失败提示 |
| --- | --- | --- |
| 方案基础信息 | 名称、疾病、风险等级、周期、开始日期完整 | 请补充方案基础信息 |
| 管理目标 | 阶段目标 + 至少 1 个启用量化目标 | 请至少启用一个量化目标 |
| 指标测量方案 | 至少 1 个关键指标测量项 | 请配置关键指标测量方案 |
| 预警规则 | 至少保留核心安全预警 | 请保留核心安全预警或填写停用原因 |
| 随访计划 | 有首次随访或不自动随访原因 | 请设置首次随访或说明原因 |
| 患者指导 | 患者可见说明完整 | 请补充患者指导 |
| 条件模块 | 纳入后内部字段完整 | 请补充已纳入模块字段 |
| 设备依赖 | 设备依赖目标必须配置设备监测 | 当前目标依赖设备数据，请保留设备监测方案 |
| 用药依赖 | 用药执行率目标必须配置用药方案 | 当前目标依赖用药执行，请纳入用药方案 |
| 方案冲突 | 同疾病域执行中方案冲突已处理 | 请处理当前执行方案冲突 |

校验失败时：

- 页面顶部展示错误摘要。
- 右侧审核面板展示未完成项。
- 点击错误项定位到对应模块。
- 允许保存草稿，不允许确认下发。

### 10 患者可见内容预览 SPEC

预览以小程序卡片样式展示，内容来自医生端配置自动转换。

预览结构：

```text
本阶段目标
  阶段目标说明
  患者可见量化目标

今日待完成
  测量任务
  用药/治疗执行
  症状记录
  设备同步/睡眠报告

随访提醒
  首次随访时间
  随访前准备材料

异常时怎么办
  复测提示
  联系医生提示
  线下就医提示
```

转换规则：

| 医生端配置 | 患者端展示 |
| --- | --- |
| 管理目标 | 本阶段目标 |
| 指标测量方案 | 今日记录/测量任务 |
| 症状记录方案 | 症状出现时记录或每日症状记录 |
| 用药方案 | 用药提醒和打卡 |
| 设备监测方案 | 添加设备、同步数据、查看睡眠报告 |
| 生活方式方案 | 健康建议或待办 |
| 预警规则 | 异常时怎么办 |
| 随访计划 | 随访提醒和准备材料 |
| 患者指导 | 方案说明和医生补充说明 |

预览弹窗操作：

- 返回编辑。
- 确认下发。
- 若存在缺失项，确认下发按钮置灰，并展示缺失原因。

### 12 开发拆分建议

建议按以下顺序开发：

1. 方案列表页：状态筛选、分页、方案卡片、查看/编辑入口。
2. 创建方案入口弹窗：患者、疾病、周期、创建方式。
3. 方案详情编辑页框架：顶部患者条、左侧锚点、中间模块卡片、右侧审核面板。
4. 必须模块编辑：方案基础信息、管理目标、指标测量方案、预警规则、随访计划、患者指导。
5. 条件模块编辑：症状记录、用药、设备监测、生活方式。
6. 下发校验与错误定位。
7. 患者可见内容预览。
8. 状态流转：确认下发、患者知晓、执行中、调整、停用。
9. Mock 生成患者任务、随访计划和时间轴事件。

开发验收标准：

- 医生从方案列表能进入待确认方案并看到非空草稿内容。
- 医生能编辑至少 6 个必须模块并保存草稿。
- 条件模块关闭后不校验内部字段，重新纳入后恢复校验。
- 设备依赖目标存在时，设备监测方案不可关闭。
- 确认下发前能看到患者可见内容预览。
- 校验不通过时不能下发，并能定位缺失模块。
- 校验通过后方案进入“已下发待患者确认”。
- 模拟患者确认后方案进入“执行中”。
- 下发和确认事件写入时间轴。
