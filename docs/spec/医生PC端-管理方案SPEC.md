# 医生PC端-管理方案交互开发SPEC

> 来源：医生PC端PRD 15 管理方案模块。本文档承接管理方案页面交互、组件字段、状态按钮、下发校验、患者可见内容预览、前端 Mock Schema 与开发拆分。主 PRD 仅保留业务规则与范围说明。

## 管理方案交互开发 SPEC

本 SPEC 用于进入前端 Mock 闭环开发。目标不是一次性做完整模板后台，而是先把管理方案从“空草稿”升级为可查看、可编辑、可校验、可预览、可下发、可流转的生产级闭环。

### 1 本期开发目标

本期必须跑通以下闭环：

```text
方案列表
  -> 创建/查看方案草稿
  -> 进入方案详情编辑页
  -> 编辑必须模块和条件模块
  -> 保存草稿
  -> 查看患者可见内容预览
  -> 确认下发
  -> 方案进入“已下发待患者确认”
  -> 模拟患者确认已知晓
  -> 方案进入“执行中”
  -> Mock 生成患者端待办、计划内随访和时间轴事件
```

P0 暂不开发：

- 模板后台配置。
- 复杂规则引擎。
- 多医生协同编辑。
- 批量下发。
- 字段级版本 diff。
- 真实患者端页面联调。

### 2 页面与弹层清单

| 页面/弹层 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 方案管理列表页 | 集中处理草稿、待患者确认、执行中和历史方案 | 状态筛选、提醒标签、患者信息、方案名称、来源、版本、关键模块摘要、患者确认状态 | 查看/编辑、确认下发、复制方案、创建随访 |
| 创建方案入口弹窗 | 避免医生进入空白页 | 患者、适用疾病、方案周期、创建方式 | 下一步进入方案编辑页 |
| 方案详情编辑页 | 医生审核和编辑方案的主页面 | 顶部患者条、证据摘要、模块卡片、审核面板 | 保存草稿、患者可见内容预览、确认下发、驳回推荐 |
| 模块编辑抽屉 | 编辑单个方案模块 | 动态字段、推荐值、医生修改原因 | 保存模块、取消、恢复系统推荐 |
| 患者可见内容预览弹窗 | 医生确认患者端展示内容 | 阶段目标、今日待完成、随访提醒、异常处理提示 | 返回编辑、确认下发 |
| 确认下发弹窗 | 下发前二次确认 | 校验结果、冲突提示、旧方案处理方式 | 确认下发、返回修改 |
| 驳回推荐弹窗 | 驳回系统草稿 | 驳回原因、医生备注 | 确认驳回 |

### 3 方案管理列表页 SPEC

顶部筛选：

- 方案状态：全部、草稿、已下发待患者确认、执行中、已驳回、已停用、已完成。
- 提醒标签：全部、患者未确认、患者有疑问、患者无法执行、即将到期、待复盘。
- 方案来源：全部、系统草稿、医生创建、模板创建、历史复制。
- 疾病：全部、糖尿病、慢阻肺、睡眠呼吸障碍、高血压、多病共管。
- 搜索：患者姓名、患者 ID、手机号、方案名称。

列表字段：

| 字段 | 展示内容 |
| --- | --- |
| 患者 | 姓名、性别年龄、患者 ID、主责医生关系 |
| 方案 | 方案名称、版本、来源、创建/更新时间 |
| 疾病与风险 | 适用疾病、风险等级、健康风险分 |
| 方案摘要 | 管理目标摘要、已纳入模块数量、执行负担 |
| 状态 | 方案生命周期状态、患者知晓状态 |
| 提醒标签 | 缺失模块、冲突、患者未确认、患者疑问、患者无法执行、即将到期、待复盘 |
| 操作 | 查看/编辑、确认下发、更多 |

列表操作规则：

- 草稿：主按钮“审核/继续编辑”。
- 已下发待患者确认：主按钮“提醒患者确认”，更多中支持“标记已知晓”。
- 执行中：主按钮“查看方案”，更多中支持“调整方案”“停用方案”“创建随访”“复测指标”“发送医生建议”。
- 已驳回：主按钮“查看详情”，更多中支持“复制为新方案”。
- 已停用：主按钮“查看历史”，更多中支持“复制为新方案”。
- 已完成：主按钮“查看复盘”，更多中支持“复制为新方案”。

### 4 创建方案入口弹窗 SPEC

打开入口：

- 方案管理列表页“新建管理方案”。
- 患者详情 - 管理方案 Tab“新建管理方案”。
- 预警处理后医生明确需要修改长期方案。
- 随访结论后医生明确需要修改长期方案。

字段：

| 字段 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 患者 | patient selector | 是 | 从入口带入时不可改；从列表新建时需选择 |
| 适用疾病 | multi-select | 是 | 糖尿病、慢阻肺、睡眠呼吸障碍、高血压；从患者确认疾病或风险标签中选择 |
| 方案周期 | select/date range | 是 | 7/14/30/90/180 天 |
| 创建方式 | radio card | 是 | 系统推荐模板、复制历史方案、空白创建 |
| 历史方案 | selector | 条件必填 | 选择“复制历史方案”时必填 |
| 创建说明 | textarea | 否 | 医生内部备注，患者不可见 |

若从执行中方案进入调整流程，进入编辑页前必须先补充：

| 字段 | 类型 | 必填 | 规则 |
| --- | --- | --- | --- |
| 调整类型 | select | 是 | 预警后调整、随访后调整 |
| 调整原因 | textarea | 是 | 患者不可见 |

交互：

- 默认选中“系统推荐模板”。
- 选择疾病后，右侧展示将自动带出的核心目标和模块摘要。
- 选择“空白创建”时提示：系统仍会提供基础测量、预警、随访建议，但不会生成治疗类内容。
- 点击“开始编辑”后进入方案详情编辑页。

### 5 方案详情编辑页 SPEC

布局：

```text
顶部患者条
  患者基础信息、疾病标签、风险分、方案状态、版本、来源

布局（P0 两栏）：

```text
顶部患者条
  患者基础信息、疾病标签、风险分、方案状态、版本、来源

中间编辑区（含锚点导航条）
  方案基础信息
  管理目标
  指标测量方案
  症状记录方案
  用药方案
  设备监测方案
  生活方式方案
  预警规则
  随访计划
  患者指导

右侧审核面板
  完成度、执行负担、快速审核入口、患者可见内容入口、固定操作按钮
```

P0 不做左侧证据区独立栏。证据信息下沉到各模块卡片的"系统推荐依据"和"最近数据"区域。

右侧审核面板
  完成度、风险提示、执行负担、患者可见内容入口、固定操作按钮
```

顶部患者条：

- 姓名、性别、年龄、手机号脱敏。
- 医生确认疾病、疾病风险标签。
- 健康风险分、风险状态。
- 当前方案状态、版本号、创建来源。
- 最近预警或最近随访结论摘要。

锚点导航条（编辑区顶部固定）：

- 展示模块名称和状态点。
- 点击定位到对应模块。
- 有缺失字段时展示红点。
- 已修改模块展示蓝点。
- 未纳入条件模块展示灰点。

右侧审核面板：

| 区块 | 内容 |
| --- | --- |
| 必填完成度 | 例如 6/6 必须模块完成，2/4 条件模块纳入 |
| 下发风险 | 缺失字段、关闭建议模块、设备依赖冲突、方案冲突 |
| 执行负担 | 每日待办数、预计耗时、连续记录天数、随访频率 |
| 快速审核入口 | 点击进入快速审核页面（仅系统草稿场景展示） |
| 患者可见内容 | 进入预览弹窗 |
| 操作按钮 | 保存草稿、确认下发、驳回推荐（仅系统草稿） |

按钮状态：

- 保存草稿：始终可用。
- 患者可见内容预览：必须模块至少有基础内容后可用；缺字段时仍可预览并标记“待补充”。
- 确认下发：所有下发校验通过后可用。
- 驳回推荐：仅系统草稿审核模式展示。

### 6 模块卡片与编辑抽屉通用 SPEC

模块卡片结构：

```text
模块标题 + 状态标签
摘要内容
系统推荐依据 / 最近数据
风险或缺失提示
操作区
```

通用字段：

| 字段 | 说明 |
| --- | --- |
| `module_key` | 模块编码 |
| `module_name` | 模块名称 |
| `module_type` | required/conditional/optional |
| `included` | 是否纳入本方案 |
| `status` | completed/incomplete/warning/modified/excluded |
| `summary` | 模块摘要 |
| `recommendation_reason` | 系统推荐依据 |
| `exclude_reason` | 未纳入原因 |
| `patient_visible` | 是否患者端可见 |
| `doctor_modified` | 是否医生修改过 |
| `fields` | 模块字段对象 |

编辑抽屉通用交互：

- 抽屉宽度建议 520-640px。
- 抽屉顶部展示模块名称、是否患者可见、系统推荐标签。
- 字段按“基础设置、执行规则、医生备注”分组。
- 患者端说明不由医生手动填写，由系统根据管理目标、执行规则、随访安排和异常处理动作自动编译生成，并在编辑页只读展示。
- 自动生成的患者端说明必须避免专业术语和规则配置语言，例如将 AHI、SpO2、CPAP 等转换为患者可理解的睡眠呼吸事件、血氧、呼吸机/通气设备等表达。
- 修改系统推荐值后，字段旁展示“已修改”。
- 支持“恢复系统推荐”。
- 保存模块时只校验当前模块字段。
- 关闭抽屉时若有未保存修改，弹出离开确认。

### 7 各模块字段 SPEC

方案基础信息：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 方案名称 | input | 是 | 疾病 + 周期生成 |

| 方案周期 | select/date range | 是 | 7/14/30/90/180 天 |
| 计划开始日期 | date | 是 | 患者确认次日开始 |

| 医生内部备注 | textarea | 否 | 患者不可见 |

管理目标（P0 简化）：

| 字段 | 类型 | 必填 | 默认值/规则 |
| --- | --- | --- | --- |
| 阶段目标 | textarea/template | 是 | 系统推荐，可编辑，20-80 字 |
| 目标周期 | select | 是 | 默认跟随方案周期 |
| 量化目标列表 | target list | 是 | 系统预置，医生可启用/停用/调整目标值；至少启用 1 项 |

量化目标每项字段（P0 简化）：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 目标项 | 预置不可改 | 是 | 从疾病目标库预置，P0 不支持自定义创建 |
| 单位 | auto | 是 | 选择目标项后自动带出 |
| 目标值 | number/range | 是 | 预置默认值，医生可调整；调整时自动留痕 |
| 是否启用 | switch | 是 | 至少启用 1 项 |

P0 删除字段（P1 延后）：个体化原因（改为修改时自动留痕）、关联疾病、关联方案项、目标类型、统计周期、是否用于预警辅助、患者端可见开关（P0 默认全部可见）。

| 患者端说明 | system-generated readonly | 否 | 根据指标、频率、时间窗和目标自动生成，面向患者说明何时测、怎么记录 |

预警规则：

| 字段 | 类型 | 必填 | 默认值/规则 |

| 每日/每周要做什么 | generated list + edit | 是 | 来自测量、用药、设备、随访配置 |
| 异常时怎么办 | system-generated readonly | 是 | 根据预警规则和患者端动作自动生成复测、联系医生、线下就医提示 |
| 医生补充说明 | textarea | 否 | 患者可见，需通俗表达 |

### 8 状态机与按钮规则

方案主状态只表达方案生命周期。患者反馈、复盘提醒、调整需求等不进入主状态，统一作为提醒标签或事件记录。

方案状态枚举：

| 状态 | 含义 |
| --- | --- |
| draft | 医生编辑中或系统推荐待确认，患者不可见（含原 pending_doctor） |
| pending_patient | 医生已确认下发，待患者确认知晓 |
| active | 患者已确认知晓并开始执行 |
| rejected | 医生驳回系统推荐，仅系统草稿场景 |
| stopped | 医生停用，不再执行 |
| completed | 阶段结束 |

提醒标签（不作为主状态）：

| 标签 | 展示/处理方式 |
| --- | --- |
| patient_unconfirmed | 已下发待患者确认 + "患者未确认"标签 |
| patient_question | 执行中 + "患者有疑问"标签，医生可发送建议或调整方案 |
| patient_unable | 执行中 + "患者无法执行"标签，医生可发送建议、调整方案或停用方案 |
| expiring_soon | 执行中 + "即将到期"标签 |
| review_due | 执行中 + "待复盘"标签，医生可结束本阶段或调整方案 |
| adjustment_draft | 草稿 + "调整版本"标签 |

按钮规则：

| 状态 | 主按钮 | 次级操作 |
| --- | --- | --- |
| draft（含系统推荐） | 审核/继续编辑 | 保存草稿、确认下发、驳回推荐（仅系统草稿）、删除草稿 |
| pending_patient | 提醒患者确认 | 标记已知晓、撤回下发 |
| active | 查看方案 | 调整方案（原方案进入调整态）、停用方案、创建随访、结束本阶段 |
| rejected | 查看详情 | 复制为新方案 |
| stopped | 查看历史 | 复制为新方案 |
| completed | 查看复盘 | 复制为新方案 |

状态流转：

```text
draft -> pending_patient: 医生确认下发
draft -> rejected: 医生驳回系统推荐
pending_patient -> active: 患者确认已知晓并开始执行
pending_patient -> draft: 医生撤回下发
active -> draft: 医生调整方案（原方案进入调整态，保留同一方案记录和版本轨迹）
active -> stopped: 医生停用方案
active -> completed: 医生结束本阶段
rejected/stopped/completed -> draft: 复制为新方案
active + "患者有疑问"标签: 医生处理后去除标签、发送建议或调整方案
active + "患者无法执行"标签: 医生处理后去除标签、发送建议、调整方案或停用
active + "待复盘"标签: 医生复盘后继续执行、在原方案上调整或结束本阶段
```

任务版本换代规则：

- 方案调整不新增方案主记录，原方案进入 `draft + adjustment_draft`，版本号递增。
- 调整编辑和待患者确认期间，患者端继续执行上一已生效版本任务。
- 患者确认新版方案后，系统关闭同一方案旧版本下未完成任务，状态置为 `closed`，并记录关闭原因“方案版本调整”。
- 已完成任务、设备同步结果、手动记录和随访证据保留在原版本，不随新版配置重算。
- 系统按新版方案模块重新生成患者端任务，任务需写入 `relatedPlanId` 和 `planVersion`，用于医生端执行概览按版本追踪。

### 9 下发校验规则

确认下发时必须通过以下校验：

| 校验项 | 规则 | 失败提示 |
| --- | --- | --- |
| 方案基础信息 | 名称、疾病、周期、开始日期完整 | 请补充方案基础信息 |
| 管理目标 | 阶段目标 + 至少 1 个启用量化目标 | 请至少启用一个量化目标 |
| 指标测量方案 | 至少 1 个关键指标测量项 | 请配置关键指标测量方案 |
| 预警规则 | 至少保留核心安全预警 | 请保留核心安全预警或填写停用原因 |
| 随访计划 | 选择安排随访时需设置首次随访时间；不安排随访时仅关闭随访安排 | 请设置首次随访时间或关闭随访安排 |
| 患者指导 | 系统可根据目标、待办、随访和异常动作生成患者可见说明 | 请补充生成患者指导所需的目标或执行规则 |
| 条件模块 | 纳入后内部字段完整 | 请补充已纳入模块字段 |
| 设备依赖 | 设备依赖目标必须配置设备监测 | 当前目标依赖设备数据，请保留设备监测方案 |
| 用药依赖 | 用药执行率目标必须配置用药方案 | 当前目标依赖用药执行，请纳入用药方案 |
| 方案冲突 | 同疾病域执行中方案冲突已处理（P0 仅检测疾病域冲突） | 请处理当前执行方案冲突 |

校验失败时：

- 页面顶部展示错误摘要。
- 右侧审核面板展示未完成项。
- 点击错误项定位到对应模块。
- 允许保存草稿，不允许确认下发。

### 12 快速审核通道

系统推荐方案的大部分内容医生无需逐模块审核。P0 提供快速审核入口，帮助医生在 3 分钟内完成审核下发。

快速审核页面结构：

```text
顶部患者条（同详情页）

  方案名称、适用疾病、周期
  系统推荐依据摘要（1-2句）

### 14 条件模块关闭后果预览

医生点击"关闭纳入"时，弹窗展示关闭后果：

- 将失去哪些患者端功能（如：关闭用药方案 → 患者端不再有用药提醒和打卡）
- 将影响哪些量化目标（如：关闭设备监测方案 → AHI 和最低血氧目标失去数据来源）
- 是否影响核心安全预警（如：关闭设备监测方案 → 夜间低氧预警失去设备数据触发条件）

不可关闭的模块：后果预览中标注"不可关闭"，并展示原因（如：当前方案启用 AHI 目标依赖设备数据）。

医生确认关闭后：关联量化目标自动标记"无数据来源"，患者端移除相关待办。

P0 简化：后果预览用静态规则判断（模块→目标→预警的依赖关系由模板预置），不做动态规则引擎。

### 15 开发拆分建议

建议按以下顺序开发（P0）：

1. 方案列表页：6 状态筛选、分页、方案卡片、查看/编辑入口。
2. 创建方案入口弹窗：患者、疾病、周期、创建方式。
3. 方案详情编辑页框架：顶部患者条、锚点导航条、中间模块卡片、右侧审核面板（两栏布局）。
4. 必须模块编辑：方案基础信息、管理目标（简化字段）、指标测量方案、预警规则、随访计划、患者指导。
5. 条件模块编辑：症状记录、用药、设备监测、生活方式。关闭时展示后果预览。
6. 下发校验与错误定位（P0 仅疾病域冲突检测）。
7. 患者可见内容预览。
8. 状态流转（6 状态）：确认下发、患者知晓、执行中、原方案调整、停用、完成、驳回。
9. 快速审核通道（系统草稿 3 分钟审核）。
10. 执行概览：目标 vs 实际对比、依从概况、风险提示。
11. Mock 生成患者任务、随访计划和时间轴事件。

开发验收标准：

- 医生从方案列表能进入待确认方案并看到非空草稿内容。
- 医生能编辑至少 6 个必须模块并保存草稿。
- 条件模块关闭后不校验内部字段，关闭时展示后果预览（影响的目标、预警、患者端功能）。
- 设备依赖目标存在时，设备监测方案不可关闭。
- 确认下发前能看到患者可见内容预览。
- 校验不通过时不能下发，并能定位缺失模块。
- 校验通过后方案进入"已下发待患者确认"。
- 模拟患者确认后方案进入"执行中"。
- 方案状态仅 6 种：草稿、已下发待患者确认、执行中、已驳回、已停用、已完成。
- 执行中方案展示执行概览（目标 vs 实际、依从概况）。
- 系统草稿提供快速审核通道入口。
- 下发和确认事件写入时间轴。
- 管理目标量化目标不支持自定义创建，仅启用/停用/调整预置目标值。
