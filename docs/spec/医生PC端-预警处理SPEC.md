# 医生PC端-预警处理SPEC

> 来源：主 PRD《预警闭环PRD》与旧版医生 PRD 预警章节。本文档聚焦医生端预警列表、详情、处理动作、跳转关系与开发落地。

## 1. 本期目标

P0 必须跑通：

```text
预警列表
  -> 查看预警详情
  -> 理解证据链
  -> 做出处置动作
  -> 生成复测指标 / 创建随访 / 发送医生建议 / 必要时方案调整
  -> 记录处理结果和关闭原因
```

P0 暂不做：

- 批量处理
- 复杂大屏
- 多医生协同分派
- 复杂 AI 预测页

## 2. 页面清单

| 页面/弹层 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 预警列表页 | 医生集中处理异常 | 顶部统计卡、筛选、预警列表 | 查看详情、快速处理 |
| 预警详情页 | 单条预警的证据与处置页 | 预警结论、证据链、风险分变化、处理区 | 复测指标、创建随访、发送建议、调方案、关闭 |
| 关闭预警弹窗 | 关闭留痕 | 关闭原因、补充说明 | 确认关闭 |
| 创建随访抽屉 | 从预警转随访 | 计划时间、准备材料、患者说明 | 确认创建 |
| 医生建议抽屉 | 从预警发建议 | 模板、文案、是否生成待办 | 发送建议 |

## 3. 预警列表页

### 3.1 页面结构

```text
顶部统计卡
  全部 / 提醒 / 重要 / 紧急 / 待医生处理 / 今日新增

筛选区
  时间 / 等级 / 疾病 / 来源 / 状态 / 负责人

预警列表
  患者 / 预警信息 / 关键证据 / 风险变化 / 状态 / 操作
```

### 3.2 筛选项

| 筛选 | 选项 |
| --- | --- |
| 时间 | 今日 / 本周 / 近30天 / 自定义 |
| 等级 | 提醒 / 重要 / 紧急 |
| 疾病 | 糖尿病 / 慢阻肺 / 睡眠呼吸障碍 / 高血压 / 多病共管 |
| 来源 | 指标异常 / 症状 / 用药依从性 / 设备同步 / 睡眠报告 |
| 状态 | 待患者处理 / 待医生处理 / 已处理 / 已关闭 |
| 负责人 | 当前医生 / 家庭医生 / 健康管理师 |

### 3.3 列表字段

| 字段 | 内容 |
| --- | --- |
| 患者 | 姓名、年龄、绑定关系 |
| 预警信息 | 标题、等级、触发时间、来源疾病 |
| 关键证据 | 1-2 条摘要证据 |
| 风险变化 | 健康风险分变化、影响专项分 |
| 状态 | 待患者处理 / 待医生处理 / 已处理 / 已关闭 |
| 操作 | 查看详情、快速关闭 |

### 3.4 列表交互原则

- 默认按紧急程度和触发时间排序
- 紧急预警使用显著状态色
- 行内操作只保留 `查看详情`
- `快速关闭` 仅对提醒级或已明确误报场景开放

## 4. 预警详情页

### 4.1 页面结构

```text
顶部患者条
  患者信息 / 疾病 / 风险等级 / 当前方案 / 最近随访

左侧证据区
  预警结论
  触发规则
  原始证据
  趋势图
  风险分变化

中间主内容区
  患者备注 / 症状 / 用药 / 设备状态
  相似历史与处理历史

右侧处理区
  推荐动作
  医生处理动作按钮
  内部备注
```

### 4.2 必须展示内容

- 预警结论
- 预警等级
- 触发前后健康风险分变化
- 影响专项分
- 触发规则/规则版本
- 原始证据记录
- 趋势图
- 患者备注、症状、用药、设备状态
- 推荐处理动作

### 4.3 处理动作

| 动作 | 患者端可见 | 说明 |
| --- | --- | --- |
| 复测指标 | 是 | 打开复测指标抽屉，生成复测任务 |
| 发送医生建议 | 是 | 打开建议抽屉 |
| 调整记录频率 | 是 | 更新患者端待办规则 |
| 调整管理方案 | 是 | 仅在长期方案确需变化时使用，进入前需填写调整类型与原因 |
| 创建随访 | 是 | 打开创建随访抽屉 |
| 建议线下就医 | 是 | 紧急或疑似急性风险 |
| 继续观察 | 是 | 需填写原因 |
| 关闭预警 | 可选 | 需填写关闭原因 |
| 内部备注 | 否 | 仅医生端可见 |

### 4.4 处理要求

- 紧急预警必须填写处理意见
- 系统建议必须记录采纳、修改或驳回
- 关闭预警必须记录原因
- 处理后需追踪患者是否执行

## 5. 关闭预警弹窗

### 5.1 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 关闭原因 | select | 是 | 已处理、数据误差、设备误差、患者已线下处理、无需继续处理、其他 |
| 补充说明 | textarea | 否 | 面向医生留痕 |

### 5.2 交互

- 紧急预警关闭前需二次确认
- 关闭后事件保留在时间轴和历史列表中

## 6. 与其他模块的跳转

| 来源/去向 | 行为 |
| --- | --- |
| 患者管理列表 -> 预警详情 | 定位到对应预警 |
| 预警详情 -> 复测指标 | 自动带入患者、预警和推荐指标 |
| 预警详情 -> 创建随访 | 自动带入患者与预警 |
| 预警详情 -> 调整方案 | 跳到该患者当前方案，并先填写调整类型与原因 |
| 预警详情 -> 发送医生建议 | 自动带入预警来源 |
| 预警详情 -> 患者详情页 | 查看更完整背景 |

## 7. 状态与按钮口径

- `待患者处理`：显示患者是否已完成复测/补录
- `待医生处理`：医生可做完整处置
- `已处理`：显示处理动作摘要
- `已关闭`：显示关闭原因

## 8. 开发注意点

- 预警处理和复测指标/方案/随访/建议必须互相可跳转
- 证据链要能打开原始记录详情
- 关闭预警不删除事件

## 9. 一句话总结

**预警详情页不是信息页，它本质上是医生的异常处置工作台。**
