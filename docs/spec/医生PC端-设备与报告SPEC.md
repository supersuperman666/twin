# 医生PC端-设备与报告SPEC

> 来源：主 PRD《设备与报告闭环PRD》与旧版医生 PRD 设备/报告章节。本文档聚焦医生端设备状态查看、报告详情和可信度提示。

## 1. 本期目标

P0 必须跑通：

```text
设备与报告区域
  -> 查看设备状态
  -> 查看睡眠/血氧报告
  -> 判断数据是否可信
  -> 从报告进入预警 / 随访 / 方案处理
```

## 2. 页面清单

| 页面/区域 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 设备状态区 | 设备总览 | 型号、设备号、同步状态 | 查看详情 |
| 睡眠报告列表 | 报告入口 | 报告时间、摘要、有效性 | 查看完整报告 |
| 报告详情页/抽屉 | 单次报告详情 | AHI、ODI、最低血氧等 | 标记无效、建议复测、创建随访 |

## 3. 设备状态展示

### 3.1 字段

- 已绑定设备
- 设备型号
- 设备号
- 绑定时间
- 最近同步时间
- 电量和同步状态
- 支持指标能力
- 异常和无效数据

### 3.2 交互原则

- 同步异常要显著提示
- 关键设备未绑定要能被医生识别

## 4. 睡眠报告展示

### 4.1 关键信息

- 睡眠时长、入睡时间、起床时间
- AHI、ODI、最低血氧、平均血氧、低氧累计时长
- 呼吸暂停、低通气、事件类型
- 睡眠分期、体位、鼾声、体动
- CPAP 使用时长、漏气、残余 AHI
- 报告有效性和设备能力说明

### 4.2 医生动作

- 查看完整报告
- 标记报告无效
- 建议复测睡眠
- 建议睡眠门诊或 CPAP 随访

## 5. 数据可信度提示

需要显式提示：

- 数据来源差异
- 报告是否无效
- 是否连续缺失
- 是否因设备型号不同导致能力差异

## 6. 与其他模块的跳转

| 来源/去向 | 行为 |
| --- | --- |
| 报告详情 -> 预警处理 | 处理因报告触发的预警 |
| 报告详情 -> 创建随访 | 带入报告摘要 |
| 报告详情 -> 调整方案 | 进入当前方案 |

## 7. 一句话总结

**设备与报告页解决的是“这些数据可不可信，以及我下一步该怎么用它们”。**
