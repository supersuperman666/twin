# 患者端-首页SPEC

> 来源：主 PRD《医患闭环主PRD》、患者端总览 SPEC 与旧版小程序 PRD 首页章节。本文档聚焦患者端首页作为主工作台的布局、状态分层和卡片交互。

## 1. 本期目标

P0 必须跑通：

```text
首页
  -> 识别当前用户状态
  -> 展示今日最重要信息
  -> 进入方案页 / 记录页 / 预警处理 / 随访
```

## 2. 用户状态分层

| 状态 | 页面表达 | 主按钮 |
| --- | --- | --- |
| 新用户 | 待评估，提示完成健康筛查 | 开始健康筛查 |
| 已筛查未管理 | 展示风险摘要和建议进一步完善信息 | 重新筛查 |
| 管理中 | 展示今日状态、关键异常和今天需要处理的动作 | 无固定主按钮 |

## 3. 页面结构

```text
顶部状态卡
  当前状态 / 健康风险分 / 医生信息

快捷记录区
  血糖 / 血压 / 血氧 / 睡眠 / 用药

风险预警区
  重要/紧急预警

今日待办区
  今日任务列表

随访计划区
  下次随访时间 / 去查看

健康档案区
  疾病标签 / 风险标签 / 设备状态
```

## 4. 模块说明

### 4.1 顶部状态卡

展示：

- 当前管理状态
- 健康风险分
- 当前主责医生
- 一句话风险提示或当前状态摘要

不建议在首页顶部状态卡中展开展示：

- 当前阶段目标
- 方案周期
- 方案明细
- 随访准备细节

### 4.2 快捷记录区

入口：

- 血糖
- 血压
- 血氧
- 睡眠
- 用药

点击后进入对应记录页或执行页。

### 4.3 风险预警区

展示：

- 重要/紧急预警卡片
- 一句话原因
- 行动按钮：去复测 / 查看报告 / 联系医生

### 4.4 今日待办区

展示：

- 今日待完成任务
- 完成进度
- 每条任务的主按钮

定义规则：

- `今日待办` 只包含患者当天需要主动完成的执行动作
- 本期纳入：
  - 当天计划内记录任务
  - 当天复测任务
  - 当天必须完成的随访准备动作（检查报告上传、量表问卷填写）
- 本期不纳入：
  - 医生建议
  - 普通准备说明
  - 长期方案说明

### 4.5 随访计划区

展示：

- 下次随访时间
- 倒计时
- 一句话提醒
- 按钮：`去查看`

交互原则：

- 首页只承担“随访提醒”职责，不展开完整准备材料
- 随访准备细节与随访历史统一进入方案页 / 随访记录页查看

## 5. 空状态

| 场景 | 页面表现 |
| --- | --- |
| 暂无预警 | 展示今日暂无预警 |
| 暂无用药计划 | 展示还没有用药计划 |
| 暂无待办 | 展示今日任务已完成 |

## 6. 跳转关系

| 模块 | 跳转 |
| --- | --- |
| 快捷记录 | 对应记录执行页 |
| 风险预警 | 预警详情或对应处理页 |
| 今日待办 | 方案页今日任务或具体执行页 |
| 随访计划 | 方案页随访安排区或随访记录页 |
| 健康档案 | 健康档案页 |

## 7. 一句话总结

**首页是患者今天的管理工作台，重点解决“今天先做什么”，不展开完整方案和详细随访内容。**
