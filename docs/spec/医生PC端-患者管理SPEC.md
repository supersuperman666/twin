# 医生PC端-患者管理SPEC

> 来源：主 PRD《患者管理闭环PRD》与旧版医生 PRD 患者管理/患者详情章节。本文档聚焦医生端患者列表、统计卡、筛选、患者详情与快捷入口。

## 1. 本期目标

P0 必须跑通：

```text
患者管理列表
  -> 搜索和筛选
  -> 找到待处理患者
  -> 进入患者详情
  -> 从列表快速跳到预警 / 方案 / 随访 / 疾病确认
```

## 2. 页面清单

| 页面/区域 | 定位 | 关键内容 | 主要动作 |
| --- | --- | --- | --- |
| 患者管理列表页 | 默认首页、轻量工作台 | 搜索、统计卡、筛选、患者列表 | 查看详情、更多跳转 |
| 添加患者弹层 | 建立医患关系入口 | 绑定二维码、说明 | 展示二维码 |
| 患者详情页 | 单患者主工作台 | 顶部患者条、左侧导航、中间内容、右侧决策栏 | 处理预警、调方案、发建议、建随访 |

## 3. 患者管理列表页

### 3.1 页面结构

```text
顶部工具区
  搜索框：患者姓名 / 患者ID / 患者手机号
  主按钮：添加患者

统计概览
  全部患者 / 重点关注 / 疾病风险待确认 / 待处理预警 / 待确认方案 / 今日待随访 / 数据缺失 / 更多筛选

高级筛选区
  默认收起
  点击更多筛选展开

患者列表
  紧凑表格
```

### 3.2 顶部统计卡

| 统计卡 | 口径 | 点击行为 |
| --- | --- | --- |
| 全部患者 | 当前医生授权患者总数 | 清空筛选 |
| 重点关注 | 医生主动标记的重点患者 | 筛选重点关注 |
| 疾病风险待确认 | 风险已生成但医生未确认 | 筛选待确认疾病风险患者 |
| 待处理预警 | 存在待医生处理预警的患者 | 筛选预警患者 |
| 待确认方案 | 系统已生成方案但尚未医生确认 | 筛选待确认方案患者 |
| 今日待随访 | 今天计划随访或逾期未处理患者 | 筛选今日/逾期随访 |
| 数据缺失 | 连续缺失关键记录或设备未同步患者 | 筛选数据缺失患者 |
| 更多筛选 | 当前高级筛选数量 | 展开/收起筛选面板 |

### 3.3 高级筛选

| 筛选组 | 选项 |
| --- | --- |
| 重点关注 | 全部、仅看重点关注 |
| 风险状态 | 稳定、需关注、需干预 |
| 疾病风险 | 糖尿病风险、慢阻肺风险、睡眠呼吸障碍风险、高血压风险、待确认 |
| 确诊疾病 | 糖尿病、慢阻肺、睡眠呼吸障碍、高血压、多病共管 |
| 待处理事项 | 有预警、待确认方案、今日待随访、数据缺失 |
| 方案状态 | 无方案、草稿、待患者确认、执行中、已停用、已完成 |
| 随访状态 | 无随访、今日待随访、逾期随访、近期已随访 |
| 数据状态 | 正常、关键指标缺失、设备未同步、记录完整率低 |
| 医患关系 | 主责医生患者、协作医生患者 |
| 设备状态 | 已绑定、未绑定、同步异常 |

### 3.4 列表字段

| 字段 | 说明 |
| --- | --- |
| 患者信息 | 星标、姓名、性别、年龄、手机号脱敏、患者 ID、绑定时间 |
| 医患关系 | 当前主责医生、医生角色、协作医生情况 |
| 疾病标签 | 医生确认疾病、风险标签、待确认状态 |
| 今日状态 | 今日待办完成度、是否有新记录 |
| 最新关键指标 | 血糖、血压、SpO2、睡眠摘要、症状摘要 |
| 数字孪生状态 | 稳定、趋势偏离、风险升高、干预中 |
| 最新预警 | 预警等级、名称、触发时间 |
| 设备状态 | 已绑定、未绑定、同步异常 |
| 方案状态 | 无方案、草稿、待患者确认、执行中、已停用、已完成 |
| 随访状态 | 下次随访日期、是否逾期、随访类型 |
| 操作 | 查看详情、更多 |

### 3.5 行内操作

主操作：

- 查看详情

更多菜单：

| 操作 | 触发条件 | 结果 |
| --- | --- | --- |
| 去处理预警 | 有待处理预警 | 跳转预警中心并定位 |
| 去确认方案 | 有待审核方案 | 跳转方案管理并定位 |
| 去完成随访 | 今日待随访或逾期 | 跳转随访计划并定位 |
| 去确认疾病 | 有疾病风险待确认 | 进入患者详情对应区域 |
| 查看数据缺失 | 关键指标缺失或设备同步异常 | 进入患者详情对应区域 |

### 3.6 列表交互原则

- 列表默认使用紧凑表格，不做大卡片
- 顶部统计卡承担快捷筛选，不做复杂看板
- 高级筛选默认收起
- 操作栏控制为 `1 个主按钮 + 1 个更多菜单`

## 4. 添加患者弹层

### 4.1 内容

- 绑定二维码
- 医生信息
- 使用说明
- 有效期提示

### 4.2 目标

- 建立医患绑定关系
- 不在这里承载复杂分配和协作流程

## 5. 患者详情

### 5.1 页面结构

```text
顶部患者条
  基础信息 / 医患关系 / 疾病标签 / 风险等级 / 快捷操作

左侧锚点导航
  总览、数字孪生、指标趋势、记录明细、睡眠报告、设备、预警、方案、随访、医生建议

中间主内容区
  当前选中模块详情

右侧决策侧栏
  当前风险、系统建议、医生操作、最近处理记录
```

### 5.2 顶部患者条

展示：

- 姓名、性别、年龄、手机号脱敏
- 当前主责医生、医生角色
- 疾病风险标签、医生确认疾病、风险等级
- 绑定设备、最近同步时间
- 当前方案、方案执行率
- 待处理预警和待随访

快捷操作建议：

- 处理预警
- 调整方案
- 发送医生建议
- 创建随访
- 转诊

### 5.3 左侧导航

建议目录：

- 总览
- 数字孪生
- 指标趋势
- 记录明细
- 睡眠报告
- 设备
- 预警
- 管理方案
- 随访
- 医生建议
- 时间轴

### 5.4 右侧决策侧栏

根据当前患者状态动态变化：

| 状态 | 主操作 |
| --- | --- |
| 紧急预警 | 处理预警、建议就医、创建随访 |
| 待确认疾病标签 | 确认疾病、标记资料不足 |
| 疑似治疗反应不足 | 复核方案、建议检查、转专科 |
| 数据连续缺失 | 发送提醒、创建随访 |
| 方案到期 | 方案复盘、创建新方案 |

### 5.5 总览模块

目标：让医生在 1 分钟内理解当前状态。

核心内容：

- 当前结论
- 疾病标签
- 近期异常
- 今日待办完成度
- 方案执行
- 最近医生建议

### 5.6 时间轴模块

时间轴按时间倒序聚合：

- 筛查
- 记录
- 设备
- 预警
- 方案
- 随访
- 医生建议

点击事件可打开详情抽屉。

## 6. 与其他模块的关系

- 患者管理列表负责筛选和导流
- 患者详情页负责聚合证据和动作
- 预警、方案、随访、建议都从患者详情继续深入

## 7. 一句话总结

**患者管理解决“找谁处理”，患者详情解决“怎么处理这个人”。**
