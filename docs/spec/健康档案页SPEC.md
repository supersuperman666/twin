# 健康档案页SPEC

> 来源：风险筛查模块 PRD、记录指标模块 PRD、患者端 SPEC 总览、医生端 SPEC 总览与本轮健康档案讨论。本文档聚焦患者小程序与医生 PC 端健康档案页的定位、入口、页面布局、字段、编辑和数据边界。

## 1. 本期目标

P0 必须跑通：

```text
患者完成健康筛查
  -> 筛查数据沉淀为健康档案
  -> 患者在小程序查看并直接补充/修改档案
  -> 医生在患者详情查看健康档案摘要、字段来源和补充信息
```

## 2. 页面定位

健康档案是围绕患者长期健康画像组织的资料页，不等同于病历，不替代日常记录页。

| 模块 | 承载内容 | 不承载内容 |
| --- | --- | --- |
| 健康档案 | 基础画像、既往史、家族史、过敏史、生活方式、筛查健康概况 | 单次诊疗文书、病程记录、长期指标明细趋势 |
| 风险筛查 | 首次评估、评分、风险标签、管理建议、建档数据采集 | 长期档案维护 |
| 记录页 | 血糖、血压、血氧、症状、用药执行等日常动态记录 | 家族史、过敏史、职业暴露史等基础档案维护 |
| 病历/诊疗记录 | 门诊、住院、随访等医疗活动记录和证据 | 患者自由维护的长期画像表单 |

产品原则：

- 健康筛查已采集的字段自动回填健康档案，不让患者重复建档。
- 风险筛查未采到或不适合强制采集的字段，在健康档案页继续补充。
- 患者自填信息、医生补录信息和系统计算信息必须区分来源。
- 健康档案中的患者自述疾病信息不直接等同于医生确认疾病。

## 5. 患者小程序健康档案页

### 5.1 页面原则

- 页面直接可编辑，不做“查看态卡片 -> 点击编辑 -> 二级编辑页”。
- 使用分区长表单，已有字段回填，缺失字段直接展示输入控件。
- 页面底部固定主按钮 `保存健康档案`。
- 保存前完成必填校验和条件字段校验。
- 发生未保存变更时，离开页面需要提示。

### 5.2 页面布局

```text
健康档案
  顶部说明
    来源说明 / 档案完整度 / 最近更新时间

  基础信息
    性别 / 出生年月日 / 年龄 / 身高 / 体重 / BMI / 腰围 / 颈围

  既往史
    已确诊慢性疾病 / 每病确诊年限 / 其他疾病 / 手术史

  家族史
    直系亲属疾病史

  过敏史
    是否过敏 / 过敏补充

  生活方式
    吸烟 / 饮酒 / 运动 / 饮食 / 睡眠 / 精神紧张 / 久坐 / 暴露史

  筛查健康概况

    最近筛查填写的血压血糖信息

  底部固定操作
    保存健康档案
```

枚举值说明：

- 与健康筛查同源字段优先沿用当前筛查字段 code 和枚举口径，避免筛查回填时二次转换。

### 5.5 患者端保存与校验

| 场景 | 规则 |
| --- | --- |
| 健康筛查回填字段修改 | 保存到健康档案，并保留最近修改时间 |
| 筛查仅有年龄或出生年份 | 不反推出生年月日；健康档案的出生年月日保持未填写，年龄仅用于筛查快照 |
| 已选慢性疾病缺少年限 | 不允许保存，定位到缺失疾病年限 |
| 选择“无”后仍保留其他疾病 | 自动互斥清理 |
| 选择“其他”但未填写疾病名称 | P0 可允许保存并提示补充；若后续用于医生确认再升级必填 |
| 数值字段超出合理范围 | 阻止保存并提示校验范围 |
| 离开页面存在未保存修改 | 弹窗确认离开或继续编辑 |

### 5.6 患者端空状态

| 状态 | 表现 |
| --- | --- |
| 未完成健康筛查 | 页面保留基础表单，顶部提示先完成筛查可自动生成更多信息 |
| 健康档案字段缺失 | 字段控件直接为空，不单独跳“完善页” |
| 无慢病 | 慢病多选显示“无”，不展示确诊年限 |
| 无过敏史 | 过敏补充字段隐藏 |

## 6. 医生 PC 健康档案模块

### 6.1 页面原则

- 健康档案作为患者详情中的背景画像模块，服务医生快速扫读。
- PC 端默认以查看态为主，编辑动作需明确触发并留痕。
- 医生端必须展示字段来源、最近更新时间和患者自填/医生确认差异。
- 诊断确认、风险确认和病历证据不在健康档案里被混成一个状态。

### 6.3 顶部健康画像摘要

## 7. 数据来源与同步规则

### 7.1 来源优先级

健康档案字段至少保留：

- 字段值
- 来源类型
- 来源详情
- 最近更新时间
- 编辑人角色

来源类型建议：

| 来源 | 示例 |
| --- | --- |
| 患者健康筛查 | 筛查问卷回填 |
| 患者健康档案编辑 | 小程序直接修改 |
| 患者日常记录 | 最近体重记录等摘要引用 |
| 医生补录/修正 | PC 端编辑 |
| 系统计算 | BMI |

### 7.2 风险筛查同步

- 首次完成健康筛查后，生成健康档案初始值。
- 重做健康筛查后，允许更新对应健康档案字段，但需保留更新来源。
- 健康档案后续修改不反向覆盖历史筛查快照。
- 筛查评分使用当次筛查快照；后续每日风险分可按最新健康档案和健康数据重新计算。

### 7.3 与记录页同步

| 字段 | 健康档案 | 记录页 |
| --- | --- | --- |
| 身高 | 档案当前值 | 基础指标引用 |
| 体重 | 档案当前值/最近摘要 | 日常记录与趋势 |
| BMI | 档案计算值 | 基础指标展示 |
| 腰围 | 档案当前值 | P0 不做高频记录趋势 |
| 血压/血糖 | 筛查概况展示筛查填写值 | 日常记录和趋势主入口 |
| 症状 | 筛查概况展示近 1 个月摘要 | 日常症状记录主入口 |
| 用药 | 筛查概况展示规律用药概况 | 用药/治疗执行记录主入口 |

### 7.4 数据模型与接口

健康档案主模型使用 `patient_health_profile`，字段底稿以 [完整字段与接口附录](/Users/ks-hz/Desktop/数字孪生/docs/prd/完整字段与接口附录.md) 为准。

| 端 | 接口 | 方法 | 用途 |
| --- | --- | --- | --- |
| 患者小程序 | `/api/profile` | GET | 获取本人健康档案、完整度和字段来源 |
| 患者小程序 | `/api/profile` | PUT | 保存本人健康档案 |
| 医生 PC | `/api/doctor/patients/{id}/profile` | GET | 获取患者健康档案、字段来源和摘要 |
| 医生 PC | `/api/doctor/patients/{id}/profile` | PUT | 医生补录或修正健康档案字段 |

接口规则：

- `PUT` 按完整表单保存，服务端需要保留被修改字段的来源和更新时间。
- 患者端保存来源写入 `patient_profile_edit`；医生端保存来源写入 `doctor_profile_edit`。
- `GET` 返回患者自填慢病和医生确认疾病时需分字段返回，不能合并成一个疾病数组。
- 健康筛查同步建档时写入 `latest_screening_id`，历史筛查快照不被健康档案编辑覆盖。

## 8. 健康档案完整度

P0 可按字段组计算，不要求医学评分化。

推荐必备字段组：

- 性别、出生年月日、身高、体重
- 慢病选择；若有慢病则每病确诊年限
- 家族史
- 过敏史
- 吸烟、饮酒
- 腰围

推荐增强字段组：

- 颈围
- 风险暴露史
- 吸烟史详情
- 睡眠质量
- 最近体检信息

完整度只表达资料完善程度，不表达健康风险高低。

## 10. P0 验收标准

- 患者健康档案页自动回填已采集的筛查字段。
- 患者健康档案页进入后即为可编辑页面，不需要先点编辑。
- 患者可在一个页面中保存基础信息、病史、家族史、过敏史、生活方式和筛查健康概况修改。
- 已选慢性疾病必须逐项填写确诊年限后才能保存。
- 未被风险筛查采集的颈围、风险暴露史等字段可在健康档案页补充。
- BMI 根据身高体重自动计算，不允许直接修改。
- 健康档案完整度不与健康风险等级混淆。
- 医生 PC 端可在患者详情查看健康档案摘要和完整分区。
- 医生 PC 端可区分患者自填、筛查回填、医生补录和系统计算来源。
- 医生确认疾病不因患者修改健康档案而自动生效。

## 11. 后续增强

- 过敏史结构化明细。
- 家族史按亲属关系拆分。
- 每个慢病补充当前是否治疗中、诊断依据和确认机构。
- 健康档案变更历史。
- 档案缺失字段生成补充资料任务。
- 医生端对关键字段增加确认/驳回和证据挂接。

## 12. 一句话总结

**健康筛查负责生成初始档案，患者健康档案页负责直接维护长期画像，医生健康档案模块负责扫读、校验和追溯来源。**
