# 数字孪生医生 PC 前端

这是医生端生产级前端工程的开发测试骨架，不再作为设计稿维护。当前版本使用浏览器本地 `localStorage` 和共享 mock 数据跑通预警、方案管理、随访计划闭环，后续可替换为真实 API。

## 本地运行

在仓库根目录运行：

```bash
python3 -m http.server 8765
```

访问：

```text
http://127.0.0.1:8765/apps/doctor-pc/
```

也可以在 `apps/doctor-pc` 目录运行：

```bash
npm run dev
```

## 当前闭环

- P0 信息架构：患者管理、预警中心、方案管理、随访计划；不单独建设工作台。
- 患者管理：首页承接轻量工作台能力，支持顶部统计卡、快捷筛选、患者列表和患者详情入口。
- 患者详情：健康档案、健康筛查、睡眠呼吸障碍患者的睡眠深度分析、预警记录、管理方案、随访记录、时间轴、医生建议。
- 预警中心：处理预警，生成随访或方案草稿，写入患者时间轴。
- 方案管理：医生确认系统草稿，下发到患者端，模拟患者知晓确认后进入执行中。
- 随访计划：新建、改期、取消、完成随访，必要时生成方案调整草稿。
- 医生建议：统一替代“医嘱与指导”作为系统内功能命名，可生成患者端待办。

## Mock 数据

共享 mock 数据位于：

```text
apps/shared/mock/health-data.js
```

医生端和后续患者端都应基于这份结构做前端数据流转，避免两端 mock 不一致。
